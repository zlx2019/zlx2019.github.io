Rust【十一】闭包

发表于2023-04-08|更新于2023-04-09|Rust
|字数总计:2.2k|阅读时长:7分钟|阅读量:

1.闭包是什么?

闭包(closure)可以理解为是一种特殊的匿名函数,但是和普通的匿名函数有所不同。

闭包的特点:

  • 是匿名函数
  • 可以定义一个闭包,赋值到一个变量或者参数
  • 可以在一个作用域定义闭包,然后传递到其他作用域进行调用
  • 闭包可以直接引用当前作用域的属性,这也是闭包最重要的特点

简而言之: 闭包就是一个匿名函数,在这个匿名函数内部直接引用了其他作用域的属性。

这里我使用Go语言举例,我们通常在Go中创建一个goroutine很少会将值通过函数的方式传递进去,而是直接访问,然后内部就会形成一个闭包。

示例:

func main() {
	name := "张三"
	go func(){
		// 直接使用父作用域的变量
		fmt.Println(name)
	}
}

2.Rust闭包

在Rust中定义匿名函数的语法如下:

let 变量名 = |参数列表|-> 返回值{
  函数体
}

示例:

fn main() {
    let name = "张三";

    // 定义一个匿名函数
    let close_fn = |age|{
        // 使用父作用域的 name属性
        println!("我的名字是: {},我的年龄是: {}",name,age);
    };
    // 调用闭包,传入参数
    close_fn(18) // 我的名字是: 张三,我的年龄是: 18
}

似乎跟其他编程语言相比,语法上略有不同,但基本上用法是一致的。

注意
匿名函数如果不指定类型和返回类型,编译器会根据调用时的入参进行推导。
如果一个闭包没有强制指定参数类型,必须要调用编译器才可以通过。

如下:


let close_fn = |age|{ // type annotations needed
};

编译器会显示类型无法推断。

闭包会在第一次被调用时进行类型推导,会将类型直接绑定到这个闭包上。

并且闭包如果存在多次调用,那么多次调用时的参数类型必须与第一次调用闭包时的入参类型。

如下:

close_fn(18);
close_fn("18") // mismatched types,expected integer, found `&str`

在第一次调用close_fn(18)时参数类型为i32,所以这个闭包的入参类型,就绑定为i32了,之后只能使用i32类型入参调用。

当然,一个闭包也可以指定其参数和返回值类型,这样就不需要依赖于编译器的推断:

let close_fn = |age: i32| -> String{
  return String::new();
};

3.闭包的使用场景

现在有这样一个函数,入参为一个i32类型参数,当这个参数值为1、2、4时就调用一个累加计算函数求出值,如果值为3就简单输出一句话即可.

fn test_closure(num: i32) {
    let double_val = match num {
        1 => {double_multiple(num)}
        2 => {double_multiple(num)}
        4 => {double_multiple(num)}
        3 => {
            println!("value is 3");
            0
        }
        _ => {0}
    };
}
// 求一个值*2
fn double_multiple(num :i32)-> i32{
    num * 2
}

代码如上,我们可以简单分析一下,这段代码有什么问题?

就目前看来,似乎有不少重复的代码,看起来不够优雅,double_multiple(num);重复编写调用了3次。那我们来简单优化一下:

fn test_closure(num: i32) {
    let val = double_multiple(num);
    let double_val = match num {
        1 => {val}
        2 => {val}
        4 => {val}
        3 => {
            println!("value is 3");
            0
        }
        _ => {0}
    };
}

值为1、2、4时都会调用该函数,索性就将函数调用放在match之前,这样看起来简洁不少。但是是不是还存在一些问题呢？

这种情况下,假如值为3的时候,其实我们是不需要执行计算函数的,但是还是会调用,相当于做了一些没有任何意义的操作。

这个时候就可以通过闭包来实现优化:

fn test_closure(num: i32) {
    // 定义闭包
    let double_fn = ||{
        num * 2
    };
    let double_val = match num {
      	// 调用闭包,获取返回值
        1|2|4 => {double_fn()}
        3=> {
            println!("value is 3");
            0
        }
        _ => {0}
    };
}

以上虽然我们定义了一个闭包,但是闭包定义并不会被执行,只有在match匹配1|2|4时才会执行闭包函数,所以不会因为值为3也会触发函数的调用。

4.闭包的类型推导

在上面我们有提到,Rust中的闭包是可以不指定参数类型和返回值类型,那么为什么函数就需要指定而闭包就不需要呢?

其原因有如下几点:

  • 函数通常是对外暴露的,所以你无法有效控制调用者是否按照指定规范来调用。
  • 而闭包通常只会在局部作用域来定义和使用,所以编译器能够有效的推导出其类型。

函数和闭包(匿名函数)的语法区别:

fn  add(x: i32) -> i32 {x + 1} // 函数
let add_fn = |x: i32| -> i32 {x + 1}; //指定类型的匿名函数
let add_fn2 = |x| {x + 1}; //不指定类型的匿名函数
let add_fn3 = |x| x + 1;   //不指定类型,并且只有一行表达式的匿名函数

整体上区别不大,唯一的就是函数标识符为(),而匿名函数为||。

5.闭包存储到结构体

首先我们来看一段代码:

fn test_closure(num: i32) {
    // 定义闭包
    let double_fn = |num|{
        num * 2
    };
    // 多次调用闭包
    let n1 = double_fn(3);
    let n2 = double_fn(3);
    let n3 =  double_fn(3);
}

这段代码来自上面的示例改造而来,简单来说就是同一个闭包,入参的值完全一致,导致多次调用执行,造成资源浪费。

我们可以通过struct来实现闭包缓存优化此案例。

在结构体中定义闭包变量

在结构体字段中的闭包的类型必须是明确指定的。并且每个闭包的实例都有自己唯一的匿名类型，即使两个闭包签名完全一致。

闭包字段需要通过泛型来指定,至少实现以下一种trait:

  • Fn
  • FnMut
  • FnOnce

定义如下:

struct Cacher<T>
    // 指定闭包的泛型
    where T: Fn(i32)-> i32
{
    // 闭包字段
    double_fn: T,
    // 闭包的执行结果,当闭包执行过一次后,将值存储到这里,如果没有执行过,那么value则是None
    value: Option<i32>
}

impl<T> Cacher<T> where T: Fn(i32)-> i32 {
    // Cacher的构造函数,传入闭包
    fn new(double_fn: T)-> Cacher<T>{
        Cacher { double_fn: double_fn, value: None }
    }
  
  	// 获取闭包的值,如果没有值就执行闭包,并且将值缓存到value中
    fn value(&mut self,arg: i32)-> i32{
        // 判断Option是否有值
        match self.value {
            // 直接返回
            Some(v) => v,
            None => {
                // 没有值,执行闭包,将结果赋值到value中
                let v = (self.double_fn)(arg);
                self.value = Some(v);
                v
            }
        }
    }
}

使用该结构体优化示例:

fn test_closure(num: i32) {
    // 定义闭包
    let double_fn = |num|{
        num * 2
    };
    // 将闭包绑定到结构体中
    let mut cacher = Cacher::new(double_fn);
    
    // 多次调用闭包,如果有值,那么会直接获取值，而不会重复执行闭包。
    let n1 = cacher.value(num);
    let n2 = cacher.value(num);
    let n3 = cacher.value(num);
}

6.闭包的特征描述

闭包的描述一共有三种特征,就是我们之前提到的Fn、FnMut、FnOnce三种特征,它们获取到的具体内容也是不同的.

  • FnOnce: 表示闭包将捕获它们需要的环境变量的所有权（Ownership），所以每个变量只能被捕获一次。一旦闭包获取了所有权，环境变量就不能再被访问。
  • FnMut: 表示闭包将捕获引用（References）到其环境中的变量，并且可以通过可变引用（mutable reference）来修改它们。
  • Fn: 表示闭包将捕获引用（References）到其环境中的变量，但是不可变地借用它们。

在创建闭包时,编译器会根据闭包内部对值的使用推断出具体使用哪种trait。

  • 闭包都默认实现了FnOnce特征。
  • 如果闭包内部对捕获的值进行了修改操作,那么实现的就是FnMut特征。
  • 如果闭包内部只是访问了值,并没有修改,也没有移动所有权.那么实现的就是Fn特征。

move

在闭包参数列表前,使用move关键字修饰,可以强制闭包取得它所使用的环境值的所有权。

如下:

let arr = vec![1,2,3,4,5,6,7,8,9];
// 定义一个闭包
// 直接强制获取arr的所有权
let eq_fn = move ||{
  println!("{:?}",arr)
};
// 在这里再次使用arr,则会编译不通过
println!("{:?}",arr); //borrow of moved value: `arr`

使用建议

当你不知道使用那种Fn特征时,建议首先使用Fn特征,然后随着你的使用,如果有需要使用FnOnce或者FnMut时,编译器会通知你。
