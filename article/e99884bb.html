Flutter【页面路由】

发表于2022-05-04|更新于2023-03-22
|字数总计:1k|阅读时长:4分钟|阅读量:

前言

FLutter中的路由，和原生组件化的路由一样，就是页面之间的跳转，也可以称之为导航。app维护一个路由栈，路由入栈(push)操作对应打开一个新页面，路由出栈(pop)操作对应页面关闭操作，而路由管理主要是指如何来管理路由栈。

Flutter为我们提供了导航器Navigator。参数传入当前的BuildContext和要导航的页面即可。

1. 页面准备

我们创建三个页面 Home、User、About,用这三个页面来演示Flutter的路由。

2. 页面跳转

通过Fultter内置的Navigator类进行跳转。使用该类的push方法,返回要跳转的页面的Widget。实现从Home跳转到User,从User跳转到About,再从ABout返回到User。

Home修改

class Home extends StatefulWidget {
  const Home({Key? key}) : super(key: key);
  @override
  State<Home> createState() => _HomeState();
}
class _HomeState extends State<Home> {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text("首页界面"),),
      body: Center(
        child: ElevatedButton.icon(onPressed: (){
          /// 触发该事件,跳转到 User页面
          Navigator.push(context, MaterialPageRoute(builder: (context){
            return const User();
          }));
        }, icon: const Icon(Icons.home), label: const Text("Home"))
      ),
    );
  }
}

User

/// 创建用户页面
class User extends StatefulWidget {
  const User({Key? key}) : super(key: key);
  @override
  State<User> createState() => _UserState();
}

class _UserState extends State<User> {
  @override
  Widget build(BuildContext context) {

      appBar: AppBar(title: const Text("用户界面"),),
      body: Center(
          child: ElevatedButton.icon(onPressed: (){
            /// 触发该事件,跳转到 ABout 页面
            Navigator.push(context, MaterialPageRoute(builder: (context){

            }));

}

About

class ABout extends StatefulWidget {
  const ABout({Key? key}) : super(key: key);

class _ABoutState extends State<ABout> {

      body: Center(
          child: ElevatedButton.icon(onPressed: (){
            /// 退回到上一个页面: 出栈
            Navigator.pop(context);

    );
  }
}

3. 路由表

MaterialApp默认提供了路由表功能,相比较下,这种方式更常用更灵活。

class MyApp extends StatelessWidget {
  const MyApp({Key? key}):super(key: key);
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Material App',
      // 创建路由表  路由path -- 对应的页面Widget
      routes: {
        '/home': (context) => const Home(),
        '/user': (context) => const User(),
        '/about': (context) =>  ABout(),
      },
      home: Home(),
      //初始化默认显示的路由
      initialRoute: "/home",
    );
  }
}

有了路由表后,直接根据路由名称跳转即可.

///跳转到 ABout 页面
Navigator.pushNamed(context, "/about");
///跳转到 User页面
Navigator.pushNamed(context, "/user");

4. 路由参数

路由跳转时,有个可选参数,可以携带数据。

Navigator.pushNamed(context, "/user",arguments: "Zero");

在路由后的页面获取。

var param = ModalRoute.of(context)?.settings.arguments;

5. MaterialApp 路由属性

MaterialApp(
  title: 'Material App',
  // 创建路由表  路由path -- 对应的页面Widget
  routes: {
    '/home': (context) => const Home(),
    '/user': (context) => const User(),
    '/about': (context) =>  ABout(),
  },
  //首页页面
  home: const Home(),
  // initialRoute： 表示如果没有指定home属性,就使用initialRoute指定的路由为一开始显示的页面。
  //如果指定了home属性,则先将home页面压入路由栈,再将initialRoute指定的页面压入,表示一开始显示initialRoute页面,但是首页还是home页面
  initialRoute: "/home",
  //每次路由都会执行的函数
  onGenerateRoute: (RouteSettings settings){
    var routeName = settings.name;//要跳转的路由名称
  },
  //找不到路由执行的函数
  onUnknownRoute: (RouteSettings settings){
  },
);
