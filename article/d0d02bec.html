Jackson深度学习

前言

Fastjson ? 懂得都懂 漏洞多到修不完,Jackson才是永远的神!

Jackson是一个开源的Java 序列化及反序列化JSON的开源框架,但是功能可不单单只序列化JSON,功能很完善,

扩展性极高,并且与另一款不错的序列化工具gson相比,Jackson的效率较快,内存占用也较低,更是受到Spring框架的认可,将Jackson选为默认的序列化工具。

1. 模块

Jackson核心模块分为三部分

  • jackson-core，核心包，提供基于”流模式”解析的相关 API，它包括 JsonPaser 和 JsonGenerator。Jackson 内部实现正是通过高性能的流模式 API 的 JsonGenerator 和 JsonParser 来生成和解析 json。
  • jackson-annotations，注解包，提供标准注解功能。
  • jackson-databind ，数据绑定包， 提供基于”对象绑定” 解析的相关 API （ ObjectMapper ） 和”树模型” 解析的相关 API （JsonNode）；基于”对象绑定” 解析的 API 和”树模型”解析的 API 依赖基于”流模式”解析的 API。

2. 依赖引入

<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.12.3</version>
</dependency>

3. 统一配置

  1. 基础配置
//在反序列化时忽略在 json 中存在但 Java 对象不存在的属性
mapper.configure(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, false);
//在序列化时日期格式默认为 yyyy-MM-dd'T'HH:mm:ss.SSSZ
mapper.configure(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS, false);
//在序列化时自定义时间日期格式
mapper.setDateFormat(new SimpleDateFormat("yyyy-MM-dd HH:mm:ss"));
//在序列化时忽略值为 null 的属性
mapper.setSerializationInclusion(JsonInclude.Include.NON_NULL);
//在序列化时忽略值为默认值的属性
mapper.setDefaultPropertyInclusion(JsonInclude.Include.NON_DEFAULT);
  1. JDK8时间类配置
/** Java8 LocalDateTime 序列化处理配置及格式定义*/
mapper.registerModule(new Jdk8Module());
JavaTimeModule module = new JavaTimeModule();
module.addDeserializer(LocalDateTime.class,new LocalDateTimeDeserializer(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")));
module.addSerializer(LocalDateTime.class,new LocalDateTimeSerializer(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")));
mapper.registerModule(module);

4. 基础API

ObjectMapper 通过 writeValue 系列方法将 java 对象序列化为 json，并将 json 存储成不同的格式，String（writeValueAsString），Byte Array（writeValueAsString），Writer， File，OutStream 和 DataOutput。

ObjectMapper 通过 readValue 系列方法从不同的数据源像 String ， Byte Array， Reader，File，URL， InputStream 将 json 反序列化为 java 对象。

  1. Model To Json

    private void BeanToJson(ObjectMapper mapper, Person person)throws Exception {
  String value = mapper.writerWithDefaultPrettyPrinter().writeValueAsString(person);// 将POJO 序列化为 JSON
  System.out.println(value);
  JsonToBean(mapper,value);
}

  2. Json To Model

    private void JsonToBean(ObjectMapper mapper, String value)throws Exception {
    Person person = mapper.readValue(value, Person.class);
    System.out.println(person);
}

  3. List To Json

    Person person = new Person("Tom",19, null,1,false,"admin",999);
List<Person> people = Arrays.asList(person);
String value = mapper.writerWithDefaultPrettyPrinter().writeValueAsString(people);
System.out.println(value);

  4. Json To List (一)

    CollectionType collectionType = mapper.getTypeFactory().constructCollectionType(List.class, Person.class);
List<Person> list = mapper.readValue(value, collectionType);
System.out.println(list);

  5. Json To List (二)

    List<Person> personList = mapper.readValue(value, new TypeReference<List<Person>>() {});
System.out.println(personList);

  6. Json To Map

    ObjectMapper mapper = new ObjectMapper();
//第二参数是 map 的 key 的类型，第三参数是 map 的 value 的类型
MapType javaType = mapper.getTypeFactory().constructMapType(HashMap.class, String.class, Person.class);
// 造数据
Map<String, Person> map = new HashMap<>();
for (int i = 0; i < 3; i++) {
  Person person = new Person();
  person.setName("Tom");
  person.setAge(new Random().nextInt(100));
  person.setDate(new Date());
  map.put("key" + i, person);
}
System.out.println("序列化");
String jsonInString = mapper.writerWithDefaultPrettyPrinter().writeValueAsString(map);
System.out.println(jsonInString);
System.out.println("反序列化: 使用 javaType");
Map<String, Person> personMap = mapper.readValue(jsonInString, javaType);
System.out.println(personMap);
System.out.println("反序列化: 使用 TypeReference");
Map<String, Person> personMap2 = mapper.readValue(jsonInString, new TypeReference<Map<String, Person>>() {
});

5. 注解说明

@JsonIgnore

用于告诉Jackson忽略Java对象的某个属性（字段）。将JSON读入Java对象时，以及将Java对象写入JSON时，都会忽略该属性。

@JsonIgnoreProperties

该注释是类注释,通常定义在类上,可以指定该类序列化时忽略的属性集合

@JsonSetter

通常是指 JSON中的属性名与Java Bean属性名不一致,但是要绑定上时使用,将JSON中对应的属性名,通过该注解定义到Bean与之对象的属性的setter方法中,如下举例

{
  "parentId": 1001,
  "name" : "Tom",
  "age" : 19
}
public class Person {
  	private Integer id;
    private String name;
    private Integer age;
  	@JsonSetter("parentId") //绑定Json中对应的属性名
    public void setId(String id) {
      this.id = id;
    }
}

这样序列化就可以将 parentId 序列化到 id上

@JsonSerialize

自定义序列化规则,例如 将Boolean序列化成 0和1

首先自定义序列化器

/**
 * 自定义序列化规则,将false 转换为 0,true 转换为 1
 */
public class OptimizedBooleanSerializer extends JsonSerializer<Boolean> {

    @Override
    public void serialize(Boolean aBoolean, JsonGenerator jsonGenerator, SerializerProvider serializerProvider) throws IOException {
        if (aBoolean) {
            jsonGenerator.writeNumber(1);
        } else {
            jsonGenerator.writeNumber(0);
        }
    }
}

局部使用,直接在属性上定义即可

@JsonSerialize(using = OptimizedBooleanSerializer.class)
private Boolean enabled = false;

全局注册

SimpleModule simpleModule = new SimpleModule();
simpleModule.addSerializer(Boolean.class,new OptimizedBooleanSerializer());
mapper.registerModule(simpleModule);
@JsonDeserialize

自定义反序列化规则 将 0转为 false 1转为 true

public class OptimizedBooleanDeserialize extends JsonDeserializer<Boolean> {
    @Override
    public Boolean deserialize(JsonParser jsonParser, DeserializationContext deserializationContext) throws IOException, JsonProcessingException {
        String text = jsonParser.getText();
        return !text.equals("0");
    }
}

使用方式与JsonSerialize一致

@JsonProperty

用于属性，把属性的名称序列化时转换为另外一个名称。如userName 序列化为user_name

@JsonFormat

用于属性或者方法，把属性的格式序列化时转换成指定的格式。示例：@JsonFormat(timezone = “GMT+8”, pattern = “yyyy-MM-dd HH:mm”) public Date getBirthDate()

@JsonNaming

类注解。序列化的时候该注解可将驼峰命名的字段名转换为下划线分隔的小写字母命名方式。反序列化的时候可以将下划线分隔的小写字母转换为驼峰命名的字段名。示例：@JsonNaming(PropertyNamingStrategy.SnakeCaseStrategy.class)
