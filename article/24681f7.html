Python异步编程

前言

关于Coroutine(协程)的详细内容,这里就不多赘述,我在Coroutine这篇博客里有讲述过,本篇主要讲述关于Python中的协程使用。

关于协程,简单描述一句: 简而言之 其实就是在一个线程中实现代码块相互切换执行。

在Python中有很多种实现协程的方式,随着版本的更新这项技术愈发成熟,下面从旧到新的顺序来介绍一。

正常代码都是从上至下顺序执行,一个接一个的函数执行完成后,才会进入下一个函数执行.如下:

def task1():
    print("task1 start")
    print(1)
    print(2)
    print("task1 end")


def task2():
    print("task2 start")
    print(3)
    print(4)
    print("task2 end")


task1()
task2()

像这种同步执行,执行结果不用多想肯定是如下:

task1 start
1
2
task1 end
task2 start
3
4
task2 end

先执行完task1()函数,然后执行task2()函数。

如何使用协程来优化这个小示例呢? 比如让task1()函数执行一半时,切换到task2()函数执行。

实现方案如下:

  • greenlet
  • yield
  • asyncio (>py3.4)
  • async、await (>py3.5)【推荐】

1. greenlet

greenlet是一个第三方模块,使用前我们需要先下载该模块:

pip3 install greenlet

然后通过greenlet模块来实现案例的修改:

from greenlet import greenlet

def task1():
    print("task1 start")
    print(1)
    t2.switch()   # 切换到t2。开始执行task2()函数 (2)
    print(2)
    print("task1 end")
    t2.switch()   # 再次切换到t2,继续执行task2()函数 (4)
    
def task2():
    print("task2 start")
    print(3)
    t1.switch()   # 切换到t1,继续执行task1()函数 (3)
    print(4)
    print("task2 end")

t1 = greenlet(task1)
t2 = greenlet(task2)

# 将当前线程执行权,转交给t1。开始执行task1()函数 (1)
t1.switch()

执行结果:

task1 start
1
task2 start
3
2
task1 end
4
task2 end

可以看到,这两个被greenlet所包装的函数任务,在一个线程中交替执行。

这种方式是在太过于复杂笨重,一般不会使用到此方式。

2. yield

这种方式就不介绍了,不推荐，实际应用场景比较少。

3. asyncio

asyncio是Python3.4版本推出的新模块,是官方推荐的协程实现方案。这个模块基于事件循环(Event loop)来编排回调异步任务。

实现如下:

import asyncio


# @asyncio.coroutine 表示为协程函数
@asyncio.coroutine
def task1():
    print("task1 start")
    print(1)
    yield from asyncio.sleep(2)  # 当前协程睡眠2秒。协程遇到io阻塞时会释放到当前执行权
    print(2)
    print("task1 end")


# @asyncio.coroutine 表示为协程函数
@asyncio.coroutine
def task2():
    print("task2 start")
    print(3)
    yield from asyncio.sleep(2)  # 当前协程睡眠2秒
    print(4)
    print("task2 end")


# 创建多个协程任务函数
tasks = [
    asyncio.ensure_future(task1()),
    asyncio.ensure_future(task2())
]

# 创建一个事件循环
loop = asyncio.get_event_loop()
# 运行协程任务函数,并等待所有协程执行结束
loop.run_until_complete(asyncio.wait(tasks))

@asyncio.coroutine 语法装饰器可以被用于定义一个协程函数。

注意: 在Python3.11以后的版本,这种方式已经被移除了.

4. async await

在Python3.5后推出了async和await关键字,通过该关键字来实现asyncio方案的落地。这种方式是目前最常用也是最简洁的。通过这种方案实现如下:

import asyncio

# 异步函数
async def task1():
    print("task1 start")
    print(1)
    await asyncio.sleep(2)  # 当前协程睡眠2秒。协程遇到io阻塞时会释放到当前执行权
    print(2)
    print("task1 end")

# 异步函数
async def task2():
    print("task2 start")
    print(3)
    await asyncio.sleep(2)  # 当前协程睡眠2秒
    print(4)
    print("task2 end")


# 创建一个事件循环对象
loops = asyncio.new_event_loop()

# 创建多个异步任务,封装为一个列表
tasks = [
    asyncio.ensure_future(task1(), loop=loops),
    asyncio.ensure_future(task2(), loop=loops)
]

# 执行任务列表,等待所有任务执行完成
loops.run_until_complete(asyncio.wait(tasks))

执行结果:

task1 start
1
task2 start
3
2
task1 end
4
task2 end

asyncio 介绍

asyncio模块是Python3.4版本引入的标准库模块,它提供了一种基于协程的并发编程模型。可以让开发者更方便编写更高效的异步IO代码。在使用在模块之前,请先了解以下概念:

  • Event Loop

    这是asyncio模块的核心组件事件循环(Event Loop)。它负责管理异步任务的调度和协调，同时也是异步 IO 的基础。

    其本质上就是一个无限的循环,在开启之前,需要将对应的事件函数注册给该组件,当触发指定的条件时,它会异步调用事件对应的回调函数。

  • Coroutine

    协程是asyncio中核心的概念,它是被async关键字修饰的函数，可以在执行过程中暂停和恢复执行，与线程相比，协程更轻量级，更高效。它通常需要注册到Event Loop中,由Event Loop所调度。

  • Task

    一个Coroutine就是一个原生可以挂起的函数，任务则是对协程进一步封装，其中包含任务的各种状态。

  • Future

    代表将来执行或没有执行的任务的结果。它和task上没有本质的区别

  • async

    async关键字用于定义一个Coroutine(协程函数)。如下:

    # 定义一个协程函数
async def loadUser():
    pass

  • await

    await表示阻塞当前协程函数的执行,等待另一个协程函数执行结束。所以await关键字只能在协程函数内部使用,在普通函数内是无法使用的。

    await关键字后面通常会指定一个可等待的对象(支持异步协议的对象)（如Future、Task、Coroutine)

    如下:

    async def clear():
    pass


async def loadUser():
    # ...
    # 阻塞等待 clear()函数执行结束
    await clear()
    #...

异步编程

快速使用

通过async创建一个协程函数

async def showName():
    print("Hello World")

那么如何去运行一个协程函数呢?如果想执行一个协程函数,就必须通过asyncio模块来执行。

第一种方式

通过Event Loop来执行协程函数:

# 创建一个事件循环实例
loop = asyncio.new_event_loop()
# 通过事件循环来执行 异步函数
loop.run_until_complete(showName()) # Hello World

第二种方式

这种方式更加简单,是第一种方式的优化简写方式,这是Python3.7版本推出的更简洁的用法。

通过asyncio.run()函数来执行一个协程函数。

asyncio.run(showName()) #Hello World

async

被async关键字修饰的函数就是协程函数。

async def func():
  pass

协程对象就是通过协程函数获取到的对象。如下:

async def func():
  pass

# 获取func()的 协程对象
result = func()
print(type(result)) # <class 'coroutine'>

注意: func()并不会执行协程函数,只是获取它的协程对象。函数体并不会执行。

有了协程对象,就可以通过事件循环对象来执行协程:

async def func():
  print("Func")
# 获取func()的 协程对象
task = func()
asyncio.run(task) # Func

await

await只能用在异步函数内,用于阻塞式执行一个异步函数.

示例：

import asyncio

# 获取username的异步函数
async def get_username():
    await asyncio.sleep(3)
    return "张三"


async def call():
    print("call start.")
    # 阻塞等待 get_username()函数执行完成,获得结果
    username = await get_username()
    print(f"username: ${username}")
    print("call end.")

# 执行call异步函数
asyncio.run(call())

Task

Task对象是asyncio模块提供的一种并发原语,一个Task通常由一个协程对象包装而成,用Task表示一个协程的执行状态。Task对象可以被取消、暂停和恢复，以及检查协程是否已完成或出现异常等。Task对象可以用于协程的控制流程，例如等待协程完成、挂起协程、恢复协程等。此外Task对象还可以与asyncio的事件循环结合使用，实现协程的并发执行。

创建Task

可以通过asyncio.create_task()方法创建一个Task,需要传入一个协程对象:

import asyncio
# 定义一个异步方法
async def get_username():
    await asyncio.sleep(2)
    return "Hello"
# 定义异步main方法
async def main():
    # 根据一个协程对象,创建一个task任务。
    # 并且将该task任务直接注册到事件循环中
    task = asyncio.create_task(get_username())
    # 阻塞等待task任务执行完成
    username = await task
    print(f'username: {username}') # username: Hello

# 执行main方法
asyncio.run(main())

需要注意的是:

  • 执行完asyncio.create_task(get_username())这行代码,这个创建的Task会直接添加到事件循环中开始执行,并不是调用了await task才会开始执行。
  • await task只是为了阻塞等待这个task任务执行结束,确保拿到了结果值。

而且asyncio.create_task()方法是Python3.7提供的,如果小于该版本,需要通过asyncio.ensure_future()来创建Task对象:

task = asyncio.ensure_future(get_username())

等待多任务执行

如果我们有多个异步任务,并且想获取所有的任务结果可以如下这样实现:

import asyncio

async def get_username():
    await asyncio.sleep(2)
    return "Hello"

async def main():
    # 创建多个task任务,注册到事件循环中
    task_list = (
        asyncio.create_task(get_username()),
        asyncio.create_task(get_username()),
        asyncio.create_task(get_username())
    )
    # 阻塞等待所有task任务执行结束
    # tasks_result为所有的任务结果集合
    tasks_result, pending = await asyncio.wait(task_list,timeout=None)
    # 遍历每个任务的结果
    for item in tasks_result:
        print(item.result())

# 执行main方法
asyncio.run(main())

通过asyncio.awit(task[])来阻塞等待多个Task任务执行结束.参数timeout=None表示没有超时时间。

执行结果:

Hello
Hello
Hello

返回值为两个,分别为成功和失败的任务返回结果集合,如下:

import asyncio

# 任务1 睡眠1秒
async def task1():
    await asyncio.sleep(1)
    return "task1的结果"

# 任务2 睡眠2秒
async def task2():
    await asyncio.sleep(2)
    return "task2的结果"

# 任务3 睡眠4秒
async def task3():
    await asyncio.sleep(4)
    return "task3的结果"


async def main():
    # 创建多个task任务并且指定名称,注册到事件循环中
    task_list = (
        asyncio.create_task(task1(),name="t1"),
        asyncio.create_task(task2(),name="t2"),
        asyncio.create_task(task3(),name="t3"),
    )
    # 阻塞等待所有task任务执行结束,超时时间限制为3秒
    # success_result: 成功的任务结果列表
    # success_result: 失败的任务结果列表
    success_result, fail_result = await asyncio.wait(task_list,timeout=3)

    # 处理成功的任务结果
    for item in success_result:
        print(f'{item.get_name()}任务执行成功,结果为: {item.result()}')
    # 处理失败的任务结果
    for item in fail_result:
        print(f'{item.get_name()}任务执行失败, {item.done()}')
# 执行main方法
asyncio.run(main())

执行结果:

t1任务执行成功,结果为: task1的结果
t2任务执行成功,结果为: task2的结果
t3任务执行失败, False

Future

Future是asyncio执行异步任务所产生的一个结果对象,用它来承载异步任务所返回的结果。通常并不会用到该对象,因为它过于底层,并不需要开发者所关注.

Futures

在Python中的concurrent.futures模块中,也有一个Future对象,这两个对象是不同的,请注意不要搞混了.

  • asyncio.Future是用于接收异步函数的返回值。
  • concurrent.futures.Future是用于接收线程池或进程池任务的返回值。

线程池简单使用如下:

import time
from concurrent.futures import ThreadPoolExecutor


def task(number):
    time.sleep(number)
    print(number)
    return f'Task Result {number}'


# 创建线程池,先吃数量:5
pool = ThreadPoolExecutor(max_workers=5)

# 使用线程池,执行普通函数
# 第一个参数为函数,后面为函数的参数
fut = pool.submit(task,3)


print(type(fut)) # <class 'concurrent.futures._base.Future'>
# 当前任务状态
print(fut.done()) # False
# 任务结果
print(fut.result()) # Task Result 3

Future交叉使用

对于一些场景是否能用异步编程完全取决于实际情况,有些情况下并不推荐使用异步编程。例如: 对Mysql操作的情况下(Mysql目前并不支持异步式),所以哪怕程序层面是异步的,依然会在这一步阻塞住,所以这种情况下使用异步编程完全没有意义。

如下:

import asyncio
import time

# 数据访问函数
def data_base():
    # 模拟io
    time.sleep(3)
    return "root"


async def main():
    # 获取事件循环对象
    loop = asyncio.get_running_loop()
    # 使用默认的线程池来执行任务
    # 然后将concurrent中的Future转换为asyncio中的Future
    fut = loop.run_in_executor(None,data_base)
    # 阻塞等待这个任务执行完成,并且获取结果
    result = await fut
    print(f"result: {result}")

# 执行异步函数
asyncio.run(main())

异步和非异步案例

案例: 使用异步编程,来下载网络图片,这里会涉及到http请求,所以要转为非异步方式下载。

# 异步与非异步混合使用案例
import asyncio
import requests


# 异步函数,下载网络图片
async def download_image(img_url, img_name):
    print(f"开始下载 --> {img_url}")
    # 获取异步事件循环
    loop = asyncio.get_event_loop()
    # 由于要发起http请求,所以要使用到request模块,但是这个模块又是非异步的
    # 所以只能通过线程池来执行请求
    # 通过requests.get 发起http请求
    future = loop.run_in_executor(None, requests.get, img_url)
    # 阻塞等待任务完成
    response = await future
    print(f"下载结束 --> {img_url}")

    # 将图片写入到本地
    with open(img_name, mode='wb') as file:
        file.write(response.content)


if __name__ == '__main__':
    # 创建事件循环
    loop = asyncio.new_event_loop()
    # 将三个异步任务添加到事件循环中
    tasks = [
        loop.create_task(download_image("https://zlx2019.github.io/img/logo/Python-logo.png", "python.png")),
        loop.create_task(download_image("https://zlx2019.github.io/img/logo/Coroutine-logo.png", "co.png")),
        loop.create_task(download_image("https://zlx2019.github.io/img/logo/Vertx-logo.png", "vertx.png"))
    ]
    # 执行任务
    loop.run_until_complete(asyncio.wait(tasks))

异步迭代器

什么是异步迭代器？

实现了__aiter__和__anext__方法的对象被称之为异步可迭代对象。

__aiter__表示获取异步可迭代对象的异步迭代器。

__anext__表示获取下一个迭代元素。直到引发一个StopAsyncIteration异常,表示已经没有元素.

import asyncio
# 异步迭代器

# 自定义可异步迭代对象
class Reader:
    # 构造函数
    def __init__(self):
        self.count = 0

    # 获取该对象的异步迭代器
    def __aiter__(self):
        return self

    # 生成下一个迭代元素
    async def read_next(self):
        self.count +=1
        if self.count >= 100:
            return None
        return self.count

    # 异步获取下一个迭代元素
    async def __anext__(self):
        # 获取下一个迭代元素
        val = await self.read_next()
        if val == None:
            # 如果为None,表示已经没有下一个元素
            raise StopAsyncIteration
        return val


# 遍历异步可迭代对象
async def consumer_reader():
    # 创建一个异步可迭代对象
    read = Reader()
    # 异步遍历可迭代对象
    async for item in read:
        print(item)

# 运行异步函数
asyncio.run(consumer_reader())

异步控制器

在传统开发中,我们一般涉及到io操作,例如打开一个文件,为了防止资源浪费都会及时的关闭文件句柄,我们都会使用with关键字来实现自动释放。但是如果是异步编程,with是不支持的,需要使用async with组合关键字才能实现异步的控制。如下所示:

传统方式:

# 打开一个文件
with open("book.txt") as file:
    str = file.read()
    print(str) #Hello

异步方式:

创建一个对象分别实现__aenter__和__aexit__方法,该对象被async with管理时,会自动执行这两个方法。

import asyncio


# 自定义异步上下文控制器,用异步的方式来读取一个文件内容
class AsyncContextManager:
    # 构造函数,指定要读取的文件名
    def __init__(self ,file_name,mode):
        self.file_name = file_name
        self.mode = mode

    # 在这里实现所需要的异步初始化,如连接db、打开文件、建立连接等
    async def __aenter__(self):
        print("__aenter__  初始化")
        # 打开一个文件,获取到文件句柄
        self.file = await asyncio.to_thread(open,self.file_name,self.mode)
        return self

    # 异步释放资源
    async def __aexit__(self, exc_type, exc_val, exc_tb):
        print("__aexit__ 释放资源")
        self.file.close()


# 使用异步上下文控制器,来读取一个文件内容
async def async_open_file():
    # 通过async with 自动打开文件
    # 这里的 `file`其实就是AsyncContextManager的__aenter__方法返回值
    async with AsyncContextManager("book.txt",'r') as file:
        # 读取文件内容
        content = file.file.read()
        print(f'文件内容: {content}')
    # 作用域结束后.会自动执行AsyncContextManager的__aexit__方法


asyncio.run(async_open_file())

执行结果:

__aenter__  初始化
文件内容: Hello
__aexit__ 释放资源

在以上代码中我们使用到了asyncio.to_thread(),它表示将一个同步函数转换为异步函数,并且在事件循环中执行该函数。

当我们在异步代码中需要调用一个同步函数时，如果我们直接调用它，那么它将会阻塞事件循环，导致性能下降。使用asyncio.to_thread()可以将该同步函数放入一个新的线程中执行，避免了阻塞事件循环。

或者也可以这样来实现:

self.file =  await asyncio.get_event_loop().run_in_executor(None,open,self.file_name,self.mode)

该方式也是将一个同步函数转为异步函数,并且阻塞式获取函数结果。

asyncio.to_thread() 和 loop.run_in_executor() 都可以用来执行同步函数，但它们的使用方式略有不同。

asyncio.to_thread() 将同步函数放入一个新的线程中执行，并返回一个协程对象，可以通过 await 关键字或 asyncio.ensure_future() 函数来执行。

loop.run_in_executor() 将同步函数放入一个线程池中执行，并返回一个 Future 对象，可以通过 await 关键字或 asyncio.ensure_future() 函数来获取执行结果。

uvloop

uvloop是个高性能的异步非阻塞框架，他是完整的asyncio事件循环的替代品，建立在libuv基础之上，由Cython编写而成。Uvloop最终目的使得Asyncio更加快速。

安装

pip install uvloop

使用也很

简单,直接替换即可:

import asyncio
import uvloop
# 将默认的eventloop 替换为uvloop
asyncio.set_event_loop_policy(uvloop.EventLoopPolicy())


# 内部的事件循环 会自动变为uvloop
asyncio.run(...)
