Postgresql学习之旅【一】基础语法篇

发表于2021-04-23|更新于2023-03-22|Postgresql系列
|字数总计:2.5k|阅读时长:10分钟|阅读量:

1.前言

PostgreSQL是以加州大学伯克利分校计算机系开发的POSTGRES， 版本 4.2为基础的对象关系型数据库管理系统（ORDBMS）。POSTGRES 领先的许多概念在很久以后才出现在一些商业数据库系统中。

2.连接pgsql

  • 使用pgsql连接数据库语法格式如下:

    psql [option…] [dbname] [username]

    其中option是连接相关参数,psql是postgresql提供的终端命令,在安装目录bin文件下,例:

    psql -h101.37.79.47 -p5432 -Uroot -d postgres
    • -h: 数据库的IP地址
    • -p:数据库端口
    • -U: 登录的用户名
    • -d: 数据库名

3.常用命令

  • “\d” 查看当前数据库中所有表、视图、序列

  • “\d [table_name]” 在”\d”后面指定表名,则可以查看该表的定义结构

  • “\db” 查看表空间信息

    postgres=> \db
      List of tablespaces
    Name    | Owner | Location 
------------+-------+----------
 pg_default | l     | 
 pg_global  | l     |

  • “\dn” 查看所有模式信息

    postgres=> \dn
List of schemas
  Name  | Owner 
--------+-------
 public | l
(1 row)

  • “\du” 或者 “\dg” 查看数据库中所有角色和用户

    postgres=> \dg
                                   List of roles
 Role name |                         Attributes                         | Member of 
-----------+------------------------------------------------------------+-----------
 l         | Superuser, Create role, Create DB, Replication, Bypass RLS | {}
 root      |                                                            | {}

  • “\?” 查看Postgresql提供的所有命令

  • “\timing” 打开or关闭sql执行时间的开关

4.表空间

​ 如果需要把不同的表放在不同的存储介质或不同的文件系统下,则要用到 ‘表空间’ 的概念,在pgsql中,表空间为表等对象指定一个存储的目录

  • 创建表空间

    • 简单创建表空间

      空间名称: tbs_root

      对应物理文件路径: /usr/local/pgsql/pgdata/root_table_space(该文件夹要事先创建好)

      CREATE TABLESPACE tbs_root LOCATION '/usr/local/pgsql/pgdata/root_table_space';

      注意: pgsql不推荐将表空间建立在 数据目录,单独抽离最好

  • 创建表空间(space01)并且赋予权限给指定用户(root)

    create tablespace space01 owner root location '/usr/local/pgsql/tables_space/root/space01';

  • 修改表空间

    • 修改原有表空间(space01)名称 为 space02

      ALTER TABLESPACE space01 RENAME TO space02;

    • 修改表空间所属用户为 ‘root’

      alter tablespace space01 owner to root;

  • 删除表空间

    空间名称: tbs_root

    DROP TABLESPACE IF exists tbs_root;

    删除表空间必须保证该空间的所有数据已经清除,否则命令执行失败

5. 数据库操作

5.1 创建数据库

语法格式如下

CREATE DATABASE _name
  [[WITH][OWNER[=]_user_name]
  [TEMPLATE [=]_template]
  [ENCODING[=]_encoding]
  [LC_COLLATE[=]_lc_collate]
  [LC_CTYPE[=]_lc_ctype]
  [TABLESPACE[=]_table_space]
  [ALLWCONN[=]_allwconn]
  [CONNECTION LIMIT[=]_connlimit]
  [IS_TEMPLATE[=]_istemplate]
  • _name : 数据库名
  • _user_name: 该数据库所属者用户名
  • _template: 使用的模板库,默认的模板库是template1
  • _encoding: 字符集,默认使用模板库的字符集
  • _lc_collate: 该数据库使用的排序规则,这会影响Order by 语法中字符串类型的顺序,也会影响text类型的索引顺序,默认跟模板库一致
  • _lc_ctype: 该数据库的字符分类,会影响字符的分类
  • _table_space: 关联的表空间
  • _allwconn: 是否可以连接数据库
  • _connlimit: 允许并发连接的数据库个数,默认为-1 表示没有限制
  • _istemplate: 是否是模板库,默认为false
# 创建root_db数据库,所属者是root,关联space01表空间
create database root_db 
owner root
tablespace space01表空间;

5.2 修改数据库

#修改数据库名
alter database db_test rename to db_test03
#修改最大连接数
alter database db_test connection limit 100;

5.3 删除数据库

drop database if existe db_name

6. 表结构操作

6.1 创建表

  • 创建一个名为dept的表

    create table dept(
	dept_id int  NOT NULL,
	dept_name varchar NOT NULL,
	create_time TIMESTAMP 
)

  • Check约束 create_time必须大于指定的日期

    create table dept(
	dept_id int  NOT NULL,
	dept_name varchar NOT NULL,
	create_time TIMESTAMP  CONSTRAINT create_time_check CHECK(create_time > '2020-01-01 00:00:00')
)

  • 表级的Check约束,字段与字段之间的约束

    #create_time必须晚于parent_create_time
create table dept(
	dept_id int  NOT NULL,
	dept_name varchar NOT NULL,
	create_time TIMESTAMP,
	parent_id int NOT NULL,
	parent_create_time TIMESTAMP,
	CHECK(create_time > parent_create_time)
)

  • 唯一约束 [UNIQUE]

    #表示 dept_id 是唯一的,不存在相同的情况
create table dept(
	dept_id int  UNIQUE,
	dept_name varchar NOT NULL,
	create_time TIMESTAMP
)
#或
create table dept(
	dept_id int  not null,
	dept_name varchar NOT NULL,
	create_time TIMESTAMP,
  unique(dept_id)
)

  • 主键约束 [PRIMARY KEY]

    create table dept(
	dept_id int  PRIMARY KEY,
	dept_name varchar NOT NULL,
	create_time TIMESTAMP
)

  • 联合主键约束

    create table dept(
	dept_id int  not NULL,
	dept_no int not NULL
	dept_name varchar NOT NULL,
	create_time TIMESTAMP,
	PRIMARY KEY(dept_id,dept_no)
)

  • 默认值约束[DEFAULT]

    create table dept(
	dept_id int  not NULL,
	dept_name varchar DEFAULT '暂无',
	create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
)

  • 外键约束

    #创建emp表,其中dept_id为物理外键, REFERENCES 指定父表
create table emp(
	emp_id int PRIMARY KEY,
	emp_name VARCHAR not NULL,
	dept_id int not NULL REFERENCES dept,
	create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)

6.2 修改表

  • 修改表名

    alter table emp rename to emp_02;

  • 修改字段名

    ALTER TABLE emp RENAME COLUMN `username` to `nikename`;

  • 添加字段

    ALTER TABLE emp ADD COLUMN `_create_time` TIMESTAMP DEFAULT CURRENT_TIMESTAMP;

  • 删除字段

    ALTER TABLE emp DROP COLUMN `create_time`;

6.3 删除表

#删除单表
drop table if exists emp;
#删除多表
drop table if exists emp,dept;

7.CRUD

7.1 插入数据(Insert)

  • 指定字段插入数据

    insert into dept(id,name,create_time) values (1001,'人事部','2020-05-19 12:30:00')

  • 不指定字段插入数据,需要按字段顺序添加,必须填写全部字段

    insert into dept VALUES(1002,'技术部','2021-04-10')

  • 如果一张表所有字段都可以为空时,可以执行以下语句,表示所有字段都取默认值

    insert into dept DEFAULT VALUES;

  • 批量插入数据

    insert into dept (id,name,create_time)VALUES
(1003,'业务部','2021-04-10'),
(1004,'市场部','2021-04-10'),
(1005,'管理部','2021-04-10')

8. 数据类型

Postgresql 的数据类型非常丰富,除数据库常见的数据类型(如数值、字符串)还有一些高级类型,如JSON、几何类型等.

8.1 数值类型

  • 整数类型

    | 类型名称 | 存储容量 | 说明 | 取值范围 |
    | :———- | ———— | ——————— | ———————————————————— |
    | smallint | 2字节 | 小范围整数 | -32768~32767 |
    | integer | 4字节 | 整数的常见选择 | -2147483648~2147483647 |
    | bigint | 8字节 | 大范围整数 | -9223372036854775808~9223372036854775807 |

  • 任意精度数值类型

    | 类型名称 | 存储容量 | 说明 | 取值范围 |
    | ———— | ———— | ————————- | ———————————————————— |
    | numeric | 可变 | 用户指定精度,精确 | 最高小数点前131072位,以及小数点后16383位 |
    | decimal | 同上 | 同上 | 同上 |

  • 浮点类型

    | 类型名称 | 存储容量 | 说明 | 取值范围 |
    | ———————— | ———— | ———————- | ——————— |
    | real | 4字节 | 可变精度,不精确 | 6位十进制精度 |
    | Double precision | 8字节 | 同上 | 15位十进制精度 |

  • 序列类型

    序列类型并不是真正的数据类型,通常我们需要使用自增主键的时候可以直接使用以下三种类型

    | 类型名称 | 存储容量 | 说明 | 取值范围 |
    | —————- | ———— | ———————— | ——————————- |
    | smallserial | 2字节 | 自动增加的小整数 | 1~32767 |
    | serial | 4字节 | 自动递增的整数 | 1~2147483647 |
    | bigserial | 8字节 | 自动递增的大整数 | 1~9223372036854775807 |

  • 货币类型

    | 类型名称 | 存储容量 | 说明 | 取值范围 |
    | ———— | ———— | ——————— | ————————————————————— |
    | Money | 8字节 | 货币、金钱类型 | -92233720368547758.08~92233720368547758.07 |

    lc_monetary默认为美元,需要将其设置为 'zh_CN.utf8'

8.2 字符串类型

类型名称 说明
char 固定长度字符串,长度不足时以空格填充字节
varchar 有长度限制的变长字符串
text 无长度限制的变长字符串

  • 连接运算符

    | 运算符 | 返回类型 | 说明 | 示例 | 结果 |
    | ——————————- | ———— | —————————— | ————————— | —————— |
    | string||string | text | 拼接 | ‘post’||‘greSql’ | ‘postgreSql’ |
    | string|| not-string | text | 使用非字符串进行拼接 | ‘Val’||98 | ‘Val98’ |

  • 匹配运算符

    • LIKE模糊匹配

      select '今天天气真好' like '%天气%' #R: t

    • SIMILAR TO

    • POSIX正则表达式

8.3 日期类型和时间类型

类型名称 存储容量 说明 Min_val Max_val 分辨率
timestamp 8字节 包括日期和时间(无时区) 4713BC 294276AD 1毫秒/14位
timestamp 8字节 包含日期和时间(有时区) 同上 同上 同上
date 4字节 只包含日期 同上 5874897AD 一天
time 8字节 只包含时间 0:00:00 24:00:00 1毫秒/14位
time 12字节 只包含时间(带时区) 0:00:00+1459 24:00:00-1459 同上
interval 16字节 时间间隔 -178000000年 178000000年 1毫秒/14位

8.4 布尔类型

  • PostgreSQL 布尔类型有三种状态

    • true 真
    • false 假
    • unknown 位置

  • 所占字节为 ‘1’字节

  • 代表“真” 和 “假”可以使用的值

    | 真 | 假 |
    | ——— | ———- |
    | ‘t’ | ‘f’ |
    | ‘true’ | ‘false’ |
    | ‘y’ | ‘n’ |
    | ‘yes’ | ‘no’ |
    | ‘on’ | ‘off’ |
    | ‘1’ | ‘0’ |

  • 比较谓词运算符

    | 举例 | 说明 |
    | ———————————————————— | ———————————————— |
    | ‘a’ BETWEEN ‘x’ AND ‘y’ | a 在 x和y之间 |
    | ‘a’ NOT BETWEEN ‘x’ AND ‘y’ | a不在 x和y之间 |
    | ‘a’ BETWEEN SYMMETRIC ‘x’ AND ‘y’ | 在对比较值排序后,位于 x和y之间 |
    | ‘a’ NOT BETWEEN SYMMETRIC ‘x’ AND ‘y’ | 在对比较值排序后,不位于 x和y之间 |
    | ‘a’ IS DISTINCT from ‘b’ | a不等于b |
    | ‘a’ IS NOT DISTINCT from ‘b’ | a 等于 b |
    | ‘a’ IS NULL | a是空值 |
    | ‘a’ IS NOT NULL | a不是空值 |
    | ‘a’ is true | a为真 |
    | ‘a’ is not true | a为假 或 未知 |
    | ‘a’ is false | a为假 |
    | ‘a’ is not false | a为真 或 未知 |
    | ‘a’ is unknown | a 为 未知 |
    | ‘a’ is not unknown | a 为真 或 假 |

8.6 枚举类型

  • 创建枚举

    CREATE TYPE week AS ENUM ('周一','周二','周三','周四','周五','周六','周日');

  • 创建表,使用该枚举作为字段类型

    create table course(
	name VARCHAR,
	c_week week
)

  • 表DDL

    CREATE TABLE "public"."course" (
  "name" varchar COLLATE "pg_catalog"."default",
  "c_week" "public"."week"
);

  • 插入类型为枚举的数据

    insert into course VALUES ('数学课','周一')
insert into course VALUES ('英语','周二')

    c-week 字段必须插入定义的枚举才可以,否则无法插入数据

    insert into course VALUES ('未知课','周八')
#ERROR:  invalid input value for enum week: "周八"
#LINE 1: insert into course VALUES ('未知课','周八')

  • 根据枚举作为顺序条件查询

    • 枚举的大小之分是该枚举创建时所列出的值顺序而定
    • 每一种枚举都是独立的,无法和其他枚举类型进行比较

    查询周三以后的课程

    SELECT * from course where c_week > '周三'

    查询一周内最早一天和最晚一天的课程 

    SELECT * from course where c_week = (select MIN(c_week) FROM course)
SELECT * from course where c_week = (select Max(c_week) FROM course)

    根据枚举进行排序 

    SELECT * from course ORDER BY c_week DESC
