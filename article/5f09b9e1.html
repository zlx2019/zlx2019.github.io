Java【工具类合集详解】

发表于2022-07-18|更新于2023-03-22|Java

前言

本章详细介绍一些工具类的使用

IoUtils

Apache的common-io包是业界非常常用的IO工具包,不少客户端工具都是基于此依赖之上进行开发。

依赖引入

<dependency>
  <groupId>commons-io</groupId>
  <artifactId>commons-io</artifactId>
  <version>2.11.0</version>
</dependency>

copy

数据拷贝

  • copy(InputStream, OutputStream)

    以流的方式拷贝,将输入流中的数据拷贝到输出流中。返回拷贝的字节数。该方式会直接覆盖输出流中原有数据。如果数据量过大,超过2G,则返回-1。

    FileInputStream inputStream = new FileInputStream(FILE_PATH + "Hello.txt");
FileOutputStream outputStream = new FileOutputStream(FILE_PATH + "NewHello.txt");
int bytes = IOUtils.copy(inputStream, outputStream);

  • copy(Reader r,Writer w)

    以reader和writer方式拷贝。返回拷贝字节数。如果大于2G则返回-1。

    直接覆盖原有数据

    InputStreamReader reader = new InputStreamReader(Files.newInputStream(Paths.get(FILE_PATH + "Hello.txt")));
OutputStreamWriter writer = new OutputStreamWriter(Files.newOutputStream(Paths.get(FILE_PATH + "NewHello.txt")));
int bytes = IOUtils.copy(reader, writer);
writer.flush();

    追加式拷贝数据

    // 以Reader及Writer方式拷贝
FileReader fileReader = new FileReader(FILE_PATH + "Hello.txt");
// true,表示为追加模式,不会覆盖原有数据
FileWriter fileWriter = new FileWriter(FILE_PATH + "NewHello.txt", true);
int bytes = IOUtils.copy(fileReader, fileWriter);
fileWriter.flush();

  • copy(inputStream,outputStream,bufferSize)

    以流的方式拷贝,可指定缓冲区大小的数据拷贝。

  • copy(InputStream is,Writer w)

    使用平台的默认字符编码将字节从InputStream复制到Writer上的字符。

  • copy(InputStream,is,Writer w,InputCharset ic)

    使用指定的字符编码将InputStream中的字节复制到Writer上的字符。

  • copy(InputStream is,Writer w,String inputCharsetName)

    使用指定的字符编码将InputStream中的字节复制到Writer上的字符。

  • copy(Reader reader,Appendable output)

    将字符从Reader复制到Appendable。

  • copy(Reader r,Appendable output,CharBuffer buffer)

    将字符从Reader复制到Appendable。此方法使用提供的缓冲区，因此无需使用BufferedReader。

  • copy(Reader r,OutputStream os)

    使用平台的默认字符编码将字符从Reader复制到OutputStream上的字节，并调用 flush。

  • copy(Reader reader, OutputStream output,Charset outputCharset)

    使用指定的字符编码将字符从Reader复制到OutputStream上的字节，并调用 flush。

  • copy(Reader reader,OutputStream output,String outputCharsetName)

    使用指定的字符编码将字符从Reader复制到OutputStream上的字节，并调用 flush。

  • copyLarge(InputStream is, OutputStream os)

    拷贝较大的数据流。将字节从大（超过 2GB）的InputStream复制到OutputStream 。

copyLarge

大量数据拷贝

  • copyLarge(InputStream is,OutputStream os)

    将字节从大（超过 2GB）的InputStream复制到OutputStream。此方法在内部缓冲输入，因此无需使用BufferedInputStream。

  • copyLarge(final InputStream inputStream, final OutputStream outputStream, final byte[] buffer)

    将字节从大（超过 2GB）的InputStream复制到OutputStream 。此方法使用提供的缓冲区，因此无需使用BufferedInputStream 。

  • copyLarge(final Reader reader, final Writer writer)

    将字符从大型（超过 2GB） Reader复制到Writer。此方法在内部缓冲输入，因此无需使用BufferedReader。

toString

以String形式获取输入流。

  • toString(final InputStream input, final Charset charset)

  • toString(final InputStream input, final String charsetName)

    将输入流根据指定的编码格式转换为String类型。

    FileInputStream fileInputStream = new FileInputStream(FILE_PATH + "Hello.txt");
String string = IOUtils.toString(fileInputStream, StandardCharsets.UTF_8);
String string1 = IOUtils.toString(fileInputStream, "utf-8");

  • toString(final Reader reader)

  • toString(final URI uri)

  • toString(final URI uri, final Charset encoding)

  • toString(final URI uri, final String charsetName)

toByteArray

以byte[]形式获取输入流。

  • toByteArray(final InputStream inputStream)

  • toByteArray(final InputStream input, final int size)

  • toByteArray(final InputStream input, final long size)

  • toByteArray(final Reader reader)

  • toByteArray(final Reader reader, final Charset charset)

  • toByteArray(final String input)

    使用平台的默认字符编码以byte[]形式获取String的内容

  • toByteArray(final URI uri)

Paths

通常用来构建Path资源对象的工具类。

// 获取文件Path
Path path1 = Paths.get("C:", "path", "to", "nowhere", "NoFile.txt");
Path path2 = Paths.get("C:/path/to/nowhere/NoFile.txt");

一个 Path 对象表示一个文件或者目录的路径，是一个跨操作系统和文件系统的抽象，目的是在构造路径时可以不必关注底层操作系统。如果 Path 仅由一个空的名称元素组成，则认为是空的路径。使用空路径访问文件等同于访问文件系统的默认目录。Path 接口的实现是不可变且安全的，可供多个并行线程使用

使用 Path 对象可以非常容易地生成路径的某一部分：

Path path = Paths.get(FILE_PATH+"Hello.txt");

Files

从Java7开始,开发包内提供了java.nio.file.Files工具类,能够极大提供我们IO及File操作的效率。

1.获取文件信息

public static void main(String[] args) throws IOException {
  Path path = Paths.get(FILE_PATH+"Hello.txt");
  // 文件是否存在
  boolean exists = Files.exists(path);
  // 文件是否是目录
  boolean directory = Files.isDirectory(path);
  // 文件是否可执行
  boolean executable = Files.isExecutable(path);
  // 文件是否可读
  boolean readable = Files.isReadable(path);
  // 文件是否可写
  boolean writable = Files.isWritable(path);
  // 文件是否是隐藏文件
  boolean hidden = Files.isHidden(path);
  // 获取文件大小
  long size = Files.size(path);
  // 获取文件存储相关信息
  FileStore fileStore = Files.getFileStore(path);
  // 获取文件上次修改时间
  FileTime lastModifiedTime = Files.getLastModifiedTime(path);
  // 获取文件所属者
  UserPrincipal owner = Files.getOwner(path);
  // 文件是否是符号链接
  Files.isSymbolicLink(path);
  // 获取目录中的所有子文件Path
  Stream<Path> list = Files.list(Paths.get(FILE_PATH));
  // 移动文件,如果再桶目录移动则为重命名
  Path move = Files.move(filePath, Paths.get(FILE_PATH + "name.txt"));
}

2.创建文件和目录

// 创建文件 如果文件已存在,则抛出异常。
Path path = Files.createFile(Paths.get(FILE_PATH + "User.java"));
// 创建目录 如果目录已存在,则抛出异常。
Path directory = Files.createDirectory(Paths.get(FILE_PATH + "users"));
// 创建多级目录,目录存在则不执行操作
Path directories = Files.createDirectories(Paths.get(FILE_PATH + "orders/info"));

3.删除文件和目录

// 删除文件或目录,如果不存在则抛出异常
Files.delete(Paths.get(FILE_PATH+"User.java"));
// 如果文件或目录存在,则删除
boolean exists = Files.deleteIfExists(Paths.get(FILE_PATH + "orders/info"));

4.读取文件内容

// 以byte[]数组形式读取所有文件内容
// 从文件中读取所有字节。该方法确保在读取所有字节或抛出 I/O 错误或其他运行时异常时关闭文件。
// 请注意，此方法适用于方便将所有字节读入字节数组的简单情况。它不适用于读取大文件。
byte[] bytes = Files.readAllBytes(filePath);
// 从文件中读取所有行。此方法确保在读取所有字节或引发 I/O 错误或其他运行时异常时关闭文件。使用指定的字符集将文件中的字节解码为字符。
List<String> list = Files.readAllLines(filePath, StandardCharsets.UTF_8);
// 从文件中读取所有行作为Stream 。与readAllLines不同，此方法不会将所有行读入List ，而是在使用流时延迟填充。
// 使用指定的字符集将文件中的字节解码为字符，并且支持readAllLines指定的相同行终止符。
Stream<String> lines = Files.lines(filePath, StandardCharsets.UTF_8);

5.写入文件内容

// 将Hello 写入到文件内
Path write = Files.write(newFilePath, "Hello".getBytes(), StandardOpenOption.WRITE, StandardOpenOption.APPEND);

// 将List字符串数组 分行写入文件内
List<String> lines = Arrays.asList("Hello", "world");
Files.write(newFilePath,lines, StandardCharsets.UTF_8,StandardOpenOption.WRITE, StandardOpenOption.APPEND);

6.文件内容拷贝

直接覆盖原文件

public static void main(String[] args) throws IOException {
  Path filePath = Paths.get(FILE_PATH + "Hello.txt");
  Path newFilePath = Paths.get(FILE_PATH + "NewHello.txt");
  // 将filePath内容 拷贝到newFilePath文件中,直接覆盖
  Path copy = Files.copy(filePath, newFilePath, StandardCopyOption.REPLACE_EXISTING);
}

7.打开文件

打开文件时,可以添加选项,标准选项如下:

public enum StandardOpenOption implements OpenOption {
    READ, //打开以进行读取访问。
  
    WRITE,//打开以进行写访问。
  
  	//如果打开文件以进行WRITE访问，则字节将被写入文件的末尾而不是开头。
		//如果文件被其他程序打开以进行写访问，那么如果写入文件末尾是原子的，则它是文件系统特定的。
    APPEND,
  
  	//如果文件已经存在并且它被打开以进行WRITE访问，则其长度被截断为 0。如果文件仅以READ访问权限打开，则忽略此选项。
    TRUNCATE_EXISTING,
  
  	//如果不存在，则创建一个新文件。如果还设置了CREATE_NEW选项，则忽略此选项。检查文件是否存在以及如果文件不存在则创建文件对于其他文件系统操作而言是原子操作。
    CREATE,
  
  	//创建一个新文件，如果文件已经存在则失败。检查文件是否存在以及如果文件不存在则创建文件对于其他文件系统操作而言是原子操作。
    CREATE_NEW,
  
  	//关闭时删除。当此选项存在时，实现会尽最大努力在通过适当的close方法关闭文件时尝试删除文件。
    DELETE_ON_CLOSE,
  
  	//稀疏文件。当与CREATE_NEW选项一起使用时，此选项会提示新文件将是稀疏的。当文件系统不支持创建稀疏文件时，该选项将被忽略。
    SPARSE,
  
  	//要求对文件内容或元数据的每次更新都同步写入底层存储设备。
    SYNC,
  	//要求对文件内容的每次更新都同步写入底层存储设备。
    DSYNC;
}
获取流
// 打开文件,获取输入流及读取权限
InputStream inputStream = Files.newInputStream(filePath, READ);

// 打开文件,获取输入流及写和追加操作权限
OutputStream outputStream = Files.newOutputStream(filePath, WRITE, APPEND);

结合原生方式,从一个文件读取数据写入到另一个文件内:

public static void main(String[] args) throws IOException {
  Path filePath = Paths.get(FILE_PATH + "Hello.txt");
  Path newFilePath = Paths.get(FILE_PATH + "NewHello.txt");

  // 打开文件,获取输入流及读取权限
  InputStream inputStream = Files.newInputStream(filePath, READ);
  // 打开文件,获取输入流及写和追加操作权限
  OutputStream outputStream = Files.newOutputStream(newFilePath, WRITE, APPEND);
  // 缓冲载体
  byte[] buf = new byte[1024];
  // 记录每次读取的字节数量
  int len;
  // 循环读取,直到读取结束: len == -1
  while ((len = inputStream.read(buf,0,buf.length)) != -1){
    // 循环写入
    outputStream.write(buf,0,len);
  }
  // 关闭流
  inputStream.close();
  outputStream.close();
}
获取Reader和Writer
// 打开文件,获取文件BufferedReader
BufferedReader reader = Files.newBufferedReader(filePath, StandardCharsets.UTF_8);
// 打开文件,获取文件BufferedWriter及写和追加写操作权限
BufferedWriter writer = Files.newBufferedWriter(newFilePath, StandardCharsets.UTF_8,WRITE,APPEND);

结合原生方式,从一个文件读取数据写入到另一个文件内:

public static void main(String[] args) throws IOException {
  Path filePath = Paths.get(FILE_PATH + "Hello.txt");
  Path newFilePath = Paths.get(FILE_PATH + "NewHello.txt");

  // 打开文件,获取文件BufferedReader
  BufferedReader reader = Files.newBufferedReader(filePath, StandardCharsets.UTF_8);
  // 打开文件,获取文件BufferedWriter及写和追加写操作权限
  BufferedWriter writer = Files.newBufferedWriter(newFilePath, StandardCharsets.UTF_8,WRITE,APPEND);
  char[] buf = new char[1024];
  int len;
  // 开始读取并且写入
  while ((len = reader.read(buf)) != -1){
    writer.write(buf,0,len);
  }
  // 或者如下写入操作 二选一即可
  int copy = IOUtils.copy(reader, writer);
  reader.close();
  writer.close();
}
获取BufferedReader和BufferedWriter
// 打开一个文件进行读取，返回一个BufferedReader可用于以有效的方式从文件中读取文本。
BufferedReader bufferedReader = Files.newBufferedReader(filePath, StandardCharsets.UTF_8);
// 打开或创建一个文件进行写入，返回一个BufferedWriter以高效的方式将文本写入文件。文本被编码为字节以使用UTF-8 charset进行写入。
BufferedWriter bufferedWriter = Files.newBufferedWriter(newFilePath, WRITE, APPEND);
public static void main(String[] args) throws IOException {
  Path filePath = Paths.get(FILE_PATH + "Hello.txt");
  Path newFilePath = Paths.get(FILE_PATH + "NewHello.txt");
  // 打开一个文件进行读取，返回一个BufferedReader可用于以有效的方式从文件中读取文本。
  BufferedReader reader = Files.newBufferedReader(filePath, StandardCharsets.UTF_8);
  // 打开或创建一个文件进行写入，返回一个BufferedWriter以高效的方式将文本写入文件。文本被编码为字节以使用UTF-8 charset进行写入。
  BufferedWriter writer = Files.newBufferedWriter(newFilePath, WRITE, APPEND);
  // 写入方式1
  String line = null;
  while ((line = reader.readLine()) != null){
    writer.write(line);
  }
  // 写入方式2
  int copy = IOUtils.copy(reader, writer);
  reader.close();
  writer.close();
}
