1.注解描述

@ApplicationScoped

将当前类注册为一个Bean。其作用域为单例,在整个应用中只有一个实例。并且它是懒加载模式,只有在初次使用时才会实例化。使用最频繁的一个Bean注解。

@Singleton

将当前类注册为一个Bean。其作用域为单例。与@ApplicationScoped不同的是,@Singleton是没有经过代理类的,并且他是注入时就实例化好的。官方更推荐使用@ApplicationScoped。但是@Singleton性能更好。

@RequestScoped

将当前类注册为一个Bean。其作用域为绑定http请求: 每个http请求时都会产生一个全新的实例。

@SessionScoped

将当前类注册为一个Bean。作用域为Session级别,同一个会话范围内共享一个实例。

@Dependent

将当前类注册为一个Bean。每次依赖注入都会产生一个新的实例。

Rest请求注解

@GET 、@PUT 、@DELETE、@POST、@PATCH、OPTIONS,分别映射rest风格中不同类型的请求。

@PathParam与@RestPath

映射uri中的参数。不同的是@RestPath可以不指定属性名称,默认以参数名映射。而@PathParam必须指定请求中的属性名称。

@Path("/{userId}")
public SysUser get(@PathParam("userId") Long id){}

@Path("/{userId}")
public SysUser get(@RestPath Long userId){}

@QueryParam与@RestQuery

映射http请求中的query参数。@RestQuery默认以参数名映射。@QueryParam必须指定属性名称。

(@QueryParam("nikeName") String nikeName)
(@RestQuery String nikeName)

@HeaderParam与@RestHeader

映射请求头中的参数。跟上面同理,一个简化后的一个非简化后的。

@FormParam与@RestForm

映射请求表单参数。同理。

@CookieParam与@RestCookie

映射请求Cookie参数。

@Path

标识资源类或类方法将为请求提供服务的 URI 路径。类似SpringBoot中的@RequestMapping。

@Consumes

指定可以接受的媒体类型。如果未指定表示任何媒体类型都可以接受。标注在方法上会覆盖类上的标注。

简单来说就是设置该资源类下的所有API请求可以接受的http携带资源媒体类型,如application/json、text/plain。

@Produces

指定生成的媒体类型。如果未指定，则容器将假定可以生产任何类型。方法级注释覆盖类级注释。

简单来说就是设置响应的数据格式,如Json。等同于SpringBoot中的@ResponseBody。

@Inject

依赖注入。等同于Spring中的@Autowired。

可注入成员可以具有任何访问修饰符（private、package-private、protected、public）。

标识可注入的构造函数、setter方法和属性。

@Default

注入同一个类型多实例时,依赖注入的默认实例。等同于Spring中的@Primary。

@Named

多Bean实例时,指定Bean的名称,在依赖注入时也要通过该注解指定名称。

2.构建基础结构

1. Entity

@Data
public class SysUser implements Serializable {

    private static final long serialVersionUID = 1L;
    /** 用户ID*/
    private Long id;
    /** 用户名*/
    @NotBlank(message = "用户名不能为空!")
    private String userName;
    /** 密码*/
    @NotBlank(message = "密码不能为空!")
    private String passWord;
    /** 用户昵称*/
    private String nickName;
    /** 年龄*/
    @Min(value = 18,message = "年龄最小不能低于18岁")
    private Integer age;
    /** 创建时间*/
    @JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")
    private LocalDateTime createTime;
    /** 是否冻结*/
    private boolean locked;
}

2. Service

@Slf4j
@ApplicationScoped
public class UserService {

    /** 模拟数据库*/
    private static final Map<Long, SysUser> Container = new ConcurrentHashMap<>();

    /**
     * 获取所有用户信息
     */
    public List<SysUser> list(){
        return new ArrayList<>(Container.values());
    }

    /**
     * 添加用户
     */
    public boolean add(SysUser sysUser){
        if (Container.containsKey(sysUser.getId())){
            throw new RuntimeException("该用户ID已存在!");
        }
        Container.put(sysUser.getId(),sysUser);
        return Boolean.TRUE;
    }

    /**
     * 修改用户
     */
    public boolean update(SysUser sysUser){
        if (Container.containsKey(sysUser.getId())){
            throw new RuntimeException("用户不存在,无法修改");
        }
        Container.put(sysUser.getId(),sysUser);
        return Boolean.TRUE;
    }

    /**
     * 删除用户
     */
    public boolean remove(Long id){
        Container.remove(id);
        return Boolean.TRUE;
    }
}

3. Controller

// 标记请求和响应都是以Json数据格式

@Path("/users")
@Produces(MediaType.APPLICATION_JSON)
@Consumes(MediaType.APPLICATION_JSON)
public class UserController {

    /**
     * 依赖注入userService
     */
    @Inject
    UserService userService;

    /**
     * 查询所有用户列表
     */
    @GET
    public List<SysUser> list(){
        return userService.list();
    }
    /**
     * 根据Id 获取用户信息
     * 解析uri中的{id}参数,映射到id属性中
     */
    @GET
    @Path("/{id}")
    public SysUser get(@PathParam("id") Long id){
        return userService.get(id);
    }

    /**
     * 根据用户昵称 匹配用户信息
     * 解析请求中的query参数nikeName 映射到属性中
     */
    @GET
    @Path("/search")
    public List<SysUser> search(@RestQuery String nikeName){
        return userService.search(nikeName);
    }

    /**
     * 添加用户
     * 默认会将requestBody 映射到实体中
     */
    @POST
    public boolean add(@Valid SysUser sysUser){
        return userService.add(sysUser);
    }

    /**
     * 修改用户信息
     * 默认会将requestBody 映射到实体中
     */
    @PUT
    public boolean update(@Valid SysUser sysUser){
        return userService.update(sysUser);
    }

    /**
     * 删除用户
     * 解析uri中的{id}参数,映射到id属性中
     */
    @DELETE
    @Path("/{id}")
    public boolean delete(@RestPath Long id){
        return userService.remove(id);
    }
}

4.拦截器和过滤器

Quarkus的请求与响应分别通过ContainerRequestFilter和ContainerResponseFilter实现拦截。这些过滤器适用于处理与消息相关的元数据：HTTP 标头、查询参数、媒体类型和其他元数据。它们还具有中止请求处理的能力，例如当用户没有访问端点的权限时。

如下定义,具体业务自己实现即可:

/**
 * 请求过滤器
 * @author Zero.
 * @date 2022/7/13 2:24 PM
 */
@Provider
@Slf4j
public class CustomRequestFilter implements ContainerRequestFilter {

    @Override
    public void filter(ContainerRequestContext requestContext) throws IOException {
        Request request = requestContext.getRequest();
        log.info("接收到请求!");
    }
}

/**
 * 响应过滤器
 * @author Zero.
 * @date 2022/7/13 2:26 PM
 */
@Provider
@Slf4j
public class CustomResponseFilter implements ContainerResponseFilter {

    @Override
    public void filter(ContainerRequestContext requestContext, ContainerResponseContext responseContext) throws IOException {
        Object entity = responseContext.getEntity();
        log.info("响应的结果:{}",entity);
    }
}

5.跨域配置

Quarkus已经提供了关于跨域的拦截器,我们只需要在配置文件中开启并且配置即可。

quarkus.http.cors=true
quarkus.http.cors.origins=http://foo.com,http://www.bar.io
quarkus.http.cors.methods=GET,PUT,POST
quarkus.http.cors.headers=X-Custom
quarkus.http.cors.exposed-headers=Content-Disposition
quarkus.http.cors.access-control-max-age=24H

默认只要开启即可,这些参数默认不填就是允许所有。

6.统一异常处理

自定义业务异常类

public class BusinessException extends RuntimeException{
    public BusinessException(String message){
        super(message);
    }
    public BusinessException(Exception e){
        super(e);
    }
    public BusinessException(String message,Exception e){
        super(message,e);
    }
}

自定义返回类型

public class Result<T> implements Serializable {
  private static final long serialVersionUID = 1L;
  /** 响应数据*/
  private T data;
  /** 响应消息*/
  private String message;
  /** 响应码 0:操作执行成功 1:操作执行异常*/
  private int code;
  /** 操作成功*/
  public static <T> Result<T> Success(T data, String message, int code){
    return new Result<>(data,message,code);
  }
  public static <T> Result<T> Success(T data){
    return new Result<>(data,"操作成功",0);
  }
  /** 操作失败*/
  public static <T> Result<T> Failed(String message,int code){
    return new Result<>(null,message,code);
  }
}

定义统一异常处理类

@Provider
@Slf4j
public class DefaultExceptionHandler implements ExceptionMapper<Exception> {
    /**
     * 异常处理
     * @param e the exception to map to a response.
     */
    @Override
    public Response toResponse(Exception e) {
        String message = "未知异常";
        if (e instanceof BusinessException){
            message = Objects.isNull(e.getMessage()) ? "业务异常" : e.getMessage();
        }
        Result<?> result = Result.Failed(message, 1);
        return Response.status(Response.Status.INTERNAL_SERVER_ERROR).entity(result).type(MediaType.APPLICATION_JSON).build();
    }
}

}

需要注意的是,针对每个实例,我们需要定义一个特定的注解,通过该注解进行依赖注入时的区分。在注解上必须标注@Qualifier注解。

然后在使用时,也要通过该特定注解,才能注入指定的实例。

@Inject @Named("alipay")
IPayment alipay;

@Inject @Named("wechat")   
IPayment wechat;

或者

@Inject @Named("wechat")   
Instance<IPayment> factory;

public void test(){
  factory.get().info();
}

有点麻烦,不知道还有没有更好的方案~

策略模式+工厂模式 动态获取实例实现

他*的,花了半天时间才搞定.直接上代码:

// 接口
public interface IPayment {}

// 实现
@ApplicationScoped
@Named("alipay")
public class Alipay implements IPayment {
      @Override
    public void info() {
        System.out.println("alipay");
    }
}

// 实现
@ApplicationScoped
@Named("wechat")
public class Wechat implements IPayment {
    @Override
    public void info() {
        System.out.println("wechat");
    }
}


// 实例工厂
@ApplicationScoped
public class PayFactory {
    /**
     * 将IPayment的实例 全部注入到List中
     */
    @Inject
    @All
    List<InstanceHandle<IPayment>> factory;
    /**
     * 根据不同的Named.value属性 获取不同的实例
     * @param name  beanName
     * @return      实例
     */
    public IPayment getInstance(String name){
        return factory.stream().filter(i-> name.equals(i.getBean().getName()))
                .map(InstanceHandle::get).findFirst().orElseThrow(()-> new BusinessException("没有找到该实例!"));
    }
}

使用:

@Inject
PayFactory factory;
factory.getInstance("alipay").info();
factory.getInstance("wechat").info();

1.上传

创建一个映射体,用来解析请求中的文件

@Data
public class FileEntity {
  /**
   * 上传的文件
   * 指定请求表单中的属性名及数据类型为流
   */
  @FormParam("file")
  @PartType(MediaType.APPLICATION_OCTET_STREAM)
  public InputPart file;
}

上传接口定义

/**
 * 文件上传接口
 * 指定接收媒体类型 MULTIPART_FORM_DATA
 */
@POST
@Path("upload")
@Consumes(MediaType.MULTIPART_FORM_DATA)
public Result<String> fileUpload(@MultipartForm FileEntity entity) throws IOException {
  // 获取文件流
  InputStream fileStream = entity.getFile().getBody(InputStream.class, null);
  // 获取文件名
  String fileName = getFileName(entity.getFile());
  log.info("上传的文件名称:{}",fileName);
  log.info("上传文件内容:{}", IOUtils.toString(fileStream, StandardCharsets.UTF_8));
  return Result.Success("ok");
}

/**
  * 根据文件对象 获取实际文件名
  * @param file  文件对象
  * @return      文件名
  */
private String getFileName(InputPart file) {
  String[] contentDisposition = file.getHeaders().getFirst("Content-Disposition").split(";");
  String fileName = "";
  for (String filename : contentDisposition) {
    if ((filename.trim().startsWith("filename"))) {
      String[] name = filename.split("=");
      fileName = name[1].trim().replaceAll("\"", "");
    }
  }
  return fileName;
}

2.下载

/**
 * 文件下载接口
 * @return 响应文件流
 */
@GET
@Path("/download")
@Produces(MediaType.APPLICATION_OCTET_STREAM)
public Response fileDownload(){
  // 加载要下载的文件
  File file = FileUtils.getFile("/Users/zero/Desktop/你好.txt");
  // 对文件名称进行编码,防止下载后中文乱码
  String fileName = URLEncoder.encode(file.getName(), StandardCharsets.UTF_8);
  // 将文件以流形式响应客户端
  return Response.ok(file)
    .header("content-disposition", "attachment; filename=\"" + fileName + "\"")
    .header("Content-Length", file.length())
    .build();
}
