Kotlin语法糖

发表于2023-04-21|更新于2023-04-26|Kotlin
|字数总计:1.7k|阅读时长:6分钟|阅读量:

前言

Kotlin中的语法糖特别多,这也是它特别简洁的原因,本篇文章主要针对Kotlin中的语法糖来做介绍。

类型推导

这个特性众多语言都支持,简单来说就是: 让静态语言能够像动态语言那样声明变量。

如定义一个指定类型的变量:

var no: Int = 10
println(no is Int)// ture

可以简写为如下:

var no = 10
println(no is Int)// ture

函数简化

Unit

Unit是Kotlin中的一个类型，对应于Java中的void的类型,表示无返回值。

fun say(): Unit{

}

通常我们可以不用声明它:

fun say(){
  
}

以上写法结果完全一致。

单行表达式

常见的方法定义如下:

fun doRunning():String{
    //....
    //....
    //....
    return "Hello"
}

如果一个方法非常简单,只有一行表达式的时候,可以简化为如下这样:

fun doRunning():String = "Hello"

甚至可以借助类型推导进一步再次简化:

fun doRunning() = "Hello"

== & ===

Kotlin中的==和===是不同的,它们就和Js一般。

==用于比较两个变量是否值相等。

===用于比较两个变量是否引用相等(地址相等)。

如下代码:

var s1 = "123"
var s2 = "123"
println(s1 == s2) // true

==其实本质上就是调用了Java中的equals()方法。下面是编译后的代码:

String s1 = "123";
String s2 = "123";
boolean var2 = Intrinsics.areEqual(s1, s2);
System.out.println(var2);

// 具体的执行逻辑
public static boolean areEqual(Object first, Object second) {
  return first == null ? second == null : first.equals(second);
}

如果是使用===比较符呢?

var s1 = "123"
var s2 = "123"
println(s1 === s2) // true

编译后的代码如下:

String s1 = "123";
String s2 = "123";
boolean var2 = s1 == s2;
System.out.println(var2);

可以看到Kotlin中===最终编译后会使用Java中的==来比较两个变量的地址是否相等。

lambda

lambd表达式是Kotlin中非常常见的一种语法糖,在Java中也经常使用到,不过Kotlin作为一种函数式编程语言,对lambda的运用更为广泛。

简单来说,lambda其本质就是匿名函数。

无参无返回值

定义一个最简单无参数无返回值的lambda方法:

val doFunction:()-> Unit = {
  println("Hello lambda")
}
fun main() {
    // 使用lambda定义一个方法
    val doFunction = {
        println("Hello lambda")
    }
    // 执行方法
    doFunction() // Hello lambda
}

有返回值

定义一个有返回值的lambda方法。

// 使用lambda定义一个有返回值的方法
var getName: ()-> String = {
  "张三"
}
println(getName())// 张三
// 使用lambda定义一个有返回值的方法
var getName = {
  "张三"
}
println(getName())// 张三

有参数有返回值

定义一个有参数有返回值的lambda方法。

var sum: (Int,Int)-> Int = {a,b-> a + b}
println(sum(10,10)) // 20
var sum = {a: Int,b: Int-> a + b}
println(sum(10,10))// 20

闭包

Kotlin中的lambda方法,同样是支持闭包特性的,如下:

val name = "Hello"
val doName = {appendStr: String->
              name + appendStr
             }
println(doName(" word"))// Hello word

作为参数

一个lambda方法可以作为变量,当然可以作为一个方法的参数。

fun test(showName: (name: String)-> Unit,name: String){
    showName("我的名字叫: ${name}")
}

然后执行该方法:

fun main() {
    test({name-> println(name) },"张三") // 我的名字叫: 张三
}
fun main() {
    val showName = {name: String-> println(name) }
    test(showName,"李四") // 我的名字叫: 李四
}

作为返回值

可以作为参数,当然也可以作为返回值:

fun test2(): (String)->Unit{
    return {name-> println(name)}
}

然后执行该方法,获取到lambda方法,并且执行:

test2()("赵六") // 赵六
val showName:(String)-> Unit = test2()
showName("王五") //王五

关键字参数

it

当一个lambda方法只有一个参数的时候,我们在内部可以直接使用it作为参数名使用:

fun showName(func: (String)-> Unit){
  func("Hello")
}

fun main(){
  showName { println(it) } //Hello
}

_

当lambda中的参数用不到的时候,可以使用_忽略掉

fun showName(func: (String,String)-> Unit){
    func("李华","男")
}


fun main(){
      showName { name,_->
        println("我叫${name}") // 我叫李华
    }
}

简化Java接口语法

在Java中,如果想对一个接口使用lambda语法,那么这个接口必须为函数式接口,也就是必须实现了@FunctionalInterface注解的接口,这样接口中就只能存在一个抽象方法。

在Kotlin中不需要这样,只要该接口只有一个抽象方法,那么就可以直接使用lmabda来实现,如下:

定义一个Java接口:

public interface CallBack {
    void showName(String name);
}

在Kotlin中使用,作为一个方法的唯一参数:

fun call(callBack: CallBack){
    callBack.showName("李华")
}

然后执行该方法:

fun main(){
    // 直接简写为如下
    call{
        println("我的名字是: $it") //我的名字是: 李华
    }
}

::

Kotlin中的::引用表达式,和Java中的很类似,表示直接引用一个方法或者函数作为一个参数来使用。

如下:

fun consumer(action: (Int,Int) -> Int): Int{
    return action(10,10)
}

fun add(a:Int,b:Int):Int{
    return a + b
}

fun main() {
    println(consumer(::add))//20
}

高阶函数

let

let通常用于和?.来使用,表示如果一个可空变量,如果不是null的话,则会执行一系列的后续操作,如果为null则不会执行。

示例: 如果name是null则追加上123字符,并且返回name的长度:

fun main(){
    var name:String? = "Hello"
    var value = name?.let {
        // name != null则会执行这里的作用域
        name += "123"
        name.length
    }
    println(value) // 8
}

如果为null则?.let以后的什么都不会执行

fun main(){
    var name:String? = null
    var value = name?.let {
      //...这里不会执行任何东西
    }
    println(value) // null
}

also

also和let非常相似,唯一区别就是let作用域要求返回一个值,而also作用域只会返回it本身。通常它只用来消费:

fun main(){
    var name:String? = "Hello"
    name?.also {
        // name != null
        println(it.length)//5
        println(it) // Hello
    }
}

?:

also作用域是当变量不为null的时候才会执行,可以通过追加?:符号来指定变量为null的时候要执行的作用域:

fun main(){
    var name:String? = null
    name?.also {
        println("name not null")
    }?:{
        println("name is null")  // name is null
    }
}

apply

apply函数是一个标准函数，它允许您在对象上执行一个代码块并返回该对象本身。这个函数非常有用，可以简化代码并使其更易读。

fun main(){
   //使用apply来创建对象
    val student = Student(18).apply {
        name = "张三"
        address = "广州"
    }
    println(student) // Student(name='张三', age=18, address='广州')
}

class Student( age: Int) {
    lateinit var name: String
    var age: Int = age
    lateinit var address: String
	  //toString...
}

run

run函数可以接受一个lambda表达式作为参数,并且返回执行结果,在内部可以省略掉对象名,直接调取方法或者属性:

fun main(){
    val name: String? = "Hello"
    // name 不为null则执行 run{}
    name?.run {
        println(this) // Hello
        println(length) //5
    }
}

with

主要用来简化变量名,在with{}作用域内部,无需变量名,可以直接访问变量和属性,最终返回一个新的结果值。

fun main(){
    val name: String = "Hello"
    val newName = with(name){
        println(this) // Hello
        println(length) // 5
        "$this word"
    }
    println(newName)// Hello word
}
