前言

java.util.concurrent.atomic包是Java从JDK5开始提供的一个并发原子包,基本上包含了所有基本类型所对应的原子类型。这个包中的原子操作类提供了一种用法简单、性能高效、线程安全地更新一个变量的方式。在Atomic包里一共提供了13个类，属于4种类型的原子更新方式，分别是原子更新基本类型、原子更新数组、原子更新引用和原子更新属性（字段）。

利用atomicInteger简单实现个同步锁

/**
 * 利用AtomicInteger 实现一个CAS锁
 *
 * @author Zero.
 * @date 2022/4/11 8:17 下午
 */
public class CustomCasLock {
    //锁标识 0:无锁状态  1:锁已被占用
    private final AtomicInteger value = new AtomicInteger(0);
    //持有锁的线程
    private Thread lockedThread;
    /**
     * 加锁 直到抢到锁。
     */
    public void lock(){
        boolean lock;
        do {
            //尝试加锁,如果加锁失败,将一直循环
            lock = value.compareAndSet(0, 1);
        }while (!lock);
        //加锁成功,记录加锁线程
        lockedThread = Thread.currentThread();
    }
    /**
     * 释放锁
     */
    public void unlock(){
        //无锁状态,无需释放锁
        if (0 == value.get()){
            return;
        }
        if (lockedThread == Thread.currentThread()){
            //解锁
            value.compareAndSet(1,0);
        }
    }
}

5. AtomicStampedReference

CAS 引发的 ABA问题

ABA是什么这里就不解释了,那么如何解决呢？

Java提供了AtomicStampedReference来解决ABA问题, 其实就是借助一个版本号进行多条件控制。

public class AtomicStampedReference<V> {

    private static class Pair<T> {
        final T reference; //所包装的原子类
        final int stamp;	//版本号,修改的次数
        private Pair(T reference, int stamp) {
            this.reference = reference;
            this.stamp = stamp;
        }
        static <T> Pair<T> of(T reference, int stamp) {
            return new Pair<T>(reference, stamp);
        }
    }
}

从源码中我们可以看到,AtomicStampedReference原子类包含两个常量,一个就是包含的原子类,另一个则就是版本号。

示例:

public static void main(String[] args) {
  //创建 指定版本号起始值
  AtomicStampedReference<Simple> reference = new AtomicStampedReference<>(null,0);

  //第一次更改 预期值为null 预期版本为0
  Simple simple1 = new Simple("admin");
  boolean b = reference.compareAndSet(null, simple1, 0, 1);
  System.out.println("第一次更改后 版本为:" + reference.getStamp());

  //再次更改
  reference.compareAndSet(simple1,new Simple("root"),1,2);
  System.out.println("第二次更改后 版本为:" + reference.getStamp());
}

与其他Atomic类不同的是,执行compareAndSet方法时,需要多入两个参数,一个为预期版本值,一个为新版本值。

9. AtomicIntegerFieldUpdater

该原子类所存在的意义较为特殊,它能够保证一个类的某一int属性持有原子性。比如说现在有个User类,类中有个Age属性,现在我不想把User包装成原子类型,我只想让类中的Age属性保证原子性即可。

static class User{
  volatile int age;
}
public static void main(String[] args) {
  //创建原子类,因为需要原子性的属性是User的 指定泛型为User
  // 并且 在newUpdater()中 指定属性的字段名称
  AtomicIntegerFieldUpdater<User> updater = AtomicIntegerFieldUpdater.newUpdater(User.class,"age");

  //创建实例对象,设置 age = 0
  User user = new User();
  user.age = 0;

  //启用10个线程 进行100次 ++操作
  IntStream.rangeClosed(0,100).forEach((i)->{
    new Thread(()->{
      for (int i1 = 0; i1 < 100; i1++) {
        updater.incrementAndGet(user);
      }
    }).start();
  });

  System.out.println(user.age); //最终结果: 10000
}

基本上与AtomicIntegerFieldUpdater

注意: 有些情况下是无法支持该原子类的操作 如下

  1. 不能使private修饰的变量
  2. 必须用volatile所修饰

那么我们通常都会对属性进行私有封装,那该如何,接下来写一个生产业务中常用的案例。

需求: 定义一个User类,类中包含一个counter私有属性,现在我们要保证counter的原子性,代码如下:

public class User {
    //统计值
    private volatile int counter;
    //在本类中声明一个原子类 直接作为属性,这样就能访问本类的私有属性
    private AtomicIntegerFieldUpdater<User> updater =
            AtomicIntegerFieldUpdater.newUpdater(User.class,"counter");
    
    /**
     * 然后对外暴露一个 递增方法,在方法中使用 AtomicIntegerFieldUpdater进行原子增加
     * @param newValue 新值
     */
    public void add(int newValue){
        //this: 操作对象就是本类
        //counter: 当前预期值
        //newValue: 要更新的新值
        updater.compareAndSet(this,counter,newValue);
    }

    /**
     * 获取 Counter
     * @return
     */
    public int getCounter(){
        return counter;
    }
}

可以看到,上面这种方式的好处,通过CAS实现无锁机制的线程安全。
