Rust异步编程

发表于2023-04-13|更新于2023-04-17|Rust
|字数总计:2k|阅读时长:6分钟|阅读量:

何为异步编程?

异步编程,是一种被越来越多编程语言支持的并发编程模型,它能够在少量系统线程上运行一大量并发任务，同时还能通过 async/await 语法，保持原本同步编程的观感。

Rust中的异步编程允许您编写高效的、非阻塞的、基于事件驱动的代码。在异步编程中，您可以执行多个任务同时运行，而无需为每个任务分配一个独立的线程，这有助于减少内存开销和提高程序性能。

Rust的异步编程依赖于其社区中的异步运行时，称为Tokio。Tokio提供了一个基于事件循环的异步I/O运行时，它使Rust程序能够处理大量的I/O操作，而不会因为等待I/O操作而被阻塞。

Rust语言的异步编程模型和Python、NodeJs等语言相同,都是基于事件循环来实现。

并发模型

目前常见的实现异步编程的并发模型有如下几种:

  • OS 线程

    • 基于操作系统的线程,不需要改变编程模型,这使得并发很容易。
    • 性能、资源开销很大,虽然线程池可以节省一些开销,但是还是难以支持大量IO密集负载。
    • 大多与编程语言都提供了此模型。

  • Event Loop事件循环

    • 一种基于事件驱动的并发模型。该模型通常用于处理 I/O 密集型应用。
    • 事件循环是一个无限循环，它等待并监听外部事件的发生，比如 I/O 事件、计时器事件、信号事件等等。当一个事件发生时，事件循环会调用对应的回调函数来处理事件。回调函数通常是异步的，它们被放置在事件循环队列中，直到被调用执行。
    • “非线性”的控制流。数据流和错误传播通常就变得很难跟进了。
    • 如Python、Node都基于该模型实现异步编程。
  • Coroutine 协程
    • 一种轻量级线程,也被称为用户态线程。同样不需要改变编程模型。
    • 抽象了对于系统编程和自定义运行时实现非常重要的底层细节。
    • 高性能、轻量级、资源消耗少。
    • 典型的代表作: Go。
  • Actor
    • 把所有的并发计算分割成称为 actor 的单元，相互之间通过易错的消息传递进行沟通，非常类似于分布式系统。actor 模型能够很高效地实现。
    • 是它还很多没有解答的实践问题，例如流程控制和重入逻辑。
    • 如Erlang、Scala都基于该模型实现异步编程。

Rust异步

尽管很多语言都支持异步编程，但实现细节上有很多不一样。Rust 的异步实现和大部分语言的在以下方面有区别：

  • Future是惰性的

    在 Rust 中，Future 是惰性的，这意味着当您创建一个 Future 时，它并不会立即执行。相反，Future 只是一个描述了如何在某个未来的时间点计算出结果的计算过程的抽象。

    当您要求执行 Future 时，它会产生一些计算，然后可能会阻塞线程，直到结果可用。这种行为是与惰性不同的，惰性的 Future 只在需要时才进行计算，而非惰性的 Future 在创建时就开始计算。

  • Async零成本

    使用async可以无需堆内存分配,和动态调度,对性能来说是极好的。

  • 运行时并非内置

    • Rust 不提供内置运行时。相反，运行时由社区维护的库提供。也就是社区提供的Tokio库。

  • 单线程、多线程均支持

Rust异步VS线程

Rust 中异步的首选替代是使用 OS 线程，可以直接通过 std::thread 或者间接通过线程池来使用。从线程模型迁移到异步模型，或者反过来，通常需要一系列重构的工作，既包括内部实现也包括任何暴露的公开接口（如果你在构建一个库）。因此，尽早地选择适合你需要的模型能够节约大量的开发事件。

  • 线程:
    1. 适用于少量任务,有内存和CPU开销,并且线程的创建与上下文切换非常昂贵。
    2. 一个线程池库可以减轻这些开销，但并不显著。
    3. 允许重用同步代码,无需大面积更改,无需特定的编程模型。
    4. 有些系统支持修改线程优先级。
  • 异步:
    1. 显著降低内存和CPU开销。
    2. 同等条件下,支持比多线程模型多几个量级的并发任务(少量线程支持大量任务)。
    3. 可执行文件体积变大(需要生成状态机,每个可执行文件都会捆绑一个异步运行时)。

async/await

async/.await是Rust中的特殊语法,也是目前大多数编程语言都支持的语法,其原语为: 在发生IO阻塞后,让当前线程让出执行权,把执行资源让给其他需要的执行线程,从而使得并发执行效率最大化。

async关键字负责将一个作用域转化为实现了Future特征的状态机。

使用async有两种方式,一种是用来修饰函数,一种是修饰作用域代码块。

定义一个异步函数:

// 异步函数
async fn hello(){
    println!("Hello, async");
}

通过async fn定义出的函数就是异步函数,编译器会将该函数编译为Future状态机。

这个异步函数完全等同于下面这个函数:

返回一个异步作用域:

use futures::Future;

fn hello1() -> impl Future<Output=()>{
    async{
        println!("Hello, async");
    }
}

该函数虽然是一个普通函数,但是它是声明式返回一个Future状态机,并且是没有结果的Future。

async fn和async blocks都实现了 Future trait的值。

执行异步函数(1)

异步函数可以通过futures::executor::block_on()执行器来执行:

use futures::executor;

fn main() {
    executor::block_on(hello()); //Hello, async
    executor::block_on(hello1());//Hello, async
}

block_on()会阻塞式执行一个异步函数,直到函数执行完毕。

执行异步函数(2)

还有一种执行异步函数的方式,就是通过.await语法进行调用。它会阻塞式等待异步任务的结果。一般用于在一个异步函数内,有序执行其他异步任务。

use futures::executor;

async fn sing(){
    println!("唱")
}

async fn skip(){
    println!("跳")
}

async fn rap(){
    println!("rap")
}

// 顺序执行多个异步任务
async fn chicken_show_time(){
    // 按顺序执行 sing(), skip(), rap() 三个异步任务
    sing().await;
    skip().await;
    rap().await;
}

// 主程序入口, 用于启动异步任务
fn main() {
    executor::block_on(chicken_show_time())
}

执行结果:

唱
跳
rap

.await;表示这里会进入阻塞,然后等待Future返回Ready,从而获取到结果值。

并且.await只能在async区域才可使用。

执行异步函数(3)

如果想同时执行多个异步任务,并且等待所有任务执行结束,可以使用futures::join!()宏来执行:

async fn sing(){
    // 睡眠2秒
    sleep(Duration::from_secs(2));
    println!("唱")
}

async fn skip(){
    // 睡眠1秒
    sleep(Duration::from_secs(1));
    println!("跳")
}

async fn rap(){
    println!("rap")
}


// 顺序执行多个异步任务
async fn chicken_show_time(){
    // 同时执行 sing(), skip(), rap() 三个异步任务,并且等待所有任务执行完毕
    futures::join!(skip(),sing(), rap());
}

Future trait

Future trait是Rust异步编程的核心,它是一种异步任务所产生的结果对象,用来承载一个目前可能还不可用,但是未来可用的值。

Future通过调用poll函数来获取结果。

如果任务已经完成,则返回Ready(result_value)，其中result_value就是任务结果。

如果任务还未完成,则返回Rending,并且可以为它注册一个wake()回调函数,当任务执行完成后,回执行wake()回调函数。
