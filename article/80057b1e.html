Rust 模块管理

模块管理是每个编程语言必须支持的特点之一,因为随着项目的不断扩大项目会变得难以维护,更重要的是一个项目可能会有几十上百人共同协作开发,这更是难以维护。所以一个语言需要更好的维护和管理,模块系统是必不可少的。

Rust 管理结构

Rust 有许多功能可以让你管理代码的组织，包括哪些内容可以被公开，哪些内容作为私有部分，以及程序每个作用域中的名字。

这有时被称为 “模块系统（the module system）”，包括:

  • 包(Package): Cargo 的一个功能，它允许你构建、测试和分享 crate。
  • Crates: 一个模块的树形结构，它形成了库或二进制项目。
  • 模块(Modules): 和 use： 允许你控制作用域和路径的私有性。
  • 路径(Path): 一个命名例如结构体、函数或模块等项的方式

包-Package

一个包中至多 只能 包含一个库 crate（library crate）；包中可以包含任意多个二进制 crate（binary crate）；包中至少包含一个 crate，无论是库的还是二进制的。

下面通过Cargo来创建一个名为myPackage的包:

% cargo new myPackage
     Created binary (application) `myPackage` package

% cd myPackage
% tree .
.
├── Cargo.toml
└── src
    └── main.rs

当我们执行了cargo new xx命令后,Cargo 会创建一个项目,也称之为包(package),并且会为包创建一个Cargo.toml文件。并且还生成了一个src/main.rs文件。这个文件就是包的crate根文件,也就是Rust编译器打包时的起始源文件入口。

我们创建的每一个xxx.rsRust文件都是包的create。一般分为三类：

  1. src/main.rs

    root crate 包的根文件。每个包必须包含该文件。

  2. src/lib.rs

    该文件也是一种特殊的crate,被称为库crate。每个包只能包含一个该文件,也可以不包含。

  3. src/bin/xxx.rs

    自定义二进制crate。每个包可以包含无数个。

包的规则:

  • 一个Package中必须包含一个crate。
  • 一个包中可以不包含库crate(lib.rs)。如果包含仅只能包含一个lib.rs。
  • 一个包中可以包含无数自定义二进制crate。

模块-Modules

模块 让我们可以将一个 crate 中的代码进行分组，以提高可读性与重用性。模块还可以控制项的 私有性，即项是可以被外部代码使用的（public），还是作为一个内部实现的内容，不能被外部代码使用（private）。

下面来创建一个模块:

% cargo new --lib restaurant
     Created library `restaurant` package
     
% cd restaurant 
% tree .

├── Cargo.lock
├── Cargo.toml
├── src
│   └── lib.rs

模块的目录结构与包很相似,区别就是src的main.rs换成了lib.rs。

然后我们就可以在lib.rs文件中定义模块:

// 定义模块,pub表示允许其他模块访问
pub mod front_of_house {
    // 定义子模块
    pub(crate) mod hosting {
        // 定义函数
        pub(crate) fn add_to_waitlist() {}
    }
    pub mod serving {
        pub fn take_order() {}
        pub fn server_order() {}
        pub fn take_payment() {}
    }
}

每个模块使用mod定义,一个模块中又可以包含多个模块。

pub用于指定函数、结构体、枚举、模块或模块中的项的可见性。可见性指的是该项是否可以被其他模块或crate访问和使用,如果不知道为pub那么默认为私有的。

如果想要访问模块中的函数或者其他资源时,我们需要通过绝对路径或者相对路径来访问。

// 定义模块,pub表示允许其他模块访问
pub mod front_of_house {
    // 定义子模块
    pub(crate) mod hosting {
        // 定义函数
        pub(crate) fn add_to_waitlist() {}
    }
}

pub fn eat(){
    // 使用绝对路径访问
    crate::front_of_house::hosting::add_to_waitlist();

    //使用相对路径,因为本身就在同一个模块中，所以可以使用相对路径
    front_of_house::hosting::add_to_waitlist();
}

模块之间,可以通过super关键字来访问父模块的资源。

pub mod front_of_house {
    // 定义子模块
    pub(crate) mod hosting {
        // 定义函数
        pub(crate) fn add_to_waitlist() {}
    }
    pub mod serving {
        fn take_order() {
            // 通过super:访问父模块的函数
            super::hosting::add_to_waitlist();
        }
    }
}

模块中定义结构体

模块内可以定义函数,自然也是可以定义结构体的:

pub mod user_service {
    // 定义结构体
    pub struct User{
        pub name: String,
        pub age: i8,
        pub address: String,
    }
    // User关联函数
    impl User{
        pub fn summer(_name: &str) -> User{
            User{
                name: String::from(_name),
                age: 18,
                address: String::from("广州")
            }
        }
    }
}

// 构建user函数
pub fn get_user(){
    let mut u = user_service::User::summer("小张");
    u.name = "New_小张".to_string()
}

use关键的使用

通过use关键字,能够将指定的模块导入当前作用域,这样就可以不用使用绝对路径或者相对路径来访问

pub mod user_service{
   pub mod user_child_service{
      pub  mod fun_mod{
           pub fn test(str: String){

            }
        }
    }
}

// 导入user_service模块下的user_child_service模块下的fun_mod函数
use user_service::user_child_service::fun_mod::test;
fn call_test(){
    // 直接调用该函数
    test("Hello".to_string())
}

引入Rust原生的库也是通过该关键字

use std::fmt::Result;
use std::io::Result as IoResult;

如果导入不同的模块中有同名的冲突,可以使用as关键字设置一个别名。

使用第三方库

引入第三方库首先要在Cargo.toml文件中引入依赖.

[dependencies]
#引入rand库,版本为0.8.5
rand = "0.8.5"

然后就可以在自定的crate中使用该库中的模块和资源。

// 引入rand模块的Rug特征
use rand::Rng;

pub fn random_number(){
    // 随机生成1-100之间的随机数
    let number: i32 = rand::thread_rng().gen_range(1..101);
}

运行测试:

#[cfg(test)]
mod tests {
    use crate::random_number;
    #[test]
    fn it_works() {
        println!("{}",random_number()) //76
    }
}

use可以嵌套引入,一次性引入多个资源.

use rand::{CryptoRng,distributions::Standard,Rng};

或者引入一个模块中的所有资源.

use std::io::*;
