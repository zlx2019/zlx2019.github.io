Go【七】Channel

发表于2022-05-09|更新于2023-03-24|Go
|字数总计:2.1k|阅读时长:7分钟|阅读量:

前言

Channel是Go中的一个类型,一种引用类型,通过 Channel可以解决大量并发问题,如多协程通信。虽然可以使用共享内存进行数据交换,但是不同的 goroutine操作共享数据容易发生竞争,为了保证数据的安全及正确性,必须使用锁机制,这种做法势必造成性能降低。如果说goroutine是Go程序并发的执行体，channel就是它们之间的连接。channel是可以让一个goroutine发送特定值到另一个goroutine的通信机制。

Go语言的并发模型是CSP（Communicating Sequential Processes），提倡通过通信共享内存而不是通过共享内存而实现通信。

Go 语言中的通道（channel）是一种特殊的类型。通道像一个传送带或者队列，总是遵循先入先出（First In First Out）的规则，保证收发数据的顺序。每一个通道都是一个具体类型的导管，也就是声明channel的时候需要为其指定元素类型。

1. 创建channel

声明channel语法格式如下:

var 变量名 chan 元素类型
var channel chan string //声明一个string类型的 channel
fmt.Println(channel)// <nil>

通道是引用类型,零值为nil。声明的通道需要通过make()函数之后才能使用,如下:

make (chan 元素类型,[缓冲大小])

所以通常我们会直接这样创建通道:

var channel1 = make(chan string) //创建string类型的通道 并且初始化
var channel2 = make(chan string,10)//创建string类型的通道,缓冲为10

2. channel状态

channel存在3种状态:

  • nil : 未初始化状态, 已声明但未通过make()函数初始化。
  • active: 正常使用状态,通过make()函数初始化后。
  • closed: 已关闭状态,通过close()函数关闭后。

注意: 关闭后的通道无法再往里面发送值,但是可以获取值,但是获取到的值是通道元素类型的零值

关闭后的通道特点:

1.对一个关闭的通道再发送值就会导致panic。
2.对一个关闭的通道进行接收会一直获取值直到通道为空。
3.对一个关闭的并且没有值的通道执行接收操作会得到对应类型的零值。
4.关闭一个已经关闭的通道会导致panic。

3. channel操作

channel可进行3种操作:

  • 读/接收
  • 写/发送
  • 关闭
3.1 写操作

向通道内写入数据

var c = make(chan int, 10)
c <- 1001 //向通道中写入数据
3.2 读操作

从通道内取出数据

//获取通道内元素数量
fmt.Println(len(c)) //1

var num = <-c    //从通道中读取数据
fmt.Println(num) //1001

fmt.Println(len(c)) //0

<- c //从通道中取出数据 直接忽略该数据.
3.3 关闭

通过内置的close函数来关闭通道

close(c)

Channel不同的状态下进行不同的操作结果如下:

操作 nil active closed
读 阻塞 成功或阻塞 获取到元素零值
写 阻塞 成功或阻塞 panic
关闭 panic 成功 panic

4.无缓冲channel

无缓冲通道又称之为 阻塞通道,就是不给通道指定缓存值,如c := make(chan int),表示该通道没有任何容量,那么为什么会有这个东西呢,它的意义是什么？其实这个特性类似于Java中的SynchronousQueue它没有容量的概念,通常用来当同步信号使用。如下所示

func main() {
	ch := make(chan any) //创建一个无缓冲通道
	ch <- 1
}

这段代码在编译阶段是没有任何问题的,但是当我们执行时出现错误:

fatal error: all goroutines are asleep - deadlock!

goroutine 1 [chan send]:
main.main()
        /usr/local/myproject/go-project/Coder/channel/main.go:5 +0x4c

错误信息也很明显,出现了死锁。那为什么会出现死锁呢？这个就是 Channel有缓存与无缓冲的特性。

无缓冲通道只有在有接受信号时才能进行写入数据,同样的,只有在有写入信号时才能读取数据。

也就是说只有存在两个goroutine分别对一个 无缓冲通道进行读 和 写操作,才能进行通信。如下所示

func main() {
	ch := make(chan any) //创建一个无缓冲通道
	go func() {
		time.Sleep(10 * time.Second)
		ch <- 1001 //延迟10秒后 写入数据
	}()
	fmt.Println(<-ch) //阻塞等待获取通道内的数据
}

5. 只读只写channel

func main() {
	//创建只读channel
	onlyReadChan := make(<-chan any, 10)
	onlyReadChan <- 1 //编译错误,该通道只能读取
	a := <-onlyReadChan

	//创建只写channel
	onlyWriteChan := make(chan<- any, 10)
	b := <-onlyWriteChan //编译错误,该通道只能写入
	onlyWriteChan <- 1
}

6. select 多路复用

在某些场景下我们需要同时从多个通道接收数据。通道在接收数据时，如果没有数据可以接收将会发生阻塞。你也许会写出如下代码使用遍历的方式来实现：

for{
    // 尝试从ch1接收值
    data, ok := <-ch1
    // 尝试从ch2接收值
    data, ok := <-ch2
    …
}

这种方式虽然可以实现从多个通道接收值的需求，但是运行性能会差很多。为了应对这种场景，Go内置了select关键字，可以同时响应多个通道的操作。

select的使用类似于switch语句，它有一系列case分支和一个默认的分支。每个case会对应一个通道的通信（接收或发送）过程。select会一直等待，直到某个case的通信操作完成时，就会执行case分支对应的语句。

简单来说就是可以通过 select 来监听 Channel的读写事件。如下:

package main
import (
	"fmt"
	"time"
)
func main() {
	//创建两个通道
	c1 := make(chan any, 10)
	c2 := make(chan any, 10)
	//启用一个协程,每秒向一个通道内写入数据
	go func(v1, v2 chan any) {
		i := 0
		for {
			if i++; i%2 == 0 {
				v2 <- i
			} else {
				v1 <- i
			}
			time.Sleep(time.Second)
		}
	}(c1, c2)

	for {
		//监听c1和c2两个通道的写操作
		select {
		case val1 := <-c1:
			//如果监听到c1通道中有数据可获取,则触发该case
			fmt.Println("c1 监听到有数据插入:", val1)
		case val2 := <-c2:
			//如果监听到c2通道中有数据可获取,则触发该case
			fmt.Println("c2 监听到有数据插入:", val2)
		//如果c1和c2两个通道都有数据可获取,则随机触发两个case
		default:
			//如果c1和c2都没有数据可获取,则触发default
		}
	}
}

8. 循环读取channel

当需要不断从channel中获取数据时,可以通过for-range来读取,当channel为关闭状态时,for循环会自动退出

for x := range ch{
    fmt.Println(x)
}

9. 判断通道是否关闭

通过select case可以检测到 channel是否已关闭.

func main() {
	//创建两个通道
	c1 := make(chan any, 10)
	go func() {
		c1 <- 1
	}()
	select {
	case v, ok := <-c1:
		if ok {
      //true 表示成功读取数据
			fmt.Println("read val:", v)
		} else {
			//false 表示该c1通道 已经关闭
			fmt.Println("c1 chan is close.")
		}
	}
}

控制台输出:

read val: 1

关闭通道

func main() {
	//创建两个通道
	c1 := make(chan any, 10)
	close(c1) //直接关闭通道
	select {
	case v, ok := <-c1:
		if ok {
      //true 表示成功读取数据
			fmt.Println("read val:", v)
		} else {
			//false 表示该c1通道 已经关闭
			fmt.Println("c1 chan is close.")
		}
	}
}

10. 通道超时

使用select case + time.After可以做到模拟超时效果.

func main() {
	channel := make(chan any, 10)
	for {
		select {
		case <-channel:
		case <-time.After(time.Second * 10):
			//如果超过10秒 该select下的所有case都未触发,则会触发该case,模拟超时效果
			return
		}
	}
}

11. 信号通信

使用 Channel作为同步信号时,推荐使用&struct{}作为元素,性能更为高效。

func main() {
	//作为信号,控制main退出的退出
	flag := make(chan any)

	go func(f chan any) {
		//TODO 执行任务
		time.Sleep(time.Second * 3)
		//子任务执行完成,发送信号,让main函数退出
		f <- &struct{}{}
	}(flag)
	//等待协程的执行完成
	<-flag
	fmt.Println("main is down.")
}
