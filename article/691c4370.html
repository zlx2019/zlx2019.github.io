TypeScript学习之旅【一】基础类型

发表于2021-05-09|更新于2023-03-28|TypeScript
|字数总计:1.5k|阅读时长:5分钟|阅读量:

前言

TypeScript是JavaScript的超集,它可以编译成JavaScript

1. Hello Wrold

1.1 安装ts

npm install -g typescript

1.2 永远的Hello Word

创建Hello.ts文件

console.log("Hello word!");

1.3 编译成js

tsc Hello.ts

1.4 node 运行js

node Hello.js

2.简化执行ts的步骤

  • 安装ts-node包

    npm i -g ts-node

  • 使用方式

    ts-node xxx.ts

3.变量

  • 语法

    修饰符 变量名: 数据类型;

  • 声明字符串为例(变量可赋值可不赋值)

    //修饰符  变量名: 数据类型
let username: String = 'lxz';
or
let password: String;
password = 'admin';

  • 类型推导

    //boolean 类型
let isDone = false;
//string 类型
let str = 'hello';
//数值
let num = 0;

4. TypeScript的基本数据类型

4.1 布尔值

最基本的数据类型就是简单的true/false值，在JavaScript和TypeScript里叫做boolean（其它语言中也一样）。

let isDone: boolean = false;

4.2 数值

和JavaScript一样，TypeScript里的所有数字都是浮点数。 这些浮点数的类型是 number。 除了支持十进制和十六进制字面量，TypeScript还支持ECMAScript 2015中引入的二进制和八进制字面量。

let decLiteral: number = 6;
let hexLiteral: number = 0xf00d;
let binaryLiteral: number = 0b1010;
let octalLiteral: number = 0o744;

并且Nan和Infinity也是属于number类型下的子集:

declare var NaN: number;
declare var Infinity: number;

let notNumber = NaN
let infinitNumber = Infinity

4.3 字符串

JavaScript程序的另一项基本操作是处理网页或服务器端的文本数据。 像其它语言里一样，我们使用 string表示文本数据类型。 和JavaScript一样，可以使用双引号（ "）或单引号（'）表示字符串。

let name: string = "bob";
name = "smith";

你还可以使用模版字符串，它可以定义多行文本和内嵌表达式。 这种字符串是被反引号包围（ ``），并且以${ expr }`这种形式嵌入表达式

let names: string = 'lxz'
let age: number = 25
let content: string = `我叫${names},今年${age}岁!`
console.log(content)

4.4 数组

TypeScript像JavaScript一样可以操作数组元素。 有两种方式可以定义数组。 第一种，可以在元素类型后面接上 []，表示由此类型元素组成的一个数组：

//定义数组
let list: number[] = [1,2,3,4,5]
console.log(list)

//定义泛型数组
let numbers: Array<number> = [1,2,3]
console.log(numbers)

4.5 元组

元组类型允许表示一个已知元素数量和类型的数组，各元素的类型不必相同。 比如，你可以定义一对值分别为 string和number类型的元组。

//定义元组类似于map
let x: [string,number];
x = ['age',29]
console.log(x)

//定义集合元组
let xArr : Array<[string,number]> = []
xArr[0] = ['小张',18]
xArr[1] = ['小刘',15]
xArr[2] = ['小华',12]
console.log(xArr)

4.6 枚举

enum类型是对JavaScript标准数据类型的一个补充。 像C#等其它语言一样，使用枚举类型可以为一组数值赋予友好的名字。

//定义枚举，默认值从0开始,然后依次递增,加入第一个属性设置为1,就从1开始递增
enum Week{
    周一 = 1,
    周二,
    周三,
    周四,
    周五,
    周六,
    周日
}
console.log(Week.周一);

4.7 any

any表示任意类型,它属于typescript中的top type(顶级类型)。当一个变量类型为any时,表示可以赋予任何值。

let a: any = 1
let b: any = 'hello'
let c: any = true
let d: any = {}

4.7 unknown

unknown与any一样,也是top type顶级类型.它表示的是一个未知的值。它可以是任何类型。

unknown 类型与 any 类型的区别在于，any 类型可以赋值给任意类型的变量。

unknown 类型只能赋值给 unknown 和 any 类型的变量。

4.8 void

某种程度上来说，void类型像是与any类型相反，它表示没有任何类型。 当一个函数没有返回值时，你通常会见到其返回值类型是 void:

function warnUser(): void {
    console.log("This is my warning message");
}

声明一个void类型的变量没有什么大用，因为你只能为它赋予undefined和null,赋予其他任意类型都会报错

let a: void
a = undefined
a = null

4.9 Null 和 Undefined

Null和Undefined都是表示不存在或者缺失的值，但它们的含义略微有些不同。

  • undefined

    表示一个变量或者属性已经被声明了,但是并没有赋值,或者说一个方法没有任何返回值的情况下。如下:

    // 声明一个number变量,但不给它赋值
let foo: number
console.log(foo) // 则会输出为 `undefined`

// 方法没有任何返回值
function bar(){}
console.log(bar()) // 也会输出为 `undefined`

  • null

    表示一个变量或者属性被明确地赋值为了空值。例如：

    let v: number | null = null;
console.log(v); // null

默认情况下null和undefined是所有类型的子类型。 就是说你可以把 null和undefined赋值给任意类型的变量。

并且这两种类型可以交叉赋值,可以兼容对方的类型:

let a = null
let b = undefined
// ---
let c: null = b
let d: undefined = a

4.10 Never

never类型表示的是那些永不存在的值的类型。 例如， never类型是那些总是会抛出异常或根本就不会有返回值的函数表达式或箭头函数表达式的返回值类型； 变量也可能是 never类型，当它们被永不为真的类型保护所约束时。

never类型是任何类型的子类型，也可以赋值给任何类型；然而，没有类型是never的子类型或可以赋值给never类型（除了never本身之外）。 即使 any也不可以赋值给never。

下面是一些返回never类型的函数：

// 返回never的函数必须存在无法达到的终点
function error(message: string): never {
    throw new Error(message);
}

// 推断的返回值类型为never
function fail() {
    return error("Something failed");
}

// 返回never的函数必须存在无法达到的终点
function infiniteLoop(): never {
    while (true) {
    }
}

4.11 Object

object表示非原始类型，也就是除number，string，boolean，symbol，null或undefined之外的类型。

使用object类型，就可以更好的表示像Object.create这样的API。例如：

declare function create(o: object | null): void;

create({ prop: 0 }); // OK
create(null); // OK

create(42); // Error
create("string"); // Error
create(false); // Error
create(undefined); // Error
