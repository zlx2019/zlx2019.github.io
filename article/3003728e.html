Java【9-18新特性】

发表于2022-07-10|更新于2023-03-22|Java
|字数总计:3.2k|阅读时长:12分钟|阅读量:

前言

从2017年开始,Java从原来的两年更新一版本策略更改为每半年更新一版,从而快速推动Java的发展。目前国内使用最多的还是Java8,但已经有近一半的开发人员投入了Java11的怀抱。这个趋势也在不断上升,而且目前一些主流框架如SpringBoot3强制要求使用Java17版本。本章重点讲述Java11的一些特性,因为直接从Java8跳到18还是不太现实,一步步来。

Java9 新特性

1.Optional优化

对象的非空校验通常是开发中最常见最频繁的操作,因为只有对所有的对象校验后才能保证不会引发空指针异常,但这样通常会导致代码非常丑陋,Optional就是为了简化该过程而出现。

Optional是Java8提供的类,但是在Java9中增加了大量API;

如下所示:

这是我们传统的非空校验方式.

public static void hello(String text){
  if (Objects.nonNull(text)){
    System.out.println(text.toLowerCase());
  }
}

这是使用Optional改造后的方式,视觉上来说一气呵成,简洁了不少。

public static void hello(String text){
  Optional.ofNullable(text).ifPresent(System.out::println);
}

下面针对API进行逐一介绍

  • of

    构建一个Optional类,指定元素不能为Null,如果值为Null则直接抛出NullPointerException异常。

    Optional.of(null);// run: java.lang.NullPointerException

  • ofNullable

    构建一个元素值允许为Null的Optional类。

    Optional.ofNullable(null);
Optional.ofNullable(123);
Optional.ofNullable("123");

  • empty

    构建一个空的Optional类。

    Optional.empty();

    尽管Optional提供了该方法,但不推荐通过该方式来判断一个Optional是否为空,如下

    Optional<String> empty = Optional.of("Hello");
if (empty.equals(Optional.empty())){
  //错误用法
}

    应当使用isPresent或isEmpty()来判断。

  • isPresent

    如果Optional有值就返回true,反之为false。

    boolean present = Optional.ofNullable("hello").isPresent();// true
boolean present = Optional.empty().isPresent();// false

  • isEmpty

    如果Optional没有值则返回true,反之为false。与isPresent方法截然相反。出自Java11

    Optional.ofNullable("hello").isEmpty();// false
Optional.empty().isEmpty(); //true

  • get

    获取Optional的元素值。

    Optional<String> s = Optional.ofNullable("hello");
String value = s.get();

  • orElse

    获取Optional元素值,如果Optional为空时,则获取一个指定的默认值。

    Integer value = Optional.ofNullable(100).orElse(200);// 100
Object otherValue = Optional.empty().orElse(200); // 200

  • orElstGet

    获取Optional元素值,如果不为空,则直接返回元素值。

    如果为空,则会执行指定的Supplier函数式接口,该接口返回数据类型对应该Optional元素类型。

    Optional.ofNullable("zhangsan").orElseGet(()-> {
  String otherName = "lisi".toUpperCase();
  return  otherName;
}); //zhangsan
    Optional.empty().orElseGet(() -> {
  String otherName = "lisi".toUpperCase();
  return otherName;
});//LISI

  • orElseThrow

    获取Optional元素值,如果为空直接抛出NoSuchElementException("No value present")异常。出自Java10,有些鸡肋

    Optional<Integer> optional = Optional.empty();
optional.orElseThrow(); //NoSuchElementException: No value present

  • orElseThrow(Throwable e)

    获取Optional元素值,如果为空直接抛出指定的异常。

    Optional<Integer> optional = Optional.empty();
optional.orElseThrow(()-> new RuntimeException("No value present"));

  • or

    如果当前Optional不为空,则直接返回该Optional,注意:不是元素值哦,是Optional。反之如果为空,则获取一个新的Optional。出自:Java9

    Optional<Integer> optional = Optional.of(1001);
Optional<Integer> newOptional = optional.or(() -> Optional.ofNullable(5005));
System.out.println(newOptional.get());//1001
    Optional<Integer> optional = Optional.empty();
Optional<Integer> newOptional = optional.or(() -> Optional.ofNullable(5001));
System.out.println(newOptional.get());//5001

  • IfPresent(Consumer)

    该方法需要传入一个Consumer函数式接口,显然它是一个消费方法,并且是无返回值的。消费数据自然就是Optional中的元素值了,只有元素值不为空的情况下才会进行消费~

    Optional<Integer> optional = Optional.empty();
optional.ifPresent(value-> {
  // 不会执行
  System.out.println(value);
});
    Optional<String> optional = Optional.of("hello");
optional.ifPresent(value-> {
  System.out.println(value);//hello
});

  • ifPresentOrElse(Consumer,Runnable)

    该方法类似于IfPresent(Consumer),是一个消费方法,不同的是它需要多入参一个Runnable接口,也就是说如果Optional为空的情况下,会执行Runnable接口方法。

    Optional<String> optional = Optional.of("hello");
optional.ifPresentOrElse(value->{
  System.out.println("消费值:" + value);
}, ()->{
  System.out.println("该Optional没有值,无法消费");
});
    消费值:hello
    Optional<String> optional = Optional.empty();
optional.ifPresentOrElse(value->{
  System.out.println("消费值:" + value);
}, ()->{
  System.out.println("该Optional没有值,无法消费");
});
    该Optional没有值,无法消费

4.接口私有方法

在Java8中,接口中的方法是可以通过default关键字实现默认方法体。

default void loadUser(){
  System.out.println("Default loadUser method!");
}

如果某个实现类没有重写该抽象方法,则会执行接口中的默认方法体。

public class UserImpl implements UserService{

}
public static void main(String[] args) {
  UserService userService = new UserImpl();
  userService.loadUser();
}
Default loadUser method!

以上是Java8提供的特性,那么Java9为了方便接口中的默认方法封装化,在接口中提供了私有方法。

接口中的方法访问修饰符默认都是public,如果定义成private,表示为私有方法,只有该接口中才能使用。

public interface UserService {
    private void getName(){
      
    }
}
  1. 接口中的私有方法必须有实现体。
  2. 通常只有接口中的静态方法或默认方法才能调用本接口的私有方法。
public interface UserService {
    private static String getName(){
        return "name";
    }
    default void loadUser(){
        String name = getName();
    }

    static void formDate(){
        getName();
    }
}

Java10 新特性

1.局部变量推导

大多数强类型语言其实都有类型推断特性,但是Java在10才提供。

public static void main(String[] args) {
  var name = "张三";
  var names = getName();
}

public static String getName(){
  return "zhangsan";
}

这种推导只能在局部区域才能使用 如方法内部。无法使用在类变量中。

Java11 新特性

1.Lambda语法类型推导

Consumer<String> consumer = (var s)->{
  System.out.println(s);
};

Function<Integer, String> func = (var number)->{
  return number.toString();
};

这种语法在Java11之前是无法编译通过~

2.String新增API

  • isBlank

    字符串是否为Null或者空串。

    String str = " ";
boolean blank = str.isBlank(); //true
boolean empty = str.isEmpty(); //false

    isEmpty字符串为Null或者""时返回true。但是" "空格也属于字符的一部分,所以无法校验出。

    isBlank字符串为Null或仅包含空白代码点，则返回true ，否则返回false 。

  • repeat

    将一个字符串值复制N个,然后拼接成一个字符串。

    String str = "Hello";
System.out.println(str.repeat(3));//HelloHelloHello

  • lines

    将字符串根据换行符 \r、\n 切割成多个字符串并且使用Stream流包装。

    String str = "Hello\n今天天气真不错\r对吧?";
Stream<String> lines = str.lines();
lines.forEach(System.out::println);
    Hello
今天天气真不错
对吧?

  • strip

    去除字符串前端与末端的空格。

  • stripLeading

    去除字符串前端的空格。

  • stripTrailing

    去除字符串末端的空格。

3.HttpClient

全新的Http客户端工具。替代老旧的HttpURLConnection类。

构建客户端

HttpClient.Builder builder = HttpClient.newBuilder();
// 构建客户端参数
builder.version(HttpClient.Version.HTTP_2);// 指定Http协议版本
builder.connectTimeout(Duration.ofSeconds(10));//设置此客户端的连接超时持续时间。
builder.priority(1);//设置请求优先级
builder.followRedirects(HttpClient.Redirect.NEVER);//指定请求是否会自动跟随服务器发出的重定向
// 构建客户端
HttpClient client = builder.build();

发送同步Get请求

// 构建Get请求体
HttpRequest httpRequest = HttpRequest.newBuilder()
  .GET()
  .uri(new URI("http://127.0.0.1:8080/hello"))
  .timeout(Duration.ofSeconds(20))
  .build();
// 发起请求 将响应结果解析为String
HttpResponse<String> response = client.send(httpRequest, HttpResponse.BodyHandlers.ofString());
// http响应码
System.out.println(response.statusCode());
// 响应内容
System.out.println(response.body());
200
Hello World

发送异步Get请求

// 构建Get请求体
HttpRequest httpRequest = HttpRequest.newBuilder()
  .GET()
  .uri(new URI("http://127.0.0.1:8080/hello"))
  .timeout(Duration.ofSeconds(20))
  .build();
// 发送异步请求
client.sendAsync(httpRequest, HttpResponse.BodyHandlers.ofString())
  .thenApply(response-> response.body())
  .thenAccept(System.out::println)
  .join();
Hello World

发送同步Post请求

// 构建post body
ObjectMapper mapper = new ObjectMapper();
Map<String, Object> param = new HashMap<>();
param.put("id","1001");
param.put("username","root");
// map转json
String body = mapper.writerWithDefaultPrettyPrinter().writeValueAsString(param);
// 构建Post请求体
HttpRequest httpRequest = HttpRequest.newBuilder(new URI("http://127.0.0.1:8080/example"))
  .header("Content-Type","application/json") //设置为json格式
  .POST(HttpRequest.BodyPublishers.ofString(body))    //设置Body
  .timeout(Duration.ofSeconds(10))
  .build();
// 发送同步post请求,
HttpResponse<String> response = client.send(httpRequest, HttpResponse.BodyHandlers.ofString());
System.out.println(response.body());
{"address":"上海","name":"张三s","age":18}

发送异步Post请求

// 发送异步post请求
client.sendAsync(httpRequest,HttpResponse.BodyHandlers.ofString())
  .thenApply(HttpResponse::body)
  .thenAccept(System.out::println).join();
{"address":"上海","name":"张三s","age":18}

Put、Delete等请求和Post一致~

Java12-16新特性

1.Switch表达式优化-Java14

public static void main(String[] args) {
  int weekDay = 1;
  var desc = switch (weekDay){
    case 1-> "周一";
    case 2-> "周二";
    case 3-> "周三";
    case 4-> "周四";
    case 5-> "周五";
    case 6,7-> "周末";
    default -> {
      System.out.println("没有找到该日期~");
      yield  "未知";
    }
  };
}

如上所示:

  1. Java14优化后的Switch语法,是可以有返回值的,可以作为一个函数体来使用.
  2. 并且case可以指定多个匹配值.
  3. 每一个case都可以作为一个执行体,如果有返回值则需要使用yield关键字返回。

2.instanceof优化-Java14

在以往,我们强转一个类之前为了使代码的健壮性,通常会进行类型的断言,如下:

public class Student{
  void info(){}
}

public static void main(String[] args) {
  Object obj = "";
  if (obj instanceof Student){
    Student stu = (Student)obj;
    stu.info();
  }
}

Java14优化后只需要如下即可:

public static void main(String[] args) {
  Object obj = "";
  if (obj instanceof Student student){
    student.info();
  }
}

3.空指针异常优化-Java14

空指针异常基本上是Java中最频繁出现的异常,该异常栈信息通常会指定在哪个类文件拿一行代码出现该异常。

String str = null;
System.out.println(str.length());
Exception in thread "main" java.lang.NullPointerException
	at com.zero.str.FinalString.main(FinalString.java:10)

该异常信息指出在FinalString.java的行号10出现空指针异常。

在Java14后得到进一步优化,异常信息更为完善~

如下:

String str = null;
str.length();
Exception in thread "main" java.lang.NullPointerException: Cannot invoke "String.length()" because "str" is null
	at com.zero.Main.main(Main.java:11)

Cannot invoke "String.length()" because "str" is null显然,异常信息已经将出现该异常的变量指出。

4.record-Java14

record类是Java14提供的非常重要Java基础文件.就如class、enum、interface一样。它主要用来作为纯数据载体。

record特点:

  1. 在编译阶段自动生成setter、getter、toString、hashcode等基础方法。

  2. 它是不可变的,一旦通过默认构造函数构建后就无法修改任何属性,所以它是安全的。

  3. 默认为final修饰,无法被继承。
  4. 通常定义一些pojo时非常适合使用。

定义Record

语法如下:

public record 记录名(属性列表) {
  //代码体只能声明静态属性
}

可以作为顶级类

public record Student(String name,Integer age) {
    
}

也可以作为类的成员属性

public class User {
    private record Studnet(String name,Integer age){}
}

可以作为一个匿名类

record Mail(Long id,String name){}

实例record

和类一样,通过new关键字声明。

Mail mail = new Mail(1001L,"root");

它可以像类一样定义一些静态属性和静态方法作为一个工具类。

public record DateUtil() {
    static String FORMAT = "yyyy-MM-dd HH:mm:ss";
    public static String convert(LocalDateTime time){
        return DateTimeFormatter.ofPattern(FORMAT).format(time);
    }
}

Java17新特性

1. 密封类

以往我们希望一个类不允许被继承时,只需要通过final关键词去修饰类即可,但这种方式有一定的缺陷就是太极端,直接不允许所有类去继承它.

而Java17提供了一个新的特性,约束一个类可以被某些特定的类所继承。

如下：创建一个User类,并且只允许Teacher和Student所继承。

public sealed class User permits Teacher,Student{

}
public final class Teacher extends User{

}
public final class Student extends User {

}

通过sealed关键字修饰,表示该类是密封类，密封类必须通过permits关键字声明自己可以被哪些子类所继承,并且子类必须显示继承该密封类,否则编译无法通过。

注意: 密封类的子类只能是密封类或不可继承类。其实这也是合理的,不然我们通过继承一个可继承的子类直接跳过了密封类的话显然就违背了密封类的意义。如下所示: 创建一个Member类继承User,再创建一个LevelMember1类继承Member。

public sealed class Member extends User permits LevelMember1{

}
public final class LevelMember1 extends Member {

}
public sealed class User permits Teacher,Student,Member{

}

可以看到,通过密封类我们可以手动建立起一条安全的继承链,从而保证这些类不会被其他类所侵入,当然,这也意味着会丧失所有的可扩展性。

当然也有更灵活的用法: 通过non-sealed关键字,表示自己继承自密封类同时自身又可以被任何子类所继承。

public non-sealed class Member extends User {

}

public  class LevelMember1 extends Member {

}

当然,这种方式同样可以用在接口与实现类中:

public sealed interface T permits B{}

public final class B implements T{}
