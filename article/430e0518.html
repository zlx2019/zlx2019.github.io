Rust 结构体

Rust中的结构体是一种自定义数据类型,就如同其他面向对象编程语言中的Class一般。用于将不同类型的属性组合在一起,形成一个新的数据结构。它们是一种类似于C语言中的结构体的概念，但具有更强大的功能和更严格的类型检查。

结构体定义

Rust使用struct关键字来定义结构体。结构体的名字需要描述它所组合的数据的意义。接着，在大括号中，定义每一部分数据的名字和类型，我们称为 字段（field）。

// 定义结构体
#[derive(Debug)]
struct User{
    id: i64,
    username: String,
    password: String,
    email: String,
    age: i8,
    active: bool
}

#[derive(Debug)]是为了让该结构体支持println!()输出宏能格式化输出出来。

有了结构体后,就可以实例化一个实例:

fn main() {
    let mut user: User = User {
        id:10001,
        username: String::from("root"),
        password: String::from("root"),
        email: String::from("root@163.com"),
        age: 18,
        active: true,
    };
    println!("user info: {:#?}",user)
}

输出结果:

user info: User {
    id: 10001,
    username: "root",
    password: "root",
    email: "root@163.com",
    age: 18,
    active: true,
}

操作结构体实例:

// 获取属性
let name: String = user.username;
println!("{}",name);// root

// 修改属性
user.username = "New Root".to_string();
println!("{}",user.username) //New Root

构造函数

我们可以通过函数的方式来简化结构体的初始化

// User结构体 构造函数,传入id、username,其他字段使用默认值
fn build_user(id: i64,username: String)-> User{
    User{
        id,
        username,
        password: String::from(""),
        email: String::from(""),
        age: 0,
        active: false
    }
}

有了这个函数,我们就可以调用函数来获取结构体:

let mut user = build_user(10002,String::from("张三"));

还可以通过Default来定义结构体的默认获取方法:

所有的字段都可以是零值。

// 定义User结构体的默认实例获取方法
impl Default for User {
    fn default() -> Self {
        Self{
            id: 0,
            username: String::from(""),
            password: String::from(""),
            email: String::from(""),
            age: 0,
            active: false,
        }
    }
}

然后这样获取实例:

let mut user2 = User::default();

字段拷贝

如果我们想根据一个现有的结构体对象构建一个新的对象,可以通过字段的拷贝。如下:

fn main() {
    let mut user1 = User {
        id: 10001,
        username: String::from("admin"),
        password: String::from("root"),
        email: String::from("root163.com"),
        age: 22,
        active: true,
    };
    println!("user1 {:?}",user1);
    // 构建一个新的User结构体,拷贝user1的字段值
    let mut user2 =  User{
      ..user1
    };
    println!("user2 {:?}",user2);
    // 在构建一个新的User结构体,指定id字段,其他字段拷贝user2的字段值
    let mut user3 = User{
        id: 100003,
        ..user2
    };
    println!("user3 {:?}",user3);
}
user1 User { id: 10001, username: "admin", password: "root", email: "root163.com", age: 22, active: true }
user2 User { id: 10001, username: "admin", password: "root", email: "root163.com", age: 22, active: true }
user3 User { id: 100003, username: "admin", password: "root", email: "root163.com", age: 22, active: true }

通过..user,就可以将当前结构体还未填充的字段值,全部填充上。

结构体方法

方法其实也是一种特殊的函数,两者的区别就在于作用域不同,方法是归属于结构体的。

为结构体定义方法,需要引入impl块,如下:

impl User{
    fn user_info(&self){
        println!("i name is {}",&self.username)
    }
}

impl块中的方法,每个方法的第一个参数都是&self,该参数其实就是当前实例的引用。

调用方法:

user3.user_info(); //i name is admin

方法中可以有三种self,下面一一详述:

指针 特性
&self 不可变的引用
&mut self 可变的引用
self 当前实例。会移动实例的所有权，并且不会归还。

  • &self是一个引用,我们之前不止一次提过,默认的引用是不可变的,所以如果使用&self指针的话,一般是指该方法只会做一些读取操作。

    如上面定义的user_info()方法。

  • &mut self 是一个可变引用,如果该方法需要对当前实例进行修改操作,那么肯定需要使用该指针。

    如下:

    定义一个username字段的setter方法

    impl User{
    // 修改username的方法
    fn set_username(&mut self,username : String){
        self.username = username
    }
}

fn main(){
    let mut u = User::default();
  	// 修改username
    u.set_username("满城雪".to_string());
    println!("{}",u.username); //满城雪
}

    注意: 只有被mut修饰的可变实例对象,才能调用&mut self的方法。

  • self 是一个实例,我们之前就说过,如果一个引用变量传入函数或者方法后,会将当前所有权转移出去,所以如果调用了self指针的方法后,当前实例就会失效。我们可以通过该指针实现一个结构体实例主动丢弃。

    如下:

    定义User结构体的主动销毁方法

    impl User{
    // User销毁方法
    // 调用该方法后,会将实例的所有权,转移给self指针.从而导致实例失效
    fn drop(self){
        println!("User Destroyed...")
    }
}

    演示如下:

    let mut  u1 = User::default();
u1.set_username("海问香".to_string());
println!("{}",u1.username);//海问香

// 执行销毁方法。将当前所有权转移给self指针
u1.drop();  //User Destroyed...

// 已销毁 无法再使用
println!("{}",u1.username) //borrow of moved value: `u1`

关联函数

Rust中的关联函数类似于Java中的静态方法,它与结构体的实例没有关联关系,它是和结构体本身所关联的。通过结构体可以直接访问的函数。

之前我们重写的default()函数其实就是关联函数.

// 定义User结构体的默认实例 方法
impl Default for User {
    fn default() -> Self {
        Self {
            id: 0,
            username: String::from(""),
            password: String::from(""),
            email: String::from(""),
            age: 0,
            active: false,
        }
    }
}

可以再写一个属于我们自己的关联函数。哦对了,一个结构体可以有多个impl块。

impl User{
    fn build()-> User{
        User{
            id: 0,
            username: "".to_string(),
            password: "".to_string(),
            email: "".to_string(),
            age: 0,
            active: false,
        }
    }
}

注意: 关联函数是没有self引用的,因为它和实例不关联。调用关联函数通过::域操作符来调用

let uu = User::build();
