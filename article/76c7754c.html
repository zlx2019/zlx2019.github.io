Flutter【基础组件】

前言

1. 默认基础布局

Flutter默认提供了一些基础框架Widget,这些Widget组件是整个APP的基石,一般情况下我们都是使用这些Widget作为基础页面,然后再在该基础上进行开发。

1. MaterialApp

基础页面Widget,通常一个页面级别的widget都由它包裹,如程序入口的runApp()函数所渲染的第一层Widget就是MaterialApp。

void main() {
  runApp(const MyApp());
}
class MyApp extends StatelessWidget {
  const MyApp({Key? key}) : super(key: key);
  @override
  Widget build(BuildContext context) {
    //返回一个页面 Widget
    return MaterialApp(
      //null
    )
  }
}

此时可见页面为一片黑色,因为该Widget还未设置任何内容。

MaterialApp构造函数内有title、home等可选变量可设置。

title顾名思义是指标题的意思 需要一个String类型的标题即可。而home则是一个Widget属性,很明显它的内容也是由Widget组成。

2. Scaffold

这是一个页面脚手架Widget,一般用于与作为于MaterialApp级别的Widget的子部件。可以构建出一个完整的页面。

@override
Widget build(BuildContext context) {
  return MaterialApp(
    title: "App Title",
    home: Scaffold(
    ),
  );
}

由MaterialApp与Scaffold部件组成的页面就产生了。

只不过目前是空白一片,因为这两个都是基础框架组件,内容需要我们一点一点填充。

3. AppBar

Scaffold部件,是一个页面部件,有两个可选属性appBar与body。两个属性都需要两个Widget部件来填充页面的页面头部与中间内容部分。

首先来看一下头部部分appBar。该属性可以使用AppBar部件,也是原生提供的部件.

提供了title属性,可以直接显示该页面的标题

@override
Widget build(BuildContext context) {
  return MaterialApp(
    title: "App Title",
    home: Scaffold(
      appBar: AppBar(
        title: Text("Title"),
      ),
    ),
  );
}

看一下页面效果

2. 创建有状态Widget

自定义一个Widget 作为页面的Body

///定义一个带有状态的Widget 作为页面的Home
class ZeroHome extends StatefulWidget {
  ZeroHome({Key? key}): super(key: key);

  ///重写 Widget的State方法
  @override
  State<StatefulWidget> createState() => _ZeroHomeState();
}

///定义 ZeroHome 部件的 State
class _ZeroHomeState extends State<ZeroHome> {
  @override
  Widget build(BuildContext context) {
    return Container();
  }
}

接下来的一些基础Widget都放在这个Container部件内讲解,关于这个Container我们后续再讲。

3. Text

Text部件是Fultter中最基础也是使用率最频繁的一个部件,就是用来渲染文本的,我们先来看一下它的构造函数。

const Text(
  String this.data, //要渲染的文本内容,必填项
  {
  Key? key,
  this.style,				//文本样式: TextStyle如文本大小、颜色、字体
  this.strutStyle,	//文本每行最小行高
  this.textAlign,		//文本水平对齐方式: 左对齐、水平居中、右对齐
  this.textDirection,//文本方向
  this.locale,			//
  this.softWrap,
  this.maxLines, 		//显示最大行数,多余文本直接截取
  this.overflow,		//文本截取后显示方式如显示... 或者直接截取
  this.textScaleFactor,
  this.semanticsLabel,
  this.textWidthBasis,
  this.textHeightBehavior,
})

简单渲染一个文本,然后看下效果

class _ZeroHomeState extends State<ZeroHome> {
  @override
  Widget build(BuildContext context) {
    return Container(
      width: 100,
      height: 100,
      color: Colors.blue,
      child: const Text(
        '今天是个好天气哟 嗨嗨嗨',
        //设置文本样式: 颜色、大小
        style: TextStyle(color: Colors.white,fontSize: 12),
        //设置文本在容器内 水平居中对齐
        textAlign: TextAlign.center,
        //最大显示一行文本
        maxLines: 1,
        //对于的文本 ...显示
        overflow: TextOverflow.ellipsis,
      ),
    );
  }
}

6. Container

Container类似于Html中的div 的盒子模型。通常用Container包装Widget。

接下来将一个Text包装到Container中,来看一下能够如何布局。

@override
Widget build(BuildContext context) {
  return Container(
    // color: Colors.blue,//设置Container背景颜色
    width: double.infinity, //设置宽度铺满
    height: 400,
    alignment: Alignment.center, //将容器内的Widget水平垂直都居中
    padding: EdgeInsets.all(10),//设置容器的内边距为 10
    margin: EdgeInsets.all(10),//设置容器的外边距为 10
    //为容器添加一个四边框,颜色为红色,边框宽度为2
    //注意: 边框颜色和容器背景颜色,只能设置一个
    decoration: BoxDecoration(border: Border.all(color: Colors.red,width: 2)),
    child: const Text("我是一个文本内容")
  );
}

Container的decoration与color属性存在冲突关系,但是我们又想拥有边框又想容器有背景颜色该如何处理呢?

那我们可以在decoration中设置背景颜色即可.

decoration: BoxDecoration(
  border: Border.all(color: Colors.red,width: 2),
  color: Colors.blue
),

Container的内部布局可选方案:

alignment: Alignment.center, //将容器内的Widget水平垂直都居中
  /// 左上角居中
  static const Alignment topLeft = Alignment(-1.0, -1.0);
  /// 顶部水平居中
  static const Alignment topCenter = Alignment(0.0, -1.0);
  /// 右上角居中
  static const Alignment topRight = Alignment(1.0, -1.0);
  /// 中部偏左居中
  static const Alignment centerLeft = Alignment(-1.0, 0.0);
  /// 水平垂直都居中
  static const Alignment center = Alignment(0.0, 0.0);
  /// 中部偏右居中
  static const Alignment centerRight = Alignment(1.0, 0.0);
  /// 左下角居中
  static const Alignment bottomLeft = Alignment(-1.0, 1.0);
  /// 底部水平居中
  static const Alignment bottomCenter = Alignment(0.0, 1.0);
  /// 右下角居中
  static const Alignment bottomRight = Alignment(1.0, 1.0);

Container如果不设置宽高,默认会撑满父部件宽高。

7. Center

Center部件是用来布局的widget。它能够让其子部件居中。如下所示

@override
Widget build(BuildContext context) {
  return MaterialApp(
    title: "App Title",
    home: Scaffold(
      appBar: AppBar(
        title: Text("首页"),
      ),
      body: TextButton(
          onPressed: ()=>{},
          child: const Text("按钮")
      )
    ),
  );
}

该页面body区域内容只有一个TextButton按钮，默认布局是最顶层最左侧依次排布。

但是,我们现在想要该按钮在父容器中水平且垂直居中排布,我们就可以使用Center来达到。

body: Center(
  child: TextButton(
    onPressed: ()=>{},
    child: const Text("按钮")
  ),
)

Center默认是居中,但是我们可以自己选择在容器内的排布位置.Center继承自于Align,所以我们可以直接使用Align,它的功能更强大。

@override
Widget build(BuildContext context) {
  return MaterialApp(
    title: "App Title",
    home: Scaffold(
      appBar: AppBar(
        title: Text("首页"),
      ),
      body: Align(
        alignment: Alignment.topCenter, //水平居中
        child: TextButton(
            onPressed: ()=>{},
            child: const Text("按钮")
        ),
      )
    ),
  );
}

通过Align的alignment属性来指定排布的位置。我们这里是使用的Scaffold容器来演示,起始无论父容器是什么容器,都是如此。
