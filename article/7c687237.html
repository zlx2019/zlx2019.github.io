Java【泛型】

发表于2022-03-25|更新于2023-03-22|Java
|字数总计:1.8k|阅读时长:6分钟|阅读量:

前言

1. 什么是泛型 ？

泛型是Java5提供的一个新特性,它提供了编译时类型安全检测机制,该机制允许在编码时检测到非法的类型

泛型的本质是参数类型规则化,强类型化。

2.泛型有什么好处？

假如没有泛型,我们有这样一个需求,将一个int类型数据放入一个容器中然后再取出 如下:

List list = new ArrayList();
list.add(1);

Integer value = (Integer) list.get(0);//类型强转

这里可以看出 默认的List容器所能存储的元素为Object类型,也就是任意类型数据都可以存入,也就是说这是一个不可控的因素,但是通常我们在取出的时候是必须要强行转换所需的类型,否则无法使用。

假如我们存入元素时错放了数据类型会发生什么?

list.add("hello");

Integer value = (Integer)list.get(0); // ?
java.lang.ClassCastException: java.lang.String cannot be cast to java.lang.Integer

如果没有泛型的约束,与JavaScript 语言又有何区别?

加入泛型后

List<Integer> list = new ArrayList();
list.add(1);
list.add("qwe"); //无法通过编译

指定容器的泛型,在容器入参时就已经把这些不必要的风险通通扼杀掉,减少了大量本不需要存在的操作与风险

3. 泛型擦除

Java的泛型是伪泛型，这是因为Java在编译期间，所有的泛型信息都会被擦掉，正确理解泛型概念的首要前提是理解类型擦除。Java的泛型基本上都是在编译器这个层次上实现的，在生成的字节码中是不包含泛型中的类型信息的，使用泛型的时候加上类型参数，在编译器编译的时候会去掉，这个过程成为类型擦除。

无论在代码中定义ArrayList<String>还是ArrayList<Boolean>, 编译之后都会变成List,虚拟机能看到的也只是List,由泛型附加的类型信息虚拟机是察觉不到的。Java编译器会在编译时尽可能的发现可能出错的地方，但是仍然无法在运行时刻出现的类型转换异常的情况。下面我们通过实例验证一下~

List<Boolean> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();
System.out.println(list1.getClass() == list2.getClass());//true

两个不同泛型的ArrayList他们的类信息是一样的,说明泛型类型Boolean和Integer都被擦除掉了,只保留了原始类型。

4. 泛型限制

泛型在一些场景下也可以进行一些适当的约束

class Animal{

}

class Dog extends Animal{

}

class Cat{

}
public <T extends Animal> void show(T param){

}

通过extends关键字继承某个Class,表示只有是Animal或者Anmial的子类才符合标准。

也可以通过&关键字 做多重条件,必须实现哪些接口的Class。

public <T extends Animal & Serializable & Closeable> void show(T param){

}

5. 泛型定义

  • 泛型接口

    public interface SuperRepository<T extends SuperEntity,ID> extends JpaRepository<T,ID> {

}

  • 泛型类和泛型属性

    public class Result<T>{
    private T data;
}

  • 泛型方法

    public static <T> Result<T> Success(){

}

以上为最简单也是最基础的泛型使用,掌握后一步步衍变。

6. 泛型对象与数组创建

public static <T> T of(){
  return new T();// 编译失败
}
  • 反射创建
public static <T> T of(Class<T> clazz)throws Exception{
  return clazz.getConstructor().newInstance();
}
  • Supplier接口创建
public static <T> T of(Supplier<T> supplier){
  return supplier.get();
}
String of = Animal.of(() -> new String());
  • 创建泛型数组
public static <T> T[] ofArray(Class<T> clazz,int length){
  return (T[]) Array.newInstance(clazz,length);
}

7. 无界通配符

class Animal{

}

class Cat extends Animal{

}

public static void show1(List<? extends Animal> list){

}
public static void main(String[] args) {
  List<Cat> cats = Arrays.asList(new Cat());
  show1(cats);
}

  • ?通配符与T普通泛型的区别

    T表示元素只能是T类型

    ?表示元素可以是任意类型

T 是一个 确定的 类型，通常用于泛型类和泛型方法的定义，？是一个 不确定 的类型，通常用于泛型方法的调用代码和形参，不能用于定义类和泛型方法。

8. 协变与逆变

1. 协变是什么?

在两个类型之间建立某种类型的向上转型关系称之为协变.如下代码

List<? extends Fruit> flist = new ArrayList<Apple>();
flist.add(new Apple());  // 编译错误
flist.add(new Fruit());  // 编译错误
flist.add(new Object());  // 编译错误

List容器的泛型使用?通配符,extends指出了泛型的上界为Fruit。使用通配符可以将ArrayList向上转型了，也就实现了协变。

但是你会发现,我们通过add方法添加Fruit的子类到容器,编译显示无法通过,原因是因为现在编译器也不知道List<? extends Fruit>究竟是什么类型,所以一旦执行这种类型的向上转型,也就是协变,将会失去向容器内传入任何值的能力。泛型把类型检查移到了编译期，协变过程丢掉了类型信息，编译器拒绝所有不安全的操作。

2. 逆变是什么？

逆变与协变有点相反,通过super关键字指定?通配符的下界。

如List<? super Cat> 简单来说就是只能存入Cat以及 Cat的父类元素,但是获取出的元素都为Object

class Animal{
    private String name;
}

class Cat extends Animal{

}

class RadCat extends Cat{
    
}

public static void main(String[] args) {
  List<? super Cat> list = new ArrayList<>();
  list.add(new Cat());
  list.add(new Animal());
  list.add(new RadCat()); 
}
3. 协变的使用

现在有这样几个类,三个类为继承关系 Animal > Cat > RadCat

class Animal {
    void desc() {
        System.out.println("i im animal");
    }
}

class Cat extends Animal {
    @Override
    void desc() {
        System.out.println("i im cat");
    }
}

class RadCat extends Cat {
    @Override
    void desc() {
        System.out.println("i im redCat");
    }
}

现在有这样一个需求,定义一个无返回值方法,入参为一个序列,元素类型可以是Animal及以下任意子类

并且这个方法体中只会针对于该序列做消费操作,这个时候我们就可以用?通配符 + extends关键字指定泛型的上界,达成协变。元素泛型为协变的情况下,默认从序列中获取的元素类型都是协变的上界类型。

如下

public static void forEach(List<? extends Animal> list) {
  for (Animal cat : list) {
    cat.desc();
  }
}

接下来我们来使用下这个包含协变特性的方法

public static void main(String[] args) {

  forEach(Arrays.asList(new Animal()));

  forEach(Arrays.asList(new Cat()));

  forEach(Arrays.asList(new RadCat()));
}

Result

i im animal
i im cat
i im redCat

以上就是协变的用法及常用场景

4. 逆变的使用

还是这三个类,定义一个逆变序列,通过super关键字指定下界,表示这个序列中只能添加Animal及任意子类

public static void addAll(List<? super Animal> list) {
  list.add(new RadCat());
  list.add(new Cat());
  list.add(new Animal());
}
6.PECS原则
  • 频繁往外读取内容的，适合用上界Extends。
  • 经常往里插入的，适合用下界Super。
