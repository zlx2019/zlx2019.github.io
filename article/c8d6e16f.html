1. 并发集合

1. ConcurrentHashMap

ConcurrentHashMap是一个线程安全的HashMap。所有的操作都是线程安全的。该容器在JDK1.7之前由Segment实现,但是在JDK1.8版本进行了重构,其结构与HashMap一样,同样是数组+链表+红黑树组成,但是在线程安全策略上由Segment改为了synchronized+CAS。为其带来了卓越的性能,甚至在单线程环境下不逊于HashMap。它的Key与Value都不能为Null。

该容器提供了大量的基础方法与函数式方法,下面一一讲述。

  • 构造方法

    /**
 * 无参构造函数使用默认参数的负载因子与容量初始值
 * 默认初始容量: DEFAULT_CAPACITY = 16;
 * 默认负载因子: LOAD_FACTOR = 0.75f;
 * 默认并发级别: DEFAULT_CONCURRENCY_LEVEL = 16;
 */
new ConcurrentHashMap<>();

//初始化时指定初始容量
new ConcurrentHashMap<String, String>(20);

//初始化时指定初始容量与负载因子
new ConcurrentHashMap<String, String>(20,0.80f);

//初始化时指定初始容量、负载因子、并发级别
new ConcurrentHashMap<String, String>(20,0.80f,1);

  • 基础API

    • size() ：获取元素的数量。(并发情况下不推荐使用)

    • mappingCount：获取元素的数量。

    • isEmpty()：容器是否为空。

    • get(key)：根据key获取value。

    • getOrDefault(key,defVal)：根据key获取Value,如果key不存在则返回默认的value。

    • containsKey(key)：判断给定key是否存在。

    • containsValue(value)：判断给定value是否存在。

    • put(key,value)：添加元素。

    • putAll(map)：将一个map中的所有元素添加到该map中。

    • putifAbsent(key,value)：给定的key不存在时才设置元素并且返回null,如果已存在则返回已存在的key的value。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
System.out.println(map.putIfAbsent("test", "hello"));//输出: null
      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
System.out.println(map.putIfAbsent("test", "hi"));//输出:null
System.out.println(map.putIfAbsent("test", "hello"));//输出: hi

    • remove(key)：根据key删除元素。

    • remove(key,value)：根据key和value 删除元素。key必须存在且value也匹配。

    • replace(key,newValue)：替换一个元素的value。

    • replace(key,oldvalue,newValue)：根据key替换value值。

      key必须存在且value也匹配一致。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test", "hi");
boolean replace = map.replace("test", "hi", "hello");
System.out.println(replace);//true

      旧值如果不匹配则返回false。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test", "hi");
boolean replace = map.replace("test", "1", "hello");
System.out.println(replace);//false

    • clear()：清除所有元素。

    • keySet()：获取容器中所有的key,转换为Set结构。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test","value");
ConcurrentHashMap.KeySetView<String, String> keySet = map.keySet();
keySet.forEach(System.out::println);
//test

    • keys()：获取容器中所有的key,转换为 Enumeration类。

      Enumeration<String> keys = map.keys();
while (keys.hasMoreElements()){
  System.out.println(keys.nextElement());
}
//test

    • values()：获取容器中所有的value。

      Collection<String> value = map.values();

    • entrySet()：获取所有的元素Entry,通常遍历map的方案。

  • 函数式方法

    • forEach(BiConsumer)：遍历容器所有元素。

      map.forEach((k,v)-> System.out.println(k + ":" + v));

    • replaceAll(BiFunction)：遍历替换所有元素。

      //为每个元素的value 追加字符'123'
map.replaceAll((k,v)-> v + "123");

    • compute(key,BiFunction)：根据Key获取元素,并且对元素计算处理,最终返回元素的Value。注意:如果key不存在元素,BiFunction中的V则为null。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test", "hi");
String value = map.compute("test", (k, v) -> {
  //将value 转为大写
  return v.toUpperCase(Locale.ROOT);
});
System.out.println(value);//HI

    • computeIfAbsent(key,Function): 与 compute()类似。

    • computeIfAbsent(key,Function)

      如果给定的key,不存在容器中,则将函数返回的值,作为value添加到容器中,并且返回Value。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
String value = map.computeIfAbsent("test", (k) -> {
  return "Hello";
});
System.out.println(map.get("test"));//Hello
System.out.println(value);//Hello

      如果存在则不执行函数,直接返回已存在的Value。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test", "hi");
String value = map.computeIfAbsent("test", (k) -> {
  return "Hello";
});
System.out.println(value);//hi
System.out.println(map.get("test"));//hi

    • merge(key,appendVal,BiFunction fn)

      设置元素,如果key已存在,则将已有的value与新value重新处理后再添加进去,如果不存在则直接添加进去。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test", "hello");
String merge = map.merge("test", " world!", (v1, v2) -> {
  return v1 + v2;
});
System.out.println(merge);//hello world!

  • 并行方法

    JDK1.8提供了许多提高效率的并行方法。

    • forEach(parallelismThreshold,BiConsumer)：与普通的 forEach()方法类似,不同的是这个方法是支持并行的。是基于 ForkJoin来实现的。

      • parallelismThreshold: 任务拆分的并行处理粒度。
      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test", "hello");
map.put("test1", "hello");
map.put("test2", "hello");
map.put("test3", "hello");
map.put("test4", "hello");
map.forEach(2,(k,v)->{
  System.out.println(Thread.currentThread().getName() + ": key:" + k);
});
      main: key:test4
main: key:test2
main: key:test3
ForkJoinPool.commonPool-worker-2: key:test
ForkJoinPool.commonPool-worker-1: key:test1

    • forEach(parallelismThreshold,BiFunction,Consumer ) ：升级版 forEach(),增加了中间处理层函数 BiFunction。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test", "hello");
map.put("test1", "hello");
map.put("test2", "hello");
map.put("test3", "hello");
map.put("test4", "hello");
map.forEach(2,(k,v)->{
  return v.toUpperCase(Locale.ROOT);
},r-> {
  System.out.println(r);
});
      HELLO

    • search()：并行检索复合条件的元素。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test", "hello");
map.put("test1", "hello1");
map.put("test2", "hello2");
map.put("test3", "hello3");
map.put("test4", "hello4");
String search = map.search(2, (k, v) -> {
  if ("test4".equals(k) && "hello4".equals(v)){
    return v;
  }
  return null;
});
System.out.println(search);//hello4

    • searchKeys()：并行根据key查询元素。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test", "hello");
map.put("test1", "hello1");
map.put("test2", "hello2");
map.put("test3", "hello3");
map.put("test4", "hello4");
String value = map.searchKeys(2, key -> {
  if ("test".equals(key)) {
    return map.get(key);
  } else {
    return null;
  }
});
System.out.println(value);//hello

    • searchValues()：并行根据value查询元素。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test", "hello");
map.put("test1", "hello1");
map.put("test2", "hello2");
map.put("test3", "hello3");
map.put("test4", "hello4");
String result = map.searchValues(2, value -> {
  if ("hello2".equals(value)) {
    return value;
  } else {
    return null;
  }
});
System.out.println(result);//hello2

    • searchEntity()：并行检索entity。

      ConcurrentHashMap<String, String> map = new ConcurrentHashMap<>();
map.put("test", "hello");
map.put("test1", "hello1");
map.put("test2", "hello2");
map.put("test3", "hello3");
map.put("test4", "hello4");

Map.Entry<String, String> entries = map.searchEntries(5, entity -> {
  if ("test3".equals(entity.getKey()) && "hello3".equals(entity.getValue())) {
    return entity;
  } else {
    return null;
  }
});
System.out.println(entries);//test3=hello3

2. ConcurrentSkipListMap

ConcurrentSkipListMap也是一个性能卓越并且的线程安全的Map容器。它是基于跳表数据结构实现的,也是一个不错的选择。它与ConcurrentHashMap最大的不同就是,它是一个有序的容器,内部会根据key进行排序,并且它需要大量的内存空间来换取高效的查询效率。

这里只列举一下它独有的方法,其他方法直接参考ConcurrentHashMap。

  • 构造方法

    //构造一个新的map,使用key的自然排序。
new ConcurrentSkipListMap<>();

//构造一个新的map,并且将入参map的所有元素添加进map
Map<String, String>map = new HashMap<>();
new ConcurrentSkipListMap<>(map);

    • 自定义Map的排序规则。

      这里使用Integer作为key。默认的正序排序。

      ConcurrentSkipListMap<Integer,String> map = new ConcurrentSkipListMap<>((o1, o2) -> o1 - o2);
map.put(3,"Go");
map.put(1,"Java");
map.put(20,"TypeScript");
map.put(7,"Rust");
map.put(16,"Dart");
System.out.println(map.firstEntry());//1=Java

      翻转的逆序排序。

      ConcurrentSkipListMap<Integer,String> map = new ConcurrentSkipListMap<>((o1, o2) -> o2 - o1);
map.put(3,"Go");
map.put(1,"Java");
map.put(20,"TypeScript");
map.put(7,"Rust");
map.put(16,"Dart");
System.out.println(map.firstEntry());//20=TypeScript

  • 基础API

    初始化测试数据

    ConcurrentSkipListMap<Integer,String> map = new ConcurrentSkipListMap<>();
map.put(1,"Java");
map.put(3,"Go");
map.put(7,"Rust");
map.put(16,"Dart");
map.put(20,"TypeScript");

    • ceilingKey(key)

      获取指定的key,如果该key不存在,则返回比该key大的最小key。

      //返回大于或者等于给定key的最小key
//如现在入参为5,但是表中没有key为5的元素,那么就会找比5大的最小key,最终会找到7
Integer key = map.ceilingKey(5);
System.out.println(key);//key: 7
//如果给定的key存在,则直接返回key
key = map.ceilingKey(7);
System.out.println(key);//key: 7

    • ceilingEntry(key)

      根据key获取对应的Entry,如果key不存在,则返回比该key大的最小key的Entry。

      Map.Entry<Integer, String> entry = map.ceilingEntry(7);
System.out.println(entry);//7=Rust
entry = map.ceilingEntry(10);
System.out.println(entry);//16=Dart

    • floorKey(key) 与 floorEntry(key)

      获取给定的key,如果key不存在,则返回比该key小的最大key。

      Integer key1 = map.floorKey(5);
Integer key2 = map.floorKey(7);
System.out.println(key1);//3
System.out.println(key2);//7

    • LowerKey(key) 与 lowerEntry(key)

      返回严格小于给定键的最大键，如果没有这样的键，则返回null 。

      //返回比该key小的最大key(不包括本身),如果没有则返回null
Integer key = map.lowerKey(3);
System.out.println(key);//1

    • higherKey(key) 与 higherEntry(key)

      返回严格大于给定键的最小键，如果没有这样的键，则返回null 。

  • firstKey() 与 firstEntry()

    获取排序后的第一个key和第一位Entry。

    Integer firstKey = map.firstKey();
Map.Entry<Integer, String> firstEntry = map.firstEntry();
System.out.println(firstKey);//1
System.out.println(firstEntry);//1=Java

    注意:这里的首位并不是你首位插入的数据,而是容器根据Key重写排序后的数据。

  • lastKey() 和 lastEntry()

    获取排序后的最后一位key和Entry。

    Integer lastKey = map.lastKey();
Map.Entry<Integer, String> lastEntry = map.lastEntry();
System.out.println(lastKey);//20
System.out.println(lastEntry);//20=TypeScript

  • merge(key,value,BiFunction)

    向容器设置元素,如果该key不存在,则直接设置元素。

    String merge = map.merge(50, "Flutter", (oldv, newv) -> oldv + newv);
System.out.println(merge);//Flutter
System.out.println(map.get(50));//Flutter

    如果要设置的key已存在,则可以获取到旧值与新值处理后再设置进去。(这里已合并为例)。

    String merge = map.merge(16, "Flutter", (oldv, newv) -> oldv + "-" + newv);
System.out.println(merge);//Dart-Flutter
System.out.println(map.get(16));//Dart-Flutter

  • subMap(fromKey,toKey)

    截取Map容器中两个端点Key之间的元素,返回一个视图ConcurrentNavigableMap。

    /**
 * Map截取两个端点key之间的元素
 * fromKey:         起始Key
 * fromInclusive:   是否包含起始Key元素
 * toKey:           截止Key
 * toInclusive:     是否包含截止key元素
 *
 * 截取key为10-20区间的元素(包含10 与 20) 形成一个新的ConcurrentNavigableMap
 */
ConcurrentNavigableMap<Integer, String> newMap = map.subMap(10, true, 20, true);
System.out.println(newMap);//{16=Dart, 20=TypeScript}

    ConcurrentNavigableMap并非一个新的容器,而是一个容器的部分引用视图,对视图的修改同样会影响到原容器,同样的对原容器修改也会影响到视图。

    //获取整个容器的视图
ConcurrentNavigableMap<Integer, String> allMap = map.subMap(map.firstKey(), true, map.lastKey(), true);
//对视图进行添加元素
allMap.put(15,"Node");
System.out.println(allMap);//{1=Java, 3=Go, 7=Rust, 15=Node, 16=Dart, 20=TypeScript}
System.out.println(map);//{1=Java, 3=Go, 7=Rust, 15=Node, 16=Dart, 20=TypeScript}

    但是如果对引入视图插入元素时,超过了fromKey和toKey的范围之外,则会抛出IllegalArgumentException异常。

    ConcurrentNavigableMap<Integer, String> allMap = map.subMap(1, true, 10, true);
//对视图进行添加元素
allMap.put(11,"Node");//java.lang.IllegalArgumentException: key out of range

  • headMap(toKey,inclusive)

    获取小于或等于给定Key的元素,返回一个视图ConcurrentNavigableMap。

    /**
 * 获取小于或等于给定key的元素,返回一个视图对象ConcurrentNavigableMap
 * inclusive: 是否包含给定key元素
 * 
 * 获取key小于且等于16的元素视图
 */
ConcurrentNavigableMap<Integer, String> headMap = map.headMap(16, true);
System.out.println(headMap);//{1=Java, 3=Go, 7=Rust, 16=Dart}

    同样的headMap()返回的视图如果插入元素时,key超出范围后抛出异常。

    ConcurrentNavigableMap<Integer, String> headMap = map.headMap(16, true);
headMap.put(17,"Node");//IllegalArgumentException: key out of range

  • tailMap(toKey,inclusive)

    获取大于或等于给定key的元素视图。

    ConcurrentNavigableMap<Integer, String> tailMap = map.tailMap(10, true);
System.out.println(tailMap);//{16=Dart, 20=TypeScript}
    ConcurrentNavigableMap<Integer, String> tailMap = map.tailMap(10, true);
tailMap.put(9,"Node");//IllegalArgumentException: key out of range

  • descendingMap()

    获取map逆序翻转后的视图。

    ConcurrentNavigableMap<Integer, String> descendingMap = map.descendingMap();
System.out.println(map);//{1=Java, 3=Go, 7=Rust, 16=Dart, 20=TypeScript}
System.out.println(descendingMap);//{20=TypeScript, 16=Dart, 7=Rust, 3=Go, 1=Java}

  • navigableKeySet()

    获取映射的Key视图集合,列表key按照升序排序。

    NavigableSet<Integer> keySet = map.navigableKeySet();
System.out.println(keySet);//[1, 3, 7, 16, 20]

    对key视图集合的操作会直接影响到映射表,如对其remove()。

    NavigableSet<Integer> keySet = map.navigableKeySet();
System.out.println(map);//{1=Java, 3=Go, 7=Rust, 16=Dart, 20=TypeScript}
System.out.println(keySet);//[1, 3, 7, 16, 20]
keySet.remove(7);//true
System.out.println(map);//{1=Java, 3=Go, 16=Dart, 20=TypeScript}
System.out.println(keySet);//[1, 3, 16, 20]
    NavigableSet<Integer> keySet = map.navigableKeySet();
keySet.clear();
System.out.println(keySet);//[]
System.out.println(map);//{}

    该视图无法进行 add相关操作。

  • pollFirstEntry()

    移除并返回此映射中最小key元素。

    System.out.println(map);//{1=Java, 3=Go, 7=Rust, 16=Dart, 20=TypeScript}
System.out.println(map.pollFirstEntry());//1=Java
System.out.println(map);//{3=Go, 7=Rust, 16=Dart, 20=TypeScript}

  • pollLastEntry()

    移除并返回此映射中最大key元素。

    System.out.println(map);//{1=Java, 3=Go, 7=Rust, 16=Dart, 20=TypeScript}
System.out.println(map.pollLastEntry());//20=TypeScript
System.out.println(map);//{1=Java, 3=Go, 7=Rust, 16=Dart}

3. ConcurrentSkipListSet

ConcurrentSkipListSet底层是靠ConcurrentSkipListMap的Key实现的。与HashSet和HashMao的关系一致。所以这里就不多叙述。

4. CopyOnWriteArrayList

CopyOnWriteArrayList是ArrayList的线程安全版。通常用来解决一些读多写少的场景。如果写操作过多的情况下,尽量不要使用该容器,因为该容器写操作都是需要通过底层数组拷贝来解决线程安全问题,效率极低。

add()源码分析

public boolean add(E e) {
  final ReentrantLock lock = this.lock;
  lock.lock();//加锁
  try {
    //获取到数组和数组长度
    Object[] elements = getArray();
    int len = elements.length;
    //拷贝数据到一个新数组中
    Object[] newElements = Arrays.copyOf(elements, len + 1);
    //将要添加的元素 添加到新数组中
    newElements[len] = e;
    //将新数组覆盖旧数组
    setArray(newElements);
    return true;
  } finally {
    //释放锁
    lock.unlock();
  }
}

通过源码可以看到,如果有大量的写操作,那么将会严重阻塞线程的运行,并且还会消耗大量的内存空间。如set()、remove()等操作,都会进行数组的拷贝。

5. CopyOnWriteArraySet

CopyOnWriteArraySet内部是通过CopyOnWriteArrayList来实现,不多叙述~。

6. ConcurrentLinkedQueue

ConcurrentLinkedQueue是一个基于链表实现的无边界且线程安全队列。遵循于FIFO先进先出排列。该容器不支持Null元素。

7. ConcurrentLinkedDeque

ConcurrentLinkedDeque是基于链接节点的无边界的双端队列。并发插入、删除和访问操作跨多个线程安全执行。当许多线程将共享对公共集合的访问时， ConcurrentLinkedDeque是一个合适的选择。与大多数其他并发集合实现一样，此类不允许使用null元素。

8. Collections构建安全容器

通过Collections能够构建出线程安全的各种容器。

  1. Collection

    Collection<Integer> a = Collections.synchronizedCollection(new ArrayList<>());

  2. List

    List<Integer> list = Collections.synchronizedList(new ArrayList<>());

  3. Set

    Set<Integer> set = Collections.synchronizedSet(new HashSet<>());

  4. Map

    Map<String, Object> map = Collections.synchronizedMap(new HashMap<>());

  5. SortMap

    SortedMap<String, Object> sortedMap = Collections.synchronizedSortedMap(new TreeMap<>());

  6. SortSet

    SortedSet<String> sortedSet = Collections.synchronizedSortedSet(new TreeSet<>());

  7. NavigableMap

    NavigableMap<String, Object> navigableMap = Collections.synchronizedNavigableMap(new TreeMap<>());

  8. NavigableSet

    NavigableSet<String> navigableSet = Collections.synchronizedNavigableSet(new TreeSet<>());

2. 阻塞队列

1. ArrayBlockingQueue

ArrayBlockingQueue是一个基于数组实现的阻塞队列容器。它是不可扩容的,通常在创建时指定了容量后则无法 修改。

  • API

添加元素

  1. add

    向队列插入元素,成功返回true,如果队列已满,则抛出IllegalStateException异常。

    ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(3);
queue.add("1");//true
queue.add("2");//true
queue.add("3");//true
queue.add("4");//java.lang.IllegalStateException: Queue full

  2. offer

    向队列插入元素,成功返回true,如果队列已满,则返回false,此方法比 add更可取。

    ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(3);
queue.offer("1");//true
queue.offer("2");//true
queue.offer("3");//true
queue.offer("4");//false

    指定超时时间,在指定时间内,队列还是满的,无法插入则返回false。

    queue.offer("1",1,TimeUnit.SECONDS);

  3. put

    向队列插入元素,如果队列已满,则阻塞等待。

    ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(3);
queue.put("1");
queue.put("2");
queue.put("3");
queue.put("4");//wait

获取元素

  1. poll

    获取并移除队列头部元素,如果队列为空,则返回null。

    ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(3);
System.out.println(queue.poll());//null

    可以指定超时时间,在指定的时间内,如果还是没有可用元素,则返回null。

    ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(3);
System.out.println(queue.poll(1, TimeUnit.SECONDS));//

  2. take

    获取并移除队列头部元素,如果队列为空,则会阻塞等待可用元素。

    ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(3);
new Thread(()->{
  TimeUnit.SECONDS.sleep(2);
  queue.add("1");
}).start();
System.out.println(queue.take());// 1

  3. peek

    获取队列头部元素,但不会移除该元素,如果没有可用元素,则返回null。

    ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(3);
String peek = queue.peek(); //null
queue.add("1");
System.out.println(queue.peek());// 1

  4. element

    获取队列头部元素,但不会移除该元素,如果没有可用元素,则抛出NoSuchElementException移除。

    ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(3);
queue.element(); //java.util.NoSuchElementException

  5. remove

    获取并移除队列头部元素,如果没有可移除的元素则抛出NoSuchElementException异常。

    ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(3);
queue.remove();//java.util.NoSuchElementException

    remove(Object obj)

    从队列中,移除指定的元素,成功则返回true,如果元素不存在,则返回false。

    ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(3);
queue.add("1");
queue.add("2");
queue.add("2");
System.out.println(queue.remove("2"));//true

其他API

  1. drainTo

    将队列中所有可用元素,移除到指定容器中。

    ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(3);
IntStream.range(0,3).boxed().map(Object::toString).forEach(queue::offer);

List<String> container = new ArrayList<>();
int addResult = queue.drainTo(container);
System.out.println(addResult);//3
System.out.println(container);//[0, 1, 2]

System.out.println(queue);//[]

    指定移动的元素数量

    System.out.println(queue);//[0, 1, 2]
//只移除队列前n个元素
int addResult = queue.drainTo(container,2);
System.out.println(addResult);//2
System.out.println(container);//[0, 1]

System.out.println(queue);//[2]

  2. remainingCapacity

    队列当前剩余可用容量。

2. PriorityBlockingQueue

PriorityBlockingQueue同样是基于数组实现的阻塞队列,与ArrayBlockingQueue不同的是,它是一个无边界的阻塞队列,逻辑上来说是无边界的,所以存在 会OOM的情况。并且它是一个可指定优先级的队列,也就是说他并不一定是 先进先出消费模型。该队列不支持添加Null元素。

构造函数

  1. 无参构造,一切都使用默认的
PriorityBlockingQueue<Integer> queue1 = new PriorityBlockingQueue<>();
  1. 指定队列初始容量,设置初始值的意义并不是特别大,随着数据量的增加,会自动扩容。
PriorityBlockingQueue<Integer> queue2 = new PriorityBlockingQueue<>(10);
  1. 初始化数据,在构造时传入一个列表,会将列表中的元素添加至队列中。
PriorityBlockingQueue<Integer> queue3 = new PriorityBlockingQueue<>(Arrays.asList(1,2,3,4));
  1. 设置比较规则比较器
PriorityBlockingQueue<String> queue4 = new PriorityBlockingQueue<>(10, (o1, o2) -> o1.compareTo(o2));

API与ArrayBlockingQueue基本一致。

3. LinkedBlockingQueue

LinkedBlockingQueue是一个基于链表实现的 可无界可有界限的阻塞队列。遵循与FIFO排序,链表队列通常比基于数组的队列吞吐量更高。如果不指定队列容量,那默认为Integer.MAX_VALUE。如果要使用该队列,那么必须要指定容量大小

4. LinkedBlockingDeque

LinkedBlockingDeque是一个基于链表实现的 双端阻塞队列。与LinkedBlockingQueue基本一致。

它提供了许多复合API,如addFirst(),addLast() 对应于其他队列的add()方法。

5. SynchronousQueue

SynchronousQueue是一个 同步阻塞队列。甚至不应该称之为队列,因为它甚至没有容量的概念,它更像是一个传输响应器,一个(put)插入元素操作必须对应另一个线程的(take)获取元素操作,否则将会阻塞住。与java.util.concurrent.Exchanger非常类似,只不过Exchanger是数据的双向传输，而SynchronousQueue是数据单向传输。SynchronousQueue可选公平与非公平策略。

SynchronousQueue的API与其他阻塞队列不一样,使用的时候需要谨慎,如peek()始终返回Null、remove()始终返回false。该队列通常就用put()与take()。

6. LinkedTransferQueue

LinkedTransferQueue基于链表实现的阻塞队列,它于其他队列不同的是,它可以保证队列中的元素必须被消费。它提供了一组API transfer与tryTransfer,来保证该特性。可以让元素直接被消费,无需借助于队列存储。

API 描述
tryTransfer(E e) 如果存在线程正在等待消费它(take 或者 timed poll),则立刻将元素添加并被消费,否则返回false表示不让该元素添加至队列中。
tryTransfer(E e, long timeout, TimeUnit unit) 在指定的时间内,有消费者消费,则返回true,否则返回false。
transfer(E e) 阻塞式将元素传输给消费者,如果没有人消费将会一直阻塞等待。

案例-没有消费者的情况下

public static void main(String[] args) throws InterruptedException {
  LinkedTransferQueue<Integer> queue = new LinkedTransferQueue<>();
  //1.Result: false,因为目前没有正在等待消费者
  System.out.println(queue.tryTransfer(1));

  //2. 阻塞3秒后,返回false,因为目前没有正在等待消费者
  System.out.println(queue.tryTransfer(1,3, TimeUnit.SECONDS));

  //3. 这里会一直阻塞,直到有消费者消费后。
  queue.transfer(1);
}

案例-存在消费者的情况下

public static void main(String[] args) throws InterruptedException {
  LinkedTransferQueue<Integer> queue = new LinkedTransferQueue<>();
  //启动一个消费者不断消费
  new Thread(()->{
    for (;;){
      try {
        Integer take = queue.take();
        System.out.println("消费到数据:" + take);
      } catch (InterruptedException e) {
        e.printStackTrace();
      }
    }
  }).start();
  TimeUnit.SECONDS.sleep(1);

  queue.tryTransfer(1);//true
  queue.tryTransfer(1,3, TimeUnit.SECONDS);//true
  queue.transfer(1);
}

7. DelayQueue

DelayQueue是一个无边界的延时阻塞队列,能够让元素延迟一定的时间后再被消费。使用过程基本上与其他队列一致,唯一不同的是它的元素,必须是继承于Delayed的。重写Delayed的getDelay()逻辑方法。

自定义一个延迟实体

/**
 * 可被放入延迟队列的实体 必须实现 #{@link Delayed}
 *
 * @author Zero.
 * @date 2022/4/24 9:10 下午
 */
public class DelayEntity<E> implements Delayed {
    //实体元素
    private final E entity;
    //该元素延迟结束的时间戳
    private final long expireTime;

    /**
     * 构造函数
     *
     * @param entity 元素
     * @param delay  延迟消费时长 单位:毫秒
     */
    public DelayEntity(E entity, long delay) {
        this.entity = entity;
        this.expireTime = System.currentTimeMillis() + delay;
    }

    /**
     * 构造延迟元素
     *
     * @param entity 元素
     * @param delay  延迟消费时长 单位: 毫秒
     */
    public static <E> DelayEntity<E> of(E entity, long delay) {
        return new DelayEntity<>(entity, delay);
    }

    /**
     * 获取元素
     *
     * @return E
     */
    public E getEntity() {
        return entity;
    }

    /**
     * 获取过期时间
     */
    public long getExpireTime() {
        return this.expireTime;
    }

    /**
     * 获取当前元素剩余的延迟时间
     * 如果小于0,则表示延迟时间已到达,可消费。
     */
    @Override
    public long getDelay(TimeUnit unit) {
        long diff = expireTime - System.currentTimeMillis();
        return unit.convert(diff, TimeUnit.MILLISECONDS);
    }

    /**
     * 元素添加到队列的优先比较方法,阻塞时间较短的优先放置到队列中,先行被消费。
     */
    @Override
    public int compareTo(Delayed delayed) {
        DelayEntity that = (DelayEntity) delayed;
        if (this.expireTime < that.getExpireTime()) {
            return -1;
        } else if (this.expireTime > that.getExpireTime()) {
            return 1;
        } else {
            return 0;
        }
    }
}

延迟消费案例

public static void main(String[] args) throws InterruptedException {
  DelayQueue<DelayEntity<Integer>> delayQueue = new DelayQueue<>();
  SimpleDateFormat format = new SimpleDateFormat("HH:mm:ss");
  //添加元素,延迟5秒后才可以被消费
  delayQueue.add(DelayEntity.of(1,5000));
  System.out.println(format.format(new Date()) +"添加元素,延迟为5秒...");
  //阻塞式获取元素
  Integer entity = delayQueue.take().getEntity();
  System.out.println(format.format(new Date()) +"消费元素:" + entity);
}
21:34:38添加元素,延迟为5秒...
21:34:43消费元素:1

可以看到,take()方法是可以延迟获取的,但是 peek()方法呢?

Integer entity = delayQueue.peek().getEntity();
21:38:57添加元素,延迟为5秒...
21:38:57消费元素:1

peek()方法不会延迟,它会立即获取到元素,所以使用的时候一定要注意使用的方法。

那么 poll()方法呢?我们继续测试一下,获取元素修改为poll()

Integer entity = delayQueue.poll().getEntity();
21:41:33添加元素,延迟为5秒...
Exception in thread "main" java.lang.NullPointerException

哦豁,直接抛出了异常,因为在该队列中,poll()方法默认使用的就是 peek()方法。如果给 poll(time)方法设置了超时时间呢？继续测试~

Integer entity = delayQueue.poll(5, TimeUnit.SECONDS).getEntity();
21:44:38添加元素,延迟为5秒...
21:44:43消费元素:1

可以看到,只要在poll()方法超时时间内,元素的延迟时间到达后, poll()方法也是可以做到延迟消费的。

小总结:

​ 延迟消费必须使用 take()或者 带有超时时间的poll()方法。

API

基础API大多功能一致,因为它们都继承自AbstractQueue以及实现于BlockingQueue。

添加元素 描述
add() 成功返回true,失败抛出IllegalStateException异常。
offer() 成功返回true,失败返回false。
put() 添加失败则会阻塞,直到添加成功或线程中断。
获取元素 描述
element() 获取元素,如果没有可用元素则抛出NoSuchElementException异常。
peek() 获取元素,如果没有可用元素则返回Null。
获取并移除元素 描述
remove() 获取并移除队列头部元素,没有可移除元素则抛出NoSuchElementException异常。
pop() 获取并移除队列头部元素,没有可移除元素则返回Null。
take() 获取并移除队列头部元素,没有可移除元素则会阻塞等待至成功或线程中断。
