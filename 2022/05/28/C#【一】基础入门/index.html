C#【一】基础入门

前言

C# 语言是微软推出的一款面向对象的编程语言，凭借其通用的语法和便捷的使用方法受到了很多企业和开发人员的青睐。C# 语言具备了面向对象语言的特征，即封装、继承、 多态，并且添加了事件和委托，增强了编程的灵活性。C#是作为.NET平台下的首选编程语言。

C#所擅长的领域非常之多机器学习、游戏、Windows应用等等,非常多,一众被称为微软亲儿子,当然,本人是奔着Unity而来。特别简单的知识这里不会进行记录。

这里基于.NET6哦

1. 基本数据类型

C#的数据类型分为 值类型和引用类型。

值类型包括整型、浮点型、字符型、布尔型、枚举。

引用类型包括类、接口、数组、委托、字符串。

从内存存储空间的角度而言，值类型的值是存放到栈中的，每次存取值都会在该内存中操作；引用类型首先会在栈中创建一个引用变量，然后在堆中创建对象本身，再把这个对象所在内存的首地址赋给引用变量。

1.1 整型

所谓整型就是存储整数的类型,按照所存储值的范围,C#进行了划分 分别为以下八种:

sbyte、short、int、long为有符号数。

byte、ushort、uint、ulong为 无符号数。

有符号表示值可以为负数。无符号表示值只能为正数

所占字节与值的范围如下表:

类 型 大小 取值范围
sbyte 有符号数，占用1个字节 -128 到 127
byte 无符号数，占用1个字节 0 到 255
short 有符号数，占用2个字节 -32,768 到 32,767
ushort 无符号数，占用2个字节 0 到 65,535
int 有符号数，占用4个字节 -2,147,483,648 到 2,147,483,647
uint 无符号数，占用4个字节 0 到 4,294,967,295
long 有符号数，占用8个字节 -9,223,372,036,854,775,808 到 9,223,372,036,854,775,807
ulong 无符号数，占用8个字节 0 到 18,446,744,073,709,551,615

在C#中默认的整数类型是int类型。

1.2 浮点型

浮点型有如下两种

类 型 取值范围
float 单精度浮点型，占用4个字节，最多保留7位小数
double 双精度浮点型，占用8个字节，最多保留16位小数

默认的浮点型是double类型,如果要使用单精度浮点型,需要在数值后面加上f或F,如1.23f、25.22F。

1.3 字符型

char字符型,占用两个字节,只能存储一个汉字或字母,使用单引号,如:你、我。

string字符串型,能存储多个字符,是引用类型,使用双引号包括。

1.4 布尔类型

bool 只有两个值: true 和false。

2. 常量

定义常量使用const来完成。语法如下

const string name = "Hello";

需要注意的是，在定义常量时必须为其赋值，因为不赋值的话以后就再也不能赋值了。另外，也可以同时定义多个常量。

3. 类

类是引用类型,在绝大数编程语言中都包含类的概念。

3.1 定义类

与Java一致

访问修饰符 修饰符 class 类名{
  属性
}
  • 访问修饰符下面会单独讲述
  • 修饰符: 是针对与类本身的特点描述,包括abstract、sealed、static。
    • abstract表示为抽象类。
    • sealed表示该类不能被继承。
    • static表示为静态类,不能被实例化。

3.2 访问修饰符

类、属性、方法通常都需要访问修饰符,一共有如下四种访问修饰符:

  • public

    全局公开,可以任意访问。

  • private

    只允许在本类中访问。

  • internal

    只允许在本项目中访问。

  • protected

    只允许该类下的子类访问。

如果类没有定义访问修饰符,则默认为internal。

如果类属性没有定义访问修饰符.则默认为private。

3.3 定义类方法

语法如下:

访问修饰符  修饰符  返回值类型  方法名(参数列表){
  
}

方法的访问修饰符四种都可以使用,默认为private。

方法的修饰符较多,有virtual、abstract、override、static、sealed。

  • abstract表示为抽象方法,子类必须要重写该方法。
  • static表示为静态方法,可以用过类直接调用。
  • sealed表示该方法不允许被子类重写。
  • virtual和override是继承是所使用,稍微会说到。

3.4 Lambda表达式

使用Lambda简化一个方法

public static int Add(int a, int b) => a + b;

public static double Sum(double a, double b) => a + b;

3.5 类的继承

C#和Java一样,都是单继承,主要解决代码复用问题。中通过:关键符号实现继承。

public class Animal
{
    
}

public class Dog : Animal
{
    
}

虚拟与重写

父类与子类同时实现一个方法后,如果不显式通过vartual和override来重写的话则会执行父类的实现

public class Animal
{
    public void show()
    {
        Console.WriteLine("Animal");
    }
}

public class Dog : Animal
{
    public void show()
    {
        Console.WriteLine("Dog");
    }
}

Animal animal =new Dog();
animal.show();

执行结果

Animal

可以看到,这种情况是没有通过继承而实现多态。

只有通过vartual和override才能实现多态的效果。

public class Animal
{
    public virtual void show()
    {
        Console.WriteLine("Animal");
    }
}

public class Dog : Animal
{
    public override void show()
    {
        Console.WriteLine("Dog");
    }
}
Animal animal =new Dog();
animal.show();

执行结果

Dog

通过 new重写虚方法

public class Cat : Animal
{
    public new void show()
    {
        Console.WriteLine("Cat");
    }
}
Animal cat = new Cat();
cat.show();

执行结果

Animal

virtual： 表示该方法为虚方法,允许在子类中重写该方法。默认是非虚的,不可以重写

override：与virtual紧密不可分的,表示重写虚方法

new: 同样也是重写虚方法的关键字,与override不同的是,它会隐藏此方法,所以会执行父类方法

简单来说就是: 父类引用指向子类对象的情况下虚方法通过override关键字重写后会执行子类的实现,而通过new关键字重写后会执行父类的实现。

4. ref与out

在C#中,基本数据类型是值传递,引用类型是引用传递,这与Java类似,但是它有能让基本数据类同样具备引用传递的特性。也就是通过ref与out关键字。如下所示

public class Util
{
    public static void Swap(ref int n1,ref int n2)
    {
        n1 = n1 ^ n2;
        n2 = n1 ^ n2;
        n1 = n1 ^ n2;
    }
}
int n1 = 101;
int n2 = 202;
Console.WriteLine("交换前: n1:{0} n2:{1}",n1,n2);
Util.Swap(ref n1,ref n2);
Console.WriteLine("交换后: n1:{0} n2:{1}",n1,n2);

执行结果:

交换前: n1:101 n2:202
交换后: n1:202 n2:101

在方法形参类型前添加ref关键字,表示该参数为引用传递。注意:在入参时也要使用ref关键字修饰。

ref与out的区别

  1. ref入参时必须有初始值,在方法内同样可以获取到参数的初始值。
public static void testRef(ref int n1)
{

}
// 错误用法
int num;
Util.testRef(ref num);//Local variable 'num' might not be initialized before accessing
// 正确用法
int num = 10;
Util.testRef(ref num);
  1. out在方法结束前,必须要进行赋值。
  2. out方式入参后,会将值直接抹除掉,无法在方法内部获取到值,所以一般out方式的参数不需要初始值。
public static void TestOut(out int n1)
{
	n1 = 1;
}

int num2;
Util.TestOut(out num2);
Console.WriteLine(num2);

总结: 如果想用基本类型实现引用传递则使用ref。如果只是想作为一个方法返回的载体则使用out。

5. 委托

C#中的委托,简单来说就是方法管理器,可以将多个方法提交给一个委托,由委托进行管理。

创建委托语法

delegate 方法返回类型 委托名称(方法参数列表);

如下案例,通过一个委托,执行所有技能方法。

  1. 创建技能委托
// 定义技能委托,技能方法没有返回值并且无需任何参数
delegate void Skilldele();
  1. 定义技能方法
public class Skill
{
  public static void ReleaseSkill1()
  {
    Console.WriteLine("释放技能一动画");
  }

  public static void ReleaseSkill2()
  {
    Console.WriteLine("释放技能二动画");
  }

  public static void ReleaseSkill3()
  {
    Console.WriteLine("释放技能三动画");
  }

  public static void ReleaseSkill4()
  {
    Console.WriteLine("释放技能四动画");
  }

  public void ReleaseSkill5()
  {
    Console.WriteLine("释放技能五动画");
  }
  public void ReleaseSkill6()
  {
    Console.WriteLine("释放技能六动画");
  }
}
  1. 初始化委托
// 实例化委托,并且绑定一个方法
Skilldele skilldele = new Skilldele(Skill.ReleaseSkill1);
  1. 绑定多个方法
//绑定静态方法
skilldele += Skill.ReleaseSkill2;
skilldele += Skill.ReleaseSkill3;
skilldele += Skill.ReleaseSkill4;
// 绑定普通方法
Skill skill = new Skill();
skilldele += skill.ReleaseSkill5;
skilldele += skill.ReleaseSkill6;
//移除方法
//skilldele -= Skill.ReleaseSkill1;
  1. 执行委托所有方法
//执行委托中的所有方法
skilldele.Invoke();

执行结果:

释放技能一动画
释放技能二动画
释放技能三动画
释放技能四动画
释放技能五动画
释放技能六动画
