Go【十】泛型

发表于2022-05-11|更新于2023-03-01|Go
|字数总计:1.4k|阅读时长:5分钟|阅读量:

前言

Go1.8版本推出也有了一些时间,对于新推出的 泛型,早已饥渴难耐,早期的时候一直不太想深入学习Go其中很大一部分原因就是因为没有 泛型。泛型对于一个编程语言来说,是意义非凡的,没有 泛型难免会很蛋疼的写过多重复、冗余代码,这是强迫症不能允许的。本章就对 Go泛型进行仔细深入的学习。

当然,Go的泛型是刚刚支持的,与其他更早支持 泛型的语言来说还差很远,请不要用这点来诋毁它,相信后续会越来越完善。

1. 泛型函数

定义泛型函数语法如下:

func 函数名[泛型名称 泛型类型](){
  
}

如定义一个函数,允许传入int或string类型的切片

func PrintlnSlices[E int | string](list []E) {

}

[E int|string] 表示泛型E 可以是 int 或者string,相当于一种约束。

PrintlnSlice([]string{"张三", "李四", "王五", "赵六"})
PrintlnSlice([]int{1, 2, 3, 4})

如果想要扩大泛型的约束,允许任意类型的切片传入,可以使用interface{},但是推荐使用any,any是1.8推出的一个类型,用来表示任意类型,其实本质上any就是interface{},只是一个别名。我们可以通过源码看到:

// byte is an alias for uint8 and is equivalent to uint8 in all ways. It is
// used, by convention, to distinguish byte values from 8-bit unsigned
// integer values.
type byte = uint8

// rune is an alias for int32 and is equivalent to int32 in all ways. It is
// used, by convention, to distinguish character values from integer values.
type rune = int32

// any is an alias for interface{} and is equivalent to interface{} in all ways.
type any = interface{}
// PrintlnSlice 定义一个泛型函数
// 该函数通过泛型 表示允许接受任何类型的切片
func PrintlnSlice[E any](list []E) {
	for _, item := range list {
		fmt.Print(item)
		fmt.Print(",")
	}
	fmt.Println()
}

泛型函数在使用时,可以指定类型,也可以直接入参使用:

func main() {
	PrintlnSlice([]string{"张三", "李四", "王五", "赵六"})
	PrintlnSlice([]int{1, 2, 3, 4})
  //指定函数的泛型为[float64]
	PrintlnSlice[float64] ([]float64{1.1, 2.2, 3.3})
}

输出如下:

张三,李四,王五,赵六,
1,2,3,4,
1.1,2.2,3.3,

2. 泛型切片

定义泛型切片

// List 定义一个泛型切片,泛型为任意类型
type List[T any] []T

使用该切片

func main() {
	//使用泛型切片
	list := List[string]{"张三", "李四", "王五", "赵六"}
	list2 := List[int64]{1, 2, 3, 4}
	list3 := List[struct{}]{struct{}{}, struct{}{}}
	PrintlnSlice(list)
	PrintlnSlice(list2)
	PrintlnSlice(list3)
}

输出如下:

张三,李四,王五,赵六,
1,2,3,4,
{},{},

3. 泛型map

定义泛型map

// HashMap 定义一个泛型map
type HashMap[K string, V any] map[K]V

注意:Go的map的key 不支持any,官方推荐为string

使用

hashmap := HashMap[string, bool]{}
hashmap["isActive"] = true
hashmap["isLocked"] = false
fmt.Println(hashmap)

输出如下:

map[isActive:true isLocked:false]

4. 泛型channel

定义

// Channel 定义泛型通道
type Channel[T any] chan T

使用

chanl := make(Channel[int], 10)
chanl <- 1001
chanl <- 1002

5. comparable

comparable也是go内置的一个类型,它表示的是可比较类型,如int,string,struct等等。

5. 接口组合约束

当我们要指定很多泛型的时候,直接写在函数后有时显得过于臃肿,我们可以通过interface进行复合。

type Num interface {
	int | int8 | int16 | int32 | int64
}
type Str interface {
	string
}
type NumWithStr interface {
	Num | Str
}

type Slice[T NumWithStr] []T

这样我们只需要将泛型指定为NumWithStr,那么就同时兼容了这些复合的所有类型。

6. 衍生约束

通过~表示符,表示支持该类型下衍生出的所有类型。

如下所示:

type CustomInt int
func PrintInt[T int](list []T) {}

我们通过type自定义了一个 int类型,如果将该类型作为切片元素类型当做参数,那么能否执行该方法呢?

ints := []CustomInt{1, 2, 3}
PrintInt(ints)//err

很显然,在编码阶段已经无法通过

Cannot use CustomInt as the type interface{} Type does not implement constraint 'interface{}' because type is not included in type set ('int')

这个时候,我们只需要通过~标识符,就可以解决该问题。

func PrintInt[T ~int](list []T) {

}

这表示,所有通过int衍生出的子类型,都可以作为入参的切片类型。

7. 接口约束

定义一个抽象接口

type Animal interface {
	Call()
}

该接口内持有一个Call()方法.

定义一个函数

func Each[T Animal](animals []T) {
	for _, animal := range animals {
		animal.Call()
	}
}

函数泛型类型为Animal接口。

在Go中,实现一个接口其实就是重写接口的所有方法即可。

定义实现结构体

type Dog struct {
}

func (d Dog) Call() {
	fmt.Println("汪汪汪")
}

执行泛型函数

func main() {
	dogs := []Dog{Dog{}, Dog{}}
	Each(dogs)
}
汪汪汪
汪汪汪

执行成功,因为Dog实现了Aniaml接口,如果没有实现该接口的所有方法,是无法作为该泛型入参。

那么我们再复杂一些,为该接口增加~int|~string约束.

type Animal interface {
	Call()
	~int | ~string
}

Animal接口改动后,代码编译直接报错

func main() {
	dogs := []Dog{Dog{}, Dog{}}
	Each(dogs) //编译出错
  //Cannot use Dog as the type Animal Type does not implement constraint 'Animal' because type is not included in type set ('~int', '~string')
}

意思大致: Dog没有实现约束,因为Dog没有包含在类型集中(‘int’， ‘string’)

那么复杂后的这种约束该如何满足呢?我们来梳理一下约束条件:

  1. 实现Call()方法
  2. 底层类型必须基于int或string

那么就很简单了,我们定义一个类型,基于int或者string然后实现Call()方法即可.如下:

type Cat string

func (c Cat) Call() {
	fmt.Println(c + ":喵喵喵")
}

然后使用Cat入参:

func main() {
	cats := []Cat{Cat("小花"), Cat("小喵")}
	Each(cats) //编译出错
}

输出结果:

小花:喵喵喵
小喵:喵喵喵

以上就是 接口约束 + 衍生约束的使用
