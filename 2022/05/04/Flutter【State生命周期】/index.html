Flutter【State生命周期】

发表于2022-05-04|更新于2022-05-04
|字数总计:363|阅读时长:1分钟|阅读量:

1. 两种Widget的区别

StatelessWidget与StatefulWidget的区别。

StatelessWidget

class Page1 extends StatelessWidget{
  @override
  Widget build(BuildContext context) {
    return Container();
  }
}

StatefulWidget

class Page2 extends StatefulWidget{
  @override
  State<StatefulWidget> createState() => _Page2State();
}
class _Page2State extends State<Page2>{
  @override
  Widget build(BuildContext context) {
    throw Container();
  }
}

从语法层面上看,StatefulWidget将自身的build方法抽离到了State类中。这也是我们常说的,StatelessWidget发生变动后,将会重新渲染该组件,而StatefulWidget则不会,其实就是会不会重新执行build方法。

2. State生命周期函数

1. initState

当对象渲染到组件树时将执行该函数。
框架将为它创建的每个 [State] 对象仅调用一次此方法。

2. didChangeDependencies

当此 [State] 对象的依赖项发生更改时调用。例如，如果先前对 [build] 的调用引用了后来更改的 [InheritedWidget]，则框架将调用此方法来通知此对象有关更改.
在 [initState] 之后也立即调用此方法。
该方法执行完后会执行build方法

3. didUpdateWidget

当组件的状态改变的时候就会调用didUpdateWidget,比如调用了setState.
实际上这里flutter框架会创建一个新的Widget,绑定本State，并在这个函数中传递老的Widget。
这个函数一般用于比较新、老Widget，看看哪些属性改变了，并对State做一些调整。

4. deactivate

当从组件树中删除此对象时调用。

5. dispose

当此对象从树中永久删除时调用。[deactivate]，在 [dispose] 之前调用。
