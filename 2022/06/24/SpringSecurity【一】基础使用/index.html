SpringSecurity【一】基础使用

发表于2022-06-24|更新于2022-06-24|SpringSecurity
|字数总计:478|阅读时长:2分钟|阅读量:

前言

SpringSecurity是Spring家族的安全认证框架。背后有Spring大家族支撑,也是目前非常主流且完善的安全框架。但是由于它配置自定义扩展较为复杂,一直没有深入的学习,本章就简单记录一下学习过程。

环境搭建

这里选用SpringBoot-2.5.5版本。

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.5.5</version>
        <relativePath/> <!-- lookup parent from repository -->
    </parent>
    <groupId>com.zero.auth</groupId>
    <artifactId>zeus-auth</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <name>zeus-auth</name>
    <dependencies>
        <!--Web-->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
        <!--Security 认证授权框架-->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-security</artifactId>
        </dependency>
</project>

1. 初始化

默认导入spring-boot-starter-security依赖后,SpringSecurity会默认保护所有的请求,只有经过认证后才可以访问。

当随意访问一个接口后,会跳转到/login,进行认证。

默认的用户名为:user

密码在项目启动日志中所输出: 

2022-06-24 21:22:37.961  INFO 15531 --- [           main] .s.s.UserDetailsServiceAutoConfiguration : 
Using generated security password: 0e652293-6a36-4916-8c20-84c251155959

认证通过后将会成功访问到本该访问到的资源。

默认的user用户是存储于内存中的,我们可以通过配置的方式进行修改用户名及密码：

spring:
  security:
     #默认的内存用户名与密码修改
    user:
      name: admin
      password: admin
