Unity【一】MonoBehaviour

游戏开发
Unity
发布日期:   2022-06-08
文章字数:   1.3k
阅读时长:   4 分
阅读次数:  

前言

MonoBehaviour是一个基类,所有要挂载到GameObject上的脚本都必须继承于该类。

MonoBehaviour>Behaviour>Component>Object。重点就在于Component这个类,它是脚本组件的最顶级基类。

生命周期函数

本章主要记录一下该类的周期函数。

1. Start

Start()初始化函数。该函数会在第一帧更新之前进行执行,也就是第一次执行Update()函数之前调用。并且该函数在脚本的整个生命周期中,只会被调用一次。

//  Start函数
void Start()
{
  //TODO
}

2. Update

Update()帧率函数。当挂载该脚本的GameObject为启用状态时,每一帧都会执行该函数。该函数是最常用的函数。注意:不同的计算机它的帧率是不同的,可能会导致执行次数不一致的情况。通常该函数用来获取一些状态控制信息。

// 每帧调用一次更新
void Update()
{
  //TODO
}

3. Awake

Awake()该函数会在加载脚本实例时执行。他是优先于Start()函数之前所执行的,因为Start()是属于渲染阶段的执行函数,加载阶段优先于渲染阶段。通常都是在该函数内来初始化变量或状态。

该函数会在当前场景所有的GameObject初始化完后执行,在该函数内可以安全的使用GameObject.FindWithTag("tag")等方式查找并且赋值其他GameObject。

不同的GameObject的Awake()函数的执行顺序是随机的。

该函数只会执行一次。

private void Awake()
{
  //TODO 初始化变量、状态等
  // 根据tag属性动态查找其他GameObject
  var player =GameObject.FindWithTag("Player");
}

4. Start与Awake

Start()与Awake()函数的区别如下:

  1. Awake函数执行顺序优先于Start函数。
  2. 在Awake中找到其他GameObject是安全的。

5. FixedUpdate

FixedUpdate()该函数具有物理系统的频率,帧率刷新的间隔时间是固定的,如果不调节系统设置的情况下,默认每0.02秒刷新一次帧率,每秒执行50该方法。通常有关物理操作都应该在此方法里,如角色移动操作。因为该函数不受不同计算机的帧率影响。默认每秒都是执行50次。

/// <summary>
/// 该方法具有物理系统的频率,帧率刷新的间隔时间是固定的,如果不调节系统设置的情况下,默认每0.02秒刷新一次帧率,每秒执行50该方法。
/// 通常有关物理操作都应该在此方法里,如 Rigidbody操作
/// </summary>
void FixedUpdate()
{
}

事件触发函数

1. OnCollisionEnter

当前GameObject的(刚体/碰撞体)与另一个GameObject(刚体/碰撞体)发生接触时执行该函数。

/// <summary>
/// 当该碰撞体/刚体已开始接触另一个刚体/碰撞体时，调用 OnCollisionEnter。
/// </summary>
/// <param name="collision">与该碰撞发生接触的Collision 数据。</param>
private void OnCollisionEnter(Collision collision)
{

}

2. OnCollisionEnter2D

当前GameObject的(刚体/碰撞体)与另一个GameObject(刚体/碰撞体)发生接触时执行该函数**(仅限2D物理系统)**。

3. OnCollisionExit

当前GameObject碰撞体离开其他GameObject碰撞体时执行该函数。与OnCollisionEnter相反。

/// <param name="other">解除碰撞的碰撞体信息</param>
private void OnCollisionExit(Collision other)
{

}

4. OnCollisionExit2D

当前GameObject碰撞体离开其他GameObject碰撞体时执行该函数。与OnCollisionEnter2D相反。**(仅限2D物理系统)**。

5. OnCollisionStay

当前GameObject与其他GameObject发生碰撞过程中,每一帧都会执行该函数。

6. OnCollisionStay2D

与OnCollisionStay一样。**(仅限2D物理系统)**。

7. OnTriggerEnter

当前GameObject与另一个被标记为Is Trigger的GameObject碰撞时会执行该函数。

/// <param name="other">碰撞到的碰撞体</param>
private void OnTriggerEnter(Collider other)
{

}

8. OnTriggerEnter2D

当前GameObject与另一个被标记为Is Trigger的GameObject碰撞时会执行该函数。**(仅限2D物理系统)**。

9. OnTriggerExit

当前GameObject与另一个被标记为Is Trigger的GameObject停止碰撞时会执行该函数。

10.OnTriggerExit2D

当前GameObject与另一个被标记为Is Trigger的GameObject停止碰撞时会执行该函数。**(仅限2D物理系统)**。

11.OnEnable

该函数在当前GameObject变为启用和激活状态时调用。

12.OnDisable

该函数在当前GameObject变为禁用状态时调用。当对象销毁时也会调用该函数，它可用于任何清理代码。 当编译完成后重新加载脚本时，将调用 OnDisable，并在加载脚本后调用 OnEnable。

13.OnMouseDown

当鼠标点击当前GameObject的碰撞体范围内后执行该函数。

14.OnMouseUp

当鼠标点击当前GameObject的碰撞体范围内后,松开鼠标时执行该函数。

15.OnMouseUpAsButton

鼠标点击和松开时都处于当前GameObject碰撞体范围内时执行该函数。

16.OnMouseDrag

当鼠标点击当前GameObject的碰撞体范围内后并且长按不放的情况下,每帧执行一次该函数。

17.OnMouseEnter

当鼠标移入当前GameObject的碰撞体范围内执行一次该函数。

18.OnMouseExit

当鼠标移出当前GameObject的碰撞体范围内执行一次该函数。

19.OnMouseOver

当鼠标悬停在当前GameObject碰撞体范围内时,每帧执行执行一次该函数。
