1. 什么是线程?

我们通常使用的一些APP、PC端程序等等,其实都是一种进程,而进程由线程所组成,所以线程也被称为轻量级进程。是程序执行流的最小单元,线程自己并不拥有系统资源,只拥有一点运行区间所需要的资源。在程序中同时运行多个线程完成不同的工作,称为多线程。

2. 线程基础

1. 线程状态

  • New

    创建线程后但未启动的状态

    new Thread();

  • Runnable

    调用了start()后的就绪状态,等待CPU分配时间片。

    new Thread().start();

    当线程从睡眠状态被唤醒后,也会进入就绪状态。

  • Blocked

    运行中的线程，因为某些操作被阻塞而挂起;

    如抢夺锁时,由于锁一直被其他线程持有,当前线程就会进入Blocked状态。如下所示

    启用t1去抢夺锁,延迟一秒后再让t2去获取锁,这个时候t2肯定会进入Blocked状态,直到t1释放锁。

    public static void main(String[] args) throws InterruptedException {
  new Thread(()->{
    synchronized (lock){
      while (true){ }
    }
  },"t1").start();
  TimeUnit.SECONDS.sleep(1);
  new Thread(()->{
    synchronized (lock){ }
  },"t2").start();
}

  • Waiting

    运行中的线程，因为某些操作在等待中;

    比如在同步代码块中调用了wait()方法,则会让出执行权,进入waiting状态。

    public static void main(String[] args) throws InterruptedException {
  new Thread(()->{
    synchronized (lock){
      lock.wait();
    }
  },"t1").start();
}

  • Timed Waiting

    具有等待时间的等待状态。与waiting状态不同的是,该状态能够自主唤醒或者放弃。

    如线程调用了sleep(1000)方法。

    如线程调用了Thread.join(60000)。

    如线程调用了wait(60000)都会进入TimedWaiting状态。

    public static void main(String[] args) throws InterruptedException {
  Thread.sleep(60000);
}

  • Terminated

    线程终止状态。执行完run()方法后。

2. 线程的构造

通常我们创建线程new Thread(),这个过程发生了什么？


/**
 * Allocates a new {@code Thread} object. This constructor has the same
 * effect as {@linkplain #Thread(ThreadGroup,Runnable,String) Thread}
 * {@code (null, null, gname)}, where {@code gname} is a newly generated
 * name. Automatically generated names are of the form
 * {@code "Thread-"+}<i>n</i>, where <i>n</i> is an integer.
 */
public Thread() {
    init(null, null, "Thread-" + nextThreadNum(), 0);
}

我们看到Thread的无参构造会执行一个init()方法,通过nextThreadNum()方法获取一个序号作为线程名称序号。

/**
初始化一个线程。
参数：
g – 线程组
target – 调用其 run() 方法的对象
name – 新线程的名称
stackSize – 新线程所需的堆栈大小，或为零表示忽略此参数。
acc – 要继承的 AccessControlContext，如果为 null，则为 AccessController.getContext()
inheritThreadLocals – 如果为true ，则从构造线程继承可继承线程局部变量的初始值
     */
private void init(ThreadGroup g, Runnable target, String name,
                  long stackSize, AccessControlContext acc,
                  boolean inheritThreadLocals) {
  if (name == null) {
    //线程名非空处理
    throw new NullPointerException("name cannot be null");
  }
  //赋值线程名
  this.name = name;
  //获取当前线程
  Thread parent = currentThread();
  SecurityManager security = System.getSecurityManager();
  //如果没有指定线程组,则用当前创建线程的线程组
  if (g == null) {
    if (security != null) {
      g = security.getThreadGroup();
    }
    if (g == null) {
      g = parent.getThreadGroup();
    }
  }

  //确定当前运行的线程是否有权修改此线程组。
  g.checkAccess();

  if (security != null) {
    if (isCCLOverridden(getClass())) {
      security.checkPermission(SUBCLASS_IMPLEMENTATION_PERMISSION);
    }
  }
  g.addUnstarted();

  //赋值属性,线程组、是否是守护线程、类加载器、栈大小
  this.group = g;
  this.daemon = parent.isDaemon();
  this.priority = parent.getPriority();
  if (security == null || isCCLOverridden(parent.getClass()))
    this.contextClassLoader = parent.getContextClassLoader();
  else
    this.contextClassLoader = parent.contextClassLoader;
  this.inheritedAccessControlContext =
    acc != null ? acc : AccessController.getContext();
  //赋值线程的runnable接口
  this.target = target;
  setPriority(priority);
  if (inheritThreadLocals && parent.inheritableThreadLocals != null)
    this.inheritableThreadLocals =
    ThreadLocal.createInheritedMap(parent.inheritableThreadLocals);
  /* Stash the specified stack size in case the VM cares */
  this.stackSize = stackSize;

  //生成一个线程ID，并赋值
  tid = nextThreadID();
}

3. 守护线程

Java中的线程默认都是非守护线程。如果是守护线程创建的子线程那么默认就是守护线程。也可以手动设置为守护线程。所有的守护线程都会随着主线程结束而销毁,无论守护线程是否执行完毕。注意:线程开始执行后,就无法改变线程类型。

Thread thread = new Thread();
thread.setDaemon(true);//设置该线程为守护线程
thread.isDaemon();//线程是否是守护线程

守护线程有哪些使用场景呢? 比如发送连接心跳包案例,这种连接机制很常见,很多中间件平台都会选用这种方式。而且无需担心任务线程断开连接后,子线程的状态管理。

5. Join方法

线程的join()方法调用后会一直阻塞等待该线程结束或者中断。

如下所示, main线程会等待service线程执行完后再执行后续操作。

public static void main(String[] args) throws InterruptedException {
  Thread service = new Thread(() -> {
    //TODO write data to db
    System.out.println("do execute service.");
    TimeUnit.SECONDS.sleep(100);
  });
  service.start();
  //阻塞等待 service线程的结束或者中断
  service.join();

  //TODO start main execute
  System.out.println("main start.");
}
do execute service.

线程调用join()方法后会进入Waiting状态,如果指定了超时时间join(6000)则会进入TimedWaiting状态。

//一直阻塞等待service线程结束或service线程中断后
service.join();

//阻塞等待service线程5秒
service.join(5000);

//阻塞等待service线程 5000毫秒 + 60000纳秒
service.join(5000,60000);

6. interrupt方法

线程结束/中断方法,通过该方法可以将一个线程中断。注意: 子线程必须捕获 InterruptedException异常,否则无法收到中断信号。

public static void main(String[] args) throws InterruptedException {
  //开启一个子线程
  Thread t = new Thread(()->{
    try {
      TimeUnit.SECONDS.sleep(10000);
    }catch (InterruptedException e){
      System.out.println("捕获到中断信号,结束子线程。");
      e.printStackTrace();
    }
  });
  t.start();
  System.out.println(t.isInterrupted());//当前线程是否被中断

  //子线程执行3秒后 发出中断信号
  TimeUnit.SECONDS.sleep(3);
  //中断
  t.interrupt();
  System.out.println(t.isInterrupted());
}

3. 线程同步

1. synchronized

synchronized关键字是Java提供的一种同步方式,通常我们称之为锁。可以保证某一时间某一段代码,只能有一个线程所执行。该关键字可以用在三种不同场景：修饰方法、修饰静态方法、修饰代码块,三种方式各自锁定的机制有所不同。

在之前介绍线程状态时,也有用过该关键字,接下来我们再简单使用,并且从多方面观察synchronized到底是何方神圣。

案例: 启动多个线程执行执行同步代码。

public class Main {
  //声明一个monitor对象,用来被synchronized锁定
  private final static Object LOCK = new Object();

  public static void main(String[] args) {
    Runnable task = ()->{
      synchronized (LOCK){
          Thread.sleep(100000);
      }
    };
    //启动多个线程
    new Thread(task,"t1").start();
    new Thread(task,"t2").start();
    new Thread(task,"t3").start();
  }
}

程序启动后会进入阻塞,接下来打开jconsole与jstack查看线程信息。

名称: t1
状态: TIMED_WAITING
总阻止数: 0, 总等待数: 1
名称: t2

总阻止数: 1, 总等待数: 0
名称: t3
状态: java.lang.Object@62344d98上的BLOCKED, 拥有者: t1
总阻止数: 1, 总等待数: 0

从jconsole可以看到,是线程t1抢到了LOCK对象的锁。而另外两个线程则进入到了Blocked状态。

我们再从jstack验证一下。

"t1" #11 prio=5 os_prio=31 tid=0x00007fef070a2800 nid=0x5fab waiting on condition [0x0000000306d56000]
   java.lang.Thread.State: TIMED_WAITING (sleeping)
"t2" #12 prio=5 os_prio=31 tid=0x00007fef070a3800 nid=0x9d03 waiting for monitor entry [0x0000000306e59000]

在jstack中显示,t1线程为正常运行(这里处于timedWaiting是我们主动睡眠),而 另外两个线程显示waiting for monitor entry,表示处于Blocked状态,等待抢夺 object的monitor。

synchronized是有CPU指令monitorentry与monitorexit实现。

synchronized不同使用区别

同步方法

public synchronized void doExecute() {
}

synchronized修饰普通方法时,锁定的是this,这里的this指的是值方法所属的实例本身。如下

User user = new User();
user.doExecute();

锁定的是user该实例,不同的实例之间调用同步方法不会进行冲突。

同步静态方法

public synchronized static void doExecute() {
}

synchronized修饰静态方法,锁定的是this,只不过这个this可就不是实例了,因为静态方法是属于类方法,是全局性的。也就是锁定的就是本类的Class。

User.doExecute();

同步的静态方法,同一时间只会有一个线程所执行。

同步代码块

该方式比较简单,直接锁定synchronized（）指定的对象。

public  void doExecute() {
  synchronized (this){
  }
}

该方式与 同步普通方法一致,锁定的是实例。

static class User {
  public static void doExecute() {
    synchronized (User.class) {

    }
  }
}

该方式与 同步静态方法一致,锁定的是本类。

2. 死锁排查

首先编写一个案例,让程序进入死锁。

public class Main {
    private final static Object lock1 = new Object();
    private final static Object lock2 = new Object();
    public static void main(String[] args) {
        new Thread(()->{
            synchronized (lock1){
                TimeUnit.SECONDS.sleep(1);
                synchronized (lock2){

                }
            }
        },"t1").start();
        new Thread(()->{
            synchronized (lock2){
                TimeUnit.SECONDS.sleep(1);
                synchronized (lock1){

                }
            }
        },"t2").start();
    }
}

3. wait与notify

wait方法与notify方法本身与线程是无关的,它们是属于Object的方法,也就是任意类都会持有的方法,那么它们究竟会产生什么样的作用呢？注意: 这两个方法只能在同步方法或同步代码块才能使用。

执行一个对象的wait()方法会使当前线程进入waiting等待状态,直到另一个线程执行了此对象的notify()或者notifyAll()方法。会将当前线程放入该对象的monitor中的waitSet里面,当然这只是个概念说法。该方法存在虚假唤醒情况,应该使用在循环中使用。

synchronized (obj) {
  while (<condition does not hold>)
    obj.wait();
  ... // Perform action appropriate to condition
}

而执行一个对象的notify()方法会在当前对象monitor中的waitSet内随机释放一个线程,被称之为唤醒。注意,它只能释放对象自己waitSet中的等待线程哦。notifyAll()是唤醒waitSet中的所有等待线程。被唤醒的线程将会进入Runnable状态,然后等待CPU的执行。
