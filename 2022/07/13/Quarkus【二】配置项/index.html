Quarkus【二】配置项

发表于2022-07-13|更新于2022-07-13|Quarkus
|字数总计:1k|阅读时长:4分钟|阅读量:

前言

本章讲述Quarkus配置相关

1.配置文件格式

项目初始支持properties格式,同样也支持yaml格式,但需要导入相关依赖.

<!--quarkus 支持yml配置文件格式-->
<dependency>
  <groupId>io.quarkus</groupId>
  <artifactId>quarkus-config-yaml</artifactId>
</dependency>

导入该依赖后,yml优先级高于properties配置文件,建议不要混用,选择自己喜欢的一种即可。

在配置文件中定义如下测试数据:

zero:
  username: root
  password: admin
  age: 18
  hobby:
    - 乒乓球
    - 散步
  account:
    alipay: 支付宝账户
    wechat: 微信账户

2.配置属性读取-注解式

Quarkus通过@ConfigProperty注解读取单个配置属性,类似SpringBoot中的@Value。

@ConfigProperty(name = "zero.username")
String username;

@ConfigProperty(name = "zero.password",defaultValue = "default_password")
String password;

如果没有配置该属性,则使用defaultValue作为默认值。如果没有指定defaultValue并且配置文件没有配置该属性,则启动时会报错``Failed to load config value of type class java.lang.String for: zeus.messageFailed to load config value of type class java.lang.String for: zero.username`。

@ConfigProperty(name = "zero.age")
Optional<Integer> age;

可选属性-使用Optional包装 如果没有配置则会注入Optional.empty()。

// 以List列表读取
@ConfigProperty(name = "zero.hobby")
List<String> hobby;

// 以Map结构读取
@ConfigProperty(name = "zero.account")
Map<String, String> account;

3.配置属性读取-编程式

Quarkus也提供了编程式读取方案,通过代码API动态获取配置属性。

如下:

// 读取配置中zero.username属性值,String类型
String username = ConfigProvider.getConfig().getValue("zero.username", String.class);

// 可选式读取配置属性,可以不配置
Optional<String> optional = ConfigProvider.getConfig().getOptionalValue("password", String.class);

// 读取数组结构配置
List<String> hobby = ConfigProvider.getConfig().getValues("zero.hobby", String.class);
Optional<List<String>> optional = ConfigProvider.getConfig().getOptionalValues("zero.hobby", String.class);
// 获取配置属性相关信息
ConfigValue configValue = ConfigProvider.getConfig().getConfigValue("zero.username");

4.配置属性绑定实体

在SpringBoot中,我们会写大量的xxxproperties类,用来绑定配置文件中的属性。在Quarkus中一样可以做到.

通过@ConfigMapping注解。

@ConfigMapping(prefix = "zero",namingStrategy = ConfigMapping.NamingStrategy.SNAKE_CASE)
@StaticInitSafe
public interface ZeroProperties {
  @WithName("username")
  @WithDefault("defaultUserNameValue")
  String username();
  Optional<String> password();
  List<String> hobby();
  Map<String, String> account();
}

在新版中移除了@ConfigProperties并且新增了@ConfigMapping,新的注解只能用在接口上。

prefix表示属性的前缀,namingStrategy表示命名规则。

Config通常是首先创建的东西之一。在某些情况下，可能无法正确初始化配置映射。@StaticInitSafe来将映射标记为安全，以便在此阶段使用

@WithName配置属性与方法名称不一致时,指定配置属性名称。

@WithDefault指定默认值。

5.嵌套配置

@ConfigMapping(prefix = "zero", namingStrategy = ConfigMapping.NamingStrategy.SNAKE_CASE)
@StaticInitSafe
public interface ZeroProperties {
    String username();
    Optional<String> password();
    List<String> hobby();
    Map<String, String> account();
  
  	//嵌套接口配置
		Zeus zeus();
    interface Zeus{
        boolean enabled();
    }
}
zero:
  username: root
  hobby:
    - 乒乓球
    - 散步
  account:
    alipay: 支付宝账户
    wechat: 微信账户
  zeus:
    enabled: true

如果配置文件与接口中的方法名称不一致,则通过@WithName注解指定名称。

@WithName("other")
Zeus zeus();
other:
  enabled: true

6.转换器

用来将配置属性值,映射时转换成另一种数据类型,如下：将String类型转换为LocalDateTime。

定义转换器

public class LocalDateTimeConvert implements Converter<LocalDateTime> {
    private final String FORMAT = "yyyy-MM-dd HH:mm:ss";

    @Override
    public LocalDateTime convert(String value) throws IllegalArgumentException, NullPointerException {
        if (value.isBlank()){
            throw new NullPointerException();
        }
        return LocalDateTime.parse(value,DateTimeFormatter.ofPattern(FORMAT));
    }
}

使用转换器

@WithName("time")
@WithConverter(LocalDateTimeConvert.class)
LocalDateTime successTime();

配置属性

time: 2022-07-13 10:28:00

7.属性校验

配置映射可以结合Bean Validation的注解来验证配置值.

导入相关依赖:

<!-- 数据校验-->
<dependency>
  <groupId>io.quarkus</groupId>
  <artifactId>quarkus-hibernate-validator</artifactId>
</dependency>
@Min(10)
@Max(70)
Integer age();

@Email(message = "Email 不符合邮箱规则~")
String email();
