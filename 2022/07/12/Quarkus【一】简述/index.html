Quarkus【一】简述

发表于2022-07-12|更新于2022-07-13|Quarkus
|字数总计:715|阅读时长:2分钟|阅读量:

前言

Quarkus 是一个为 Java 虚拟机（JVM）和原生编译而设计的全堆栈 Kubernetes 原生 Java 框架，用于专门针对容器优化 Java，并使其成为无服务器、云和 Kubernetes 环境的高效平台。与传统的Java项目而言,如SpringBoot,前者则更轻量级

几大特性:

  • 及其适合云原生开发
  • 统一了命令式与声明式编程
  • 生态极好,整合了大量库及中间件

话不多说开卷~

官方网站 这里有大量的学习资源,不要错过哦~

1.安装

Mac系统安装:

1.安装SdkMan

curl -s "https://get.sdkman.io" | bash

2.使用Sdkman安装Quarkus

sdk install quarkus

查看安装的版本

quarkus --version
2.10.2.Final

3.更新版本

sdk upgrade quarkus

2.命令式创建项目

创建Quarkus项目的最简单方法是打开命令行并运行以下命令：

mvn io.quarkus:quarkus-maven-plugin:1.3.1.Final:create \
    -DprojectGroupId=com.zero \ #groupid
    -DprojectArtifactId=getting-started \ # artifactid且是项目文件名称
    -DclassName="com.zero.getting.started.GreetingResource" \ #默认生成的类文件名称
    -Dpath="/hello"	#提供的默认rest服务接口path

进入项目并且运行

cd getting-started
./mvnw compile quarkus:dev

初始项目启动后默认开启http服务端口8080。

curl http://127.0.0.1:8080/hello

返回”hello”表示项目启动完成。quarkus:dev 在开发模式下运行 Quarkus 。开发模式会启用后台编译的热部署，这意味着当您修改Java文件或资源文件并刷新浏览器时，这些更改将自动生效。这也适用于配置文件之类的资源文件。

3.项目结构

.
├── README.md
├── mvnw
├── mvnw.cmd
├── pom.xml
└── src
    ├── main
    │   ├── docker
    │   │   ├── Dockerfile.jvm
    │   │   └── Dockerfile.native
    │   ├── java
    │   │   
    │   └── resources
    │       ├── META-INF
    │       │   └── resources
    │       │       └── index.html
    │       └── application.properties
    └── test

从目录结构上来看,与SpringBoot项目及其类似,不同的是它提供了不同模式Dockerfile文件。

默认导入的相关依赖

<dependencies>
  <!--Web相关-->
  <dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-resteasy</artifactId>
  </dependency>
  <!--单元测试-->
  <dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-junit5</artifactId>
    <scope>test</scope>
  </dependency>
  <dependency>
    <groupId>io.rest-assured</groupId>
    <artifactId>rest-assured</artifactId>
    <scope>test</scope>
  </dependency>
</dependencies>

4.打包部署

在项目根目录通过命令进行打包

./mvnw package

打包后会在target文件夹中生成两个jar文件

  • getting-started-1.0-SNAPSHOT.jar-仅包含项目的类和资源。
  • getting-started-1.0-SNAPSHOT-runner.jar 可执行的jar文件。

注意: 这不是一种将所有依赖打包成一个jar的方式,依赖处于target/lib文件夹下,所以如果要想移动程序则需要复制runner.jar和lib目录。

然后就可以通过启动java程序方式启动该项目。

java -jar getting-started-1.0-SNAPSHOT-runner.jar
