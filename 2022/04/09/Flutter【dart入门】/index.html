简介

Dart 在静态语法方面和 Java 非常相似，如类型定义、函数声明、泛型等，而在动态特性方面又和 JavaScript 很像，如函数式特性、异步支持等。除了融合 Java 和 JavaScript 语言之所长之外，Dart 也具有一些其它具有表现力的语法，如可选命名参数、..（级联运算符）和?.（条件成员访问运算符）以及??（判空赋值运算符）。其实，对编程语言了解比较多的读者会发现，在 Dart 中其实看到的不仅有 Java 和 JavaScript 的影子，它还具有其它编程语言中的身影，如命名参数在 Objective-C 和 Swift 中早就很普遍，而??操作符在PHP 7.0 语法中就已经存在了，因此我们可以看到 Google 对 Dart 语言给予厚望，是想把 Dart 打造成一门集百家之所长的编程语言。

3. 常量

如果您从未打算更改一个变量，那么使用 final 或 const，不是var，也不是一个类型。 一个 final 变量只能被设置一次，两者区别在于：const 变量是一个编译时常量（编译时直接替换为常量值），final变量在第一次使用时被初始化。被final或者const修饰的变量，变量类型可以省略，如

final name = 'Bob'; // Without a type annotation
final String nickname = 'Bobby';

const 关键字不仅仅可以用来定义常量，还可以用来创建 常量值，该常量值可以赋予给任何变量。你也可以将构造函数声明为 const 的，这种类型的构造函数创建的对象是不可改变的。

var name = const {
  "name": "张三"
};
var ages = const [1,2,3];

name = {
  "address": '北京'
};
ages = [4,5,6];

没有使用 final 或 const 修饰的变量的值是可以被更改的，即使这些变量之前引用过 const 的值。

const与final不同用法产生的作用

///使用const修饰的Set变量 【引用地址是无法再次改变并且值也无法改变】
const num1 = {1,2,3};
num1.add(4); //Cannot change an unmodifiable set
num1 = {}; //Error: Can't assign to the const variable 'num1'.

///使用 final修饰的Set变量【列表值可以变动,但是引用地址无法改变】
final num2 = {1,2,3};
num2.add(4); //可以添加新值
//num2 = {}; Error: Can't assign to the final variable 'num2'.

///使用 将const修饰的Set集合 赋予一个变量【集合元素无法改变,但是变量的引用可以改变】
var num3 = const{1,2,3};
// num3.add(4); Cannot change an unmodifiable set
num3 = {};//可以再次赋值

4. 内置类型

1. Numbers

  • int

    整数值；长度不超过 64 位，具体取值范围 [依赖于不同的平台](https://dart.cn/guides/language/numbers)。在 DartVM 上其取值位于 -263 至 263 - 1 之间。在 Web 上，整型数值代表着 JavaScript 的数字（64 位无小数浮点型），其允许的取值范围在 -253 至 253 - 1 之间。

  • double

    64 位的双精度浮点数字，且符合 IEEE 754 标准。

  • 数字与字符转换

///String to int 转换失败则为Null

///String to double 转换失败则为Null

2. String

Dart 字符串（String 对象）包含了 UTF-16 编码的字符序列。可以使用单引号或者双引号来创建字符串。

var s1 = 'Single quotes work well for string literals.';
var s2 = "Double quotes work just as well.";
var s3 = 'It\'s easy to escape the string delimiter.';

在字符串中,可以通过${表达式}方式使用表达式,如果表达式是一个标识符，可以省略掉 {}。如果表达式的结果为一个对象，则 Dart 会调用该对象的 toString 方法来获取一个字符串。

var name = "张三";
var age = 18;
print('hi 你好,我叫${name.trim()},今年$age岁');

== 运算符负责判断两个对象的内容是否一样，如果两个字符串包含一样的字符编码序列，则表示相等。

///字符串拼接
print("hi" + "你好!");

///多行字符
var lineStr = '''
   通过三个单引号,
   或者三个双引号,
   可以创建多行字符串
  ''';

6. Map

Map是键值对结构,其结构为哈希表由数组+链表组成。其中键和值可以使任何类型,但是官方都是推荐键使用String结构,每个键只能出现一次,值可以重复多次。

  • 使用扩展操作符...与...? 以及if for操作

5. 函数

1. 函数与方法

方法是与对象相关的,是属于对象呢,而函数于对象无关,类似于全局的概念。

Dart 是一种真正面向对象的语言，所以即便函数也是对象并且类型为 Function，这意味着函数可以被赋值给变量或者作为其它函数的参数。

定义一个函数例子:

String getName(){
  return "张三";
}
showName(String name){
  print(name);
}

虽然高效 Dart 指南建议在 公开的 API 上定义返回类型，不过即便不定义，该函数也依然有效：

/// 函数可以不定义返回值类型,但是也依然可以直接返回
getUsername(){
  return "admin";
}

如果函数只是单行表达式,可以简写为箭头函数

///函数简写
bool isEmpty(List<dynamic> list) => list.isEmpty;

2. 参数

函数可以有两种形式的参数: 必填参数和 可选参数。必填参数定义在参数列表前面, 可选参数则定义 必填参数后面。可选参数可以分为两种规则 命名规则和 顺序规则。

某些 API（特别是 Flutter 控件的构造器）只使用命名参数，即便参数是强制性的

  • 可选参数-命名规则

    定义函数

    ///定义可选命名参数函数
/// required表示为必填项(不推荐使用,因为普通参数本来就是必填的)
/// ? 表示允许为null,
void createUser({required String name,int? age}){

}

    调用该函数时,需要通过{}中的名称来入参

    createUser(name: "name");
createUser(name: "张三",age: 18);
createUser(name: "王五",age: null);

  • 可选参数-位置参数

    使用 [] 将一系列参数包裹起来作为位置参数：

    ///定义可选位置顺序参数
void createStudent(String name,int age,[bool? isMale,String? address]){

}
    createStudent("name", 18);
createStudent("name", 18,null);
createStudent("name", 18,null,null);
createStudent("name", 18,true,"北京");

    位置可选参数,再调用入参时,无需指定名称,直接按顺序入参即可,与普通参数一样。

    劣势就是假设我们定义了一个很长的 可选位置参数,现在我们要为最后一个参数入参,那么之前的参数都必须要入参才行。

  • 可选参数默认值

    定义函数时,可以通过=为可选参数指定默认值,默认值必须为编译时常量。没有默认值则为null。

    void createModel({String? name = "张三",int? age = 18}){

}
void createModel2([String? name = "张三",bool? sex = false]){

}

    复杂类型的参数同样也可以指定默认值,如List、Map。

    void forEach({List<int>? list = const [1,2,3],
            Map<String,Object>? map = const{"name": "张三"}}){

}

3. 函数一等公民

可以将函数作为参数传递给另一个函数。例如：

void main(){
  var list = [1,2,3];
  list.forEach(printElement);
}
void printElement(int e){
  print(e);
}

熟悉Java的朋友对这种写法肯定不陌生,那么为什么forEach方法能够放入我们自定义的函数呢,我们来看下它的源码

/// Invokes [action] on each element of this iterable in iteration order.
void forEach(void action(E element)) {
  for (E element in this) action(element);
}

可以看到,forEach的入参为一个函数,并且这个函数 没有返回值且只有一个入参,很明显和我们自定义的printElement函数结构是一模一样的,这就是原因所在。

写到这里,不禁感慨,Dart真的借鉴了Java很多很多特性,如函数式编程、Stream等等。

函数也可以赋值给一个变量,比如:

var func1 = ()=>{
  print("object")
};
Function func = (message)=>{
  print(message)
};

func1();
func("Hello");

4. 匿名函数

大多数方法都是有名字的，比如 main() 或 printElement()。你可以创建一个没有名字的方法，称之为 匿名函数、 Lambda 表达式 或 Closure 闭包。你可以将匿名方法赋值给一个变量然后使用它，比如将该变量添加到集合或从中删除。

Iterable中大量API都使用了匿名函数 如

list.forEach((element) {
  print(element);
});

如果匿名函数只有一行语句,可以简写成Lambda表达式

list.forEach((element) => print(element));

5. 闭包

闭包 即一个函数对象，即使函数对象的调用在它原始作用域之外，依然能够访问在它词法作用域内的变量。

函数可以封闭定义到它作用域内的变量。接下来的示例中，函数 makeAdder() 捕获了变量 addBy。无论函数在什么时候返回，它都可以使用捕获的 addBy 变量。

Function makeAdder(int addBy){
  return (int i) => addBy + i;
}

void main(){
  var adderFunc = makeAdder(3); //调用函数,返回一个函数
  var sum = adderFunc(2);		//执行返回的函数

}

9. 异常

Dart 代码可以抛出和捕获异常。异常表示一些未知的错误情况，如果异常没有捕获则会被抛出从而导致抛出异常的代码终止执行。

与 Java 不同的是，Dart 的所有异常都是非必检异常，方法不必声明会抛出哪些异常，并且你也不必捕获任何异常。

Dart 提供了 Exception 和 Error 两种类型的异常以及它们一系列的子类，你也可以定义自己的异常类型。但是在 Dart 中可以将任何非 null 对象作为异常抛出而不局限于 Exception 或 Error 类型。

示例:

void Exception(){
  throw FormatException("抛出异常");
}
void Exception(){
  throw "发送错误!";
}

捕获异常

void main(){
  try{
    Exception();
  }on FormatException catch(e){
    print('捕获到了异常 :${e.message}');
  }
}

对于可以抛出多种异常类型的代码，也可以指定多个 catch 语句，每个语句分别对应一个异常类型，如果 catch 语句没有指定异常类型则表示可以捕获任意异常类型：

try {
  breedMoreLlamas();
} on OutOfLlamasException {
  // A specific exception
  buyMoreLlamas();
} on Exception catch (e) {
  // Anything else that is an exception
  print('Unknown exception: $e');
} catch (e) {
  // No specified type, handles all
  print('Something really unknown: $e');
}

如上述代码所示可以使用 on 或 catch 来捕获异常，使用 on 来指定异常类型，使用 catch 来捕获异常对象，两者可同时使用。

你可以为 catch 方法指定两个参数，第一个参数为抛出的异常对象，第二个参数为栈信息 StackTrace 对象：

void main(){
  try{
    Exception();
  }on FormatException catch(e,s){
    print('捕获到了异常 ${e.runtimeType} 异常消息:${e.message}');
  }
}

关键字 rethrow 可以将捕获的异常再次抛出：

void main(){
  try{
    Exception();
  }on FormatException catch(e,s){
    print('捕获到了异常 ${e.runtimeType} 异常消息:${e.message}');
    rethrow; //再次将异常抛出去
  }
}

无论是否抛出异常，finally 语句始终执行，如果没有指定 catch 语句来捕获异常，则异常会在执行完 finally 语句后抛出

try{
  Exception();
}on FormatException catch(e,s){
  print('捕获到了异常 ${e.runtimeType} 异常消息:${e.message}');
  rethrow; //再次将异常抛出去
}finally{
  print("释放资源");
}

对于大多数编程语言来说在构造函数中为实例变量赋值的过程都是类似的，而 Dart 则提供了一种特殊的语法糖来简化该步骤：

User(this.username,this.password);

如果你没有声明构造函数，那么 Dart 会自动生成一个无参数的构造函数并且该构造函数会调用其父类的无参数构造方法。

dart中没有重载的概念,但是构造函数可以有多个,通过不同命名来表达更明确的意图。

///构造函数 自选入参
User.Build({String? username,String? password})
  :username = username,
password = password;

///构造函数 入参为 username
User.BuildByUserName(String username)
  : username = username;

///构造函数 入参为password
User.BuildByPasswd({String? password})
  : password = password;

记住构造函数是不能被继承的，这将意味着子类不能继承父类的命名式构造函数，如果你想在子类中提供一个与父类命名构造函数名字一样的命名构造函数，则需要在子类中显式地声明。

默认情况下，子类的构造函数会调用父类的匿名无参数构造方法，并且该调用会在子类构造函数的函数体代码执行前，如果子类构造函数还有一个 初始化列表，那么该初始化列表会在调用父类的该构造函数之前被执行，总的来说，这三者的调用顺序如下：

  1. 初始化列表
  2. 父类的无参数构造函数
  3. 当前类的构造函数

如果父类没有匿名无参数构造函数，那么子类必须调用父类的其中一个构造函数，为子类的构造函数指定一个父类的构造函数只需在构造函数体前使用（:）指定。

如下案例

///创建 Person类
///当一个类只定义了有参构造函数后,没有显示定义无参构造,就表示该类没有无参构造
class Person{
  String? firstName;
  ///自定义构造函数
  Person.fromJson(Map data){
    print('execute Person_fromJson constructor');
  }
}

///创建 Employee类 继承自Person,默认情况下,子类会默认调用父类的无参构造
///但是父类如果没有无参构造的情况下,子类必须显示调用父类的带参构造
class Employee extends Person{
  ///在构造函数后 通过 `:super.()`调用父类的构造函数
  Employee.Build(Map data): super.fromJson(data){
    print('execute Employ_Build constructor');
  }
}

void main(){
  var employee = Employee.Build({});
}

execute Employ_Build constructor

工厂构造函数

使用 factory 关键字标识类的构造函数将会令该构造函数变为工厂构造函数，这将意味着使用该构造函数构造类的实例时并非总是会返回新的实例对象。例如，工厂构造函数可能会从缓存中返回一个实例，或者返回一个子类型的实例。

如下案例:

/// 工厂构造函数案例
/// 根据不同的业务模块,实例不同的Logger对象
class Logger{
  final String serviceName;
  bool? mute = false;

  ///缓存池,存放Logger的实例对象
  static final Map<String,Logger> _instanceCache = <String,Logger>{};

  ///构造函数,传入业务模块名称,生成对应的实例对象
  Logger._init(this.serviceName);

  ///工厂构造函数,根据不同的业务模块名称,从缓存池内取出对应的实例对象
  ///如果没有,就生成新的实例对象,并且放入缓存池再返回
  factory Logger(String serviceName){
    return _instanceCache.putIfAbsent(serviceName, () => Logger._init(serviceName));
  }
  factory Logger.fromJson(Map<String,Object> map){
    return Logger(map['serviceName'].toString());
  }
}

工厂构造函数的调用方式与普通构造函数一样

var userLog = Logger("user");
var authLog = Logger("auth");
var monitorLog = Logger.fromJson({'serviceName':"monitor"});

14. 空安全

在很多语言中,其实都没有空安全这个概念,如Java,任何类型都可以是null,这种情况下就需要大量的非空判断,但是在dart中,是不需要的,因为在高版本dart中,null是不允许赋值到任何类型上,如下代码

void main(){
  printNameSize("name");//编译通过
  printNameSize(null);//编译不通过
}

printNameSize(String name){
  print("${name.length}");
}

在编译阶段,向方法内入参null是无法通过的.那么如果我们需要一个可空参数如何做到,其实很简单,在此之前我们也使用到多次,就是通过?标识符来表示该对象是允许为空。

printNameSize(String? name){
  print("${name.length}"); //编译不通过
}

但是,如果参数是?可空的,那么使用该参数的属性与方法时候,就必须要通过?或者!来使用

printNameSize(String? name){
  print("${name?.length}");
}

name?.length 表示只有name不是null的情况下,才会调用length属性,否则直接返回null。

printNameSize(String? name){
  print("${name!.length}");
}

name!.length表示如果name是null的情况下,直接抛出异常。

  • 如何已同步的方式,获取异步任务的结果,简单来说就是 任务是异步的,但我就想阻塞式的获取结果。
void main() async{
  //阻塞式等待 异步任务的结果
  var taskResult = await getUserNameAsync();
  print(taskResult);
  print('main is start!');
}

await表示同步阻塞,直到获取到任务结果,但是await关键字必须在异步方法中调用,也就是使用async修饰的方法中。

await 表达式的返回值通常是一个 Future 对象；如果不是的话也会自动将其包裹在一个 Future 对象里。 Future 对象代表一个“承诺”， await 表达式会阻塞直到需要的对象返回。
