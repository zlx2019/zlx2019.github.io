Java【Unsafe类】

发表于2022-04-11|更新于2022-04-12
|字数总计:1k|阅读时长:4分钟|阅读量:

前言

Unsafe是位于sun.misc包下的一个类，主要提供一些用于执行低级别、不安全操作的方法，如直接访问系统内存资源、自主管理内存资源等，这些方法在提升Java运行效率、增强Java语言底层资源操作能力方面起到了很大的作用。但由于Unsafe类使Java语言拥有了类似C语言指针一样操作内存空间的能力，这无疑也增加了程序发生相关指针问题的风险。在程序中过度、不正确使用Unsafe类会使得程序出错的概率变大，使得Java这种安全的语言变得不再“安全”，因此对Unsafe的使用一定要慎重。

1. Unsafe的获取

默认情况下,Unsafe是无法被使用的,因为官方认为这个类太过于危险。

public static void main(String[] args) {
  Unsafe unsafe = Unsafe.getUnsafe();
}
Exception in thread "main" java.lang.SecurityException: Unsafe

那么如何获取呢？可以通过反射来获取。这也侧面的看出反射的无孔不入,十分强大。

我们先看一下Unsafe源码

public final class Unsafe {

    private static native void registerNatives();
    static {
        registerNatives();
        sun.reflect.Reflection.registerMethodsToFilter(Unsafe.class, "getUnsafe");
    }

    private Unsafe() {}

    private static final Unsafe theUnsafe = new Unsafe();
  .....
}

可以看到,在Unsafe中有个名为theUnsafe的静态实例,我们可以通过反射获取到它。

/**
 * 通过反射获取到 Unsafe 实例
 */
private static Unsafe getUnsafe()  {
  try {
    Field theUnsafe = Unsafe.class.getDeclaredField("theUnsafe");
    theUnsafe.setAccessible(true);//强制获取私有属性
    Unsafe unsafe = (Unsafe) theUnsafe.get(null);//获取静态实例
    return unsafe;
  } catch (Exception e) {
    throw new RuntimeException("load Unsafe error!");
  }
}

2. Unsafe骚操作

1. 开天辟地

创建一个类

static class Count{
  private  int i;
  public Count(){
    i++;
    System.out.println("init Count");
  }
}

我们都知道,在Java中实例化一个类,都必须通过类的构造函数,也就是说实例化Count后 它的属性i必定是 1。

示例: 直接实例化

System.out.println(new Count().i);
//init Count
//1

示例: 通过反射

System.out.println(Count.class.newInstance().i);
//init Count
//1

结果和我们预期一样。这里需要提一下,通过ClassLoader加载类的情况下不会触发类的实例化。如下

Class<?> aClass = Class.forName("com.zero.java8.unsafe.Main$Count");
//不会触发实例化

//除非主动实例化
aClass.newInstance();
//init Count

那么如何能够实例化一个对象,并且跳过它的构造函数呢? 这种想法很危险,但也确实可以做到,没错 就是通过Unsafe就可以达到目的。

//Unsafe的allocateInstance(Class<T>) 方法,可以为一个类开辟内存空间,并且不会初始化
Unsafe unsafe = getUnsafe();
Count instance = (Count) unsafe.allocateInstance(Count.class);
System.out.println(instance.i);// 0

2. 逆天改命

通过Unsafe强制修改类的属性值

static class Service{
  private int code = 500;
  private boolean running(){
    return 200 == code;
  }

  public void worker(){
    if (running()){
      System.out.println("开始执行工作...");
    }
  }
}
Service service = new Service();
service.worker();

定义了一个任务类,直接实例化后调用worker方法,很明显该方法肯定不会执行。

接下来我们通过Unsafe修改code的值为200,然后再执行worker方法。

Unsafe unsafe = getUnsafe();
Service service = new Service();

//获取code Field
Field field = service.getClass().getDeclaredField("code");

//unsafe的objectFieldOffset(field)方法,可以获取该field在内存中的偏移量
//获取 code属性在内存中的偏移量
long codeOffset = unsafe.objectFieldOffset(field);

//unsafe.putInt(要操作的类实例,要操作的属性偏移量,新值);
//将code值改为200
unsafe.putInt(service,codeOffset,200);
service.worker();

3. 计算类所占字节

通过Unsafe可以计算出一个类大概占用的字节数

/**
  * 获取一个类的所占字节
  * @param obj   类
  * @return      字节
  */
private static long sizeByteOf(Object obj){
  Unsafe unsafe = getUnsafe();
  //获取类的所有非静态属性
  Set<Field> allFields = getAllFields(obj);

  long maxOffset = 0;
  for (Field field : allFields) {
    long offset = unsafe.objectFieldOffset(field);
    if (offset > maxOffset){
      maxOffset = offset;
    }
  }
  return ((maxOffset / 8) + 1) * 8;
}

/**
 * 获取一个类的所有属性 包括父类的属性(排除Object)
 * @return  属性集合
 */
private static Set<Field> getAllFields(Object obj){
  Class<?> clazz = obj.getClass();
  Set<Field> fieldSet = new HashSet<>();
  while (Object.class != clazz){
    Field[] fields = clazz.getDeclaredFields();
    Stream.of(fields)
      //排除 静态属性
      .filter(field -> !Modifier.isStatic(field.getModifiers()))
      .forEach(fieldSet::add);
    clazz = clazz.getSuperclass();
  }
  return fieldSet;
}
