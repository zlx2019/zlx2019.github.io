Spring Native初体验

1. 前言

Spring Native是Spring团队和GraalVM 团队合作的成果,可以将Spring应用通过 **AOT(Ahead-of-Time)**预先编译成本地可执行程序。

我们都知道,Java程序最终都是要运行在JVM上的,随着项目的扩大,项目逐渐复杂,每次通过JVM启动Java程序时都需要大量的时间来进行数据初始化、编译等,有的甚至要几分钟到上十几分钟。而通过AOT预先编译后,可以直接编译成可执行文件,直接就脱离了JVM之上,不需要安装JVM即可运行。从而获得快速启动、低内存消耗、即时峰值性能等特性，这样的特性在云原生时代显得尤为重要，但相应代价是编译构建时间更长。

2. 环境准备

SpringBoot3的前置要求是Java版本最低为17。

SpringNative则需要GraalVM发行的JDK才支持。

所以我们需要查看下自己的JDK是否达标:

% java -version
openjdk version "17.0.6" 2023-01-17
OpenJDK Runtime Environment GraalVM CE 22.3.1 (build 17.0.6+10-jvmci-22.3-b13)
OpenJDK 64-Bit Server VM GraalVM CE 22.3.1 (build 17.0.6+10-jvmci-22.3-b13, mixed mode, sharing)

如上显示,就可以进行开发了。

3. 创建项目并且打包

这里直接使用Idea的Initializr创建即可,版本依赖信息如下:

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <!--SpringBoot版本-->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.0.2</version>
        <relativePath/> <!-- lookup parent from repository -->
    </parent>
    <groupId>com.example</groupId>
    <artifactId>example-springboot3</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <name>example-springboot3</name>
    <description>example-springboot3</description>
    <properties>
        <java.version>17</java.version>
    </properties>
    <dependencies>
        <!--Web-->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
    </dependencies>
    <build>
        <plugins>
            <!-- Native打包插件,通过Idea打包时需要用到 -->
            <plugin>
                <groupId>org.graalvm.buildtools</groupId>
                <artifactId>native-maven-plugin</artifactId>
            </plugin>
            <!-- SpringBoot打包插件-->
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>

然后创建一个启动主类:

@SpringBootApplication
@RestController
public class ExampleSpringboot3Application {
    public static void main(String[] args) {
        SpringApplication.run(ExampleSpringboot3Application.class, args);
    }
    @GetMapping("/hello")
    public String hello(){
        return "Hello!";
    }
}

然后在项目目录进行终端,执行如下命令:

mvn -Pnative native:compile

编译的时间是比较长的,建议耐心等待…

[INFO] ------------------------------------------------------------------------
[INFO] BUILD SUCCESS
[INFO] ------------------------------------------------------------------------
[INFO] Total time:  01:27 min
[INFO] Finished at: 2023-02-01T18:53:20+08:00
[INFO] ------------------------------------------------------------------------

出现如上提示,表示打包完毕.

然后我们会在打包后的target目录中找到两个重要的文件: demo-0.0.1-SNAPSHOT.jar和 demo或者demo.exe。

jar文件不用多说,就是我们平常打包出的jar包,而另一个就是当前机器可直接执行的文件。

接下来我们直接运行该文件:

cd target 
./demo
