Spring【一】Bean的生命周期详解

Spring
Spring
发布日期:   2023-02-21
文章字数:   3k
阅读时长:   12 分

1. 什么是Bean

官方定义:

In Spring, the objects that form the backbone of your application and that are managed by the Spring IoC container are called beans. A bean is an object that is instantiated, assembled, and otherwise managed by a Spring IoC container. Otherwise, a bean is simply one of many objects in your application. Beans, and the dependencies among them, are reflected in the configuration metadata used by a container.

简单点说,一个Object交给SpringIOC容器实例化、组装和管理之后,在Spring中统称为Bean。

什么是Bean的生命周期?

Bean的生命周期就是SpringIOC对一个Bean从开始注入到容器中直至销毁的一个过程。

2. Bean的作用域

通常我们谈论的Bean的生命周期,一般是指Singleton作用域的Bean,也就是Spring默认的作用域的Bean。因为只有Singleton作用域的Bean才会全程被Spring所管理。

对于prototype的Bean,创建完后交给使用者后Spring将不再管理.

简单说一下Spring的作用域有哪些？

  • singleton

    表示全局只存在一个实例的单例Bean。Bean的默认作用域。

  • prototype

    表示每次获取时都会创建一个新的实例。

  • request

    每一次 HTTP 请求都会产生一个新的 bean，该 bean 仅在当前 HTTP request 内有效。

  • session

    每一次 HTTP 请求都会产生一个新的 bean，该 bean 仅在当前 HTTP session 内有效。

3. Bean的生命周期阶段

Bean的生命周期分别以下几个阶段:

实例化 > 属性设置 > 初始化 > 使用 > 销毁。

下面通过Spring提供的Bean生命周期钩子方法进行演示.

4. Bean的生命周期扩展

例如有如下Bean:

@Component
public class UserLife {

}

Bean必须需要使用@Componet标注.

实例化就是执行该类的构造方法.

public UserLife() {
  System.out.println("UserLife() -- 实例化");
}

0. BeanFactoryPostProcessor(实例化前)

该接口是容器级别的钩子函数,提供了一种在BeanFactory实例化之后但是在Bean实例化之前对象实例化的扩展点，它可以对BeanFactory中的Bean定义进行修改，添加配置，删除配置，更改bean的属性等。使用这个接口，可以改变Bean的定义，它可以改变容器实例化Bean的某些属性，也可以在Bean实例化之前添加一些额外的设置和操作，从而使程序更加灵活，满足不同的需求。

@Component
class CustomBeanFactoryProcessor implements BeanFactoryPostProcessor{
    /**
     * 在Spring容器加载了BeanDefinition之后，在Bean实例化之前执行的方法。
     * 可以通过该方法对BeanFactory进行设置和修改，它是容器的扩展点，可以用来实现一些自定义的逻辑。
     * @param beanFactory 应用程序上下文使用的 Bean 工厂
     */
    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) throws BeansException {
        System.out.println("所有Bean实例化之前 -- BeanFactoryPostProcessor -- postProcessBeanFactory()");
    }
}

运行项目:

所有Bean实例化之前 -- BeanFactoryPostProcessor -- postProcessBeanFactory()

1. InstantiationAwareBeanPostProcessor(实例化前后)

实现该接口,可以在每一个Bean的实例化前以及实例化后进行扩展。

《实例化前》 > 实例化 > 《实例化后》 > 属性设置 > 初始化 > 使用 > 销毁。

如专门对UserLifeBean进行扩展:

@Component
class UserLifeBeanInitProcess implements InstantiationAwareBeanPostProcessor{

    /**
     * 每一个Bean实例化之前回调的方法
     *
     * @param beanClass 当前实例化的Bean Class
     * @param beanName 当前实例化的BeanName
     * @return
     * @throws BeansException
     */
    @Override
    public Object postProcessBeforeInstantiation(Class<?> beanClass, String beanName) throws BeansException {
        if (beanClass.isAssignableFrom(UserLife.class) && beanName.equals("userLife")){
            System.out.println("UserLife Bean -- 实例化之前 -- InstantiationAwareBeanPostProcessor.postProcessBeforeInstantiation()");
        }
        return null;
    }

    /**
     * 每一个Bean实例化之后回调的方法
     * @param bean 已实例化完的Bean实例,但未设置属性
     * @param beanName 实例化完的Bean名称
     */
    @Override
    public boolean postProcessAfterInstantiation(Object bean, String beanName) throws BeansException {
        if (bean instanceof UserLife && beanName.equals("userLife")){
            System.out.println("UserLife Bean -- 实例化之后 -- InstantiationAwareBeanPostProcessor.postProcessAfterInstantiation()");
        }
        return true;
    }

    /**
     * 对Bean的每一个属性进行赋值之前回调
     * @param pvs 即将赋到属性上的值
     * @return 返回要赋值到 Bean 的实际属性值（可以是传入的 PropertyValues 实例），或者 null 跳过属性填充
     */
    @Override
    public PropertyValues postProcessProperties(PropertyValues pvs, Object bean, String beanName) throws BeansException {
        if (bean instanceof UserLife && beanName.equals("userLife")){
            System.out.println("UserLife Bean -- 对属性赋值之前 -- InstantiationAwareBeanPostProcessor.postProcessProperties()");
        }
        return pvs;
    }
}

运行项目后输出如下:

所有Bean实例化之前 -- BeanFactoryPostProcessor -- postProcessBeanFactory()
UserLife Bean -- 实例化之前 -- InstantiationAwareBeanPostProcessor.postProcessBeforeInstantiation()
UserLife() -- 实例化
UserLife Bean -- 实例化之后 -- InstantiationAwareBeanPostProcessor.postProcessAfterInstantiation()
UserLife Bean -- 对Bean的属性设置(赋值)  -- InstantiationAwareBeanPostProcessor.postProcessProperties()

2. BeanNameAware

Aware 类型的接口的作用就是让我们能够拿到 Spring 容器中的一些资源。基本都能够见名知意，Aware 之前的名字就是可以拿到什么资源，例如 BeanNameAware 可以拿到 BeanName，以此类推。调用时机需要注意：所有的 Aware 方法都是在初始化阶段之前调用的。

实例化前 > 实例化 > 实例化后 > 属性设置> 《BeanNameAware》 > 初始化 > 使用 > 销毁。

@Component
public class UserLife implements BeanNameAware {
    public String beanName;
    /**
     * 获取当前Bean的名称,此名称是IOC工厂中实际使用的名称,可能和自己指定的不同.
     * 实际的 Bean 名称可能已通过附加“#...”后缀。
     * 如果需要，请使用该方法 BeanFactoryUtils.originalBeanName(String) 提取原始 Bean 名称
     * @param name 当前Bean名称
     */
    @Override
    public void setBeanName(String name) {
	      System.out.println("UserLife Bean -- 获取到BeanName -- BeanNameAware.setBeanName()");
        this.beanName = name;
    }
}
@Autowired
private UserLife userLife;
@Test
void contextLoads(){
  System.out.println("userLife BeanName:" + userLife.beanName);
}

运行结果如下:

userLife BeanName:userLife

Aware的子接口非常之多,下面列举一些见名知意的子接口:

  • BeanNameAware

    获取Bean的Name接口

  • BeanClassLoaderAware

    实现这个接口的bean可以获得ClassLoader的引用，可以用来动态加载类和资源文件。

  • EnvironmentAware

    实现这个接口的bean可以获得Environment的引用，可以用来访问环境变量。

  • NotificationPublisherAware

    实现这个接口的bean可以获得NotificationPublisher的引用，可以用来发送通知。

  • NotificationListenerAware

    实现这个接口的bean可以获得NotificationListener的引用，可以用来接收通知。

  • 等等…

3. BeanFactoryAware

实现该接口,可以在Bean初始化之前获取到Bean的BeanFactory工厂。

实例化前 > 实例化 > 实例化后 > 属性设置 > 《BeanFactoryAware》 > 初始化 > 使用 > 销毁。

@Component
public class UserLife implements BeanFactoryAware {

    public UserLife() {
        System.out.println("UserLife() -- 实例化");
    }
    private  BeanFactory beanFactory;
    /**
     * 获取BeanFactory容器
     * 在Bean实例化完成后{@link InstantiationAwareBeanPostProcessor.postProcessAfterInstantiation}
     * AND
     * Bean初始化之前{@link InitializingBean.afterPropertiesSet()}时回调此方法。
     *
     * @param beanFactory Bean工厂容器
     */
    @Override
    public void setBeanFactory(BeanFactory beanFactory) throws BeansException {
      	System.out.println("UserLife Bean -- 获取到beanFactory -- BeanFactoryAware.setBeanFactory()");
        this.beanFactory = beanFactory;
    }
}

4. ApplicationContextAware

实现该接口,可以在到Bean填充完属性之后,以及执行Init()方法之前获取到ApplicationContext容器。

实例化前 > 实例化 > 实例化后 > 属性设置 > 《ApplicationContextAware》 > 初始化 > 使用 > 销毁。

@Component
public class UserLife implements BeanNameAware , BeanFactoryAware, ApplicationContextAware {

    public UserLife() {
        System.out.println("UserLife() -- 实例化");
    }
    private ApplicationContext applicationContext;
    /**
     *设置运行此对象的应用程序上下文。通常，此调用将用于初始化对象。
     *在填充正常 Bean 属性之后但在 init 回调（如 或InitializingBean.afterPropertiesSet()自定义 init 方法）之前调用。
     * 在 和 ApplicationEventPublisherAware.setApplicationEventPublisher MessageSourceAware之后ResourceLoaderAware.setResourceLoader调用，如果适用。
     *
     * @param applicationContext 上下文容器
     */
    @Override
    public void setApplicationContext(ApplicationContext applicationContext) throws BeansException {
      	System.out.println("UserLife Bean -- 获取到ApplicationContext -- ApplicationContextAware.setApplicationContext()");
        this.applicationContext = applicationContext;
    }
}

到目前的执行流程:

所有Bean实例化之前 -- BeanFactoryPostProcessor -- postProcessBeanFactory()
UserLife Bean -- 实例化之前 -- InstantiationAwareBeanPostProcessor.postProcessBeforeInstantiation()
UserLife() -- 实例化
UserLife Bean -- 实例化之后 -- InstantiationAwareBeanPostProcessor.postProcessAfterInstantiation()
UserLife Bean -- 对Bean的属性设置(赋值)  -- InstantiationAwareBeanPostProcessor.postProcessProperties()
UserLife Bean -- 获取到BeanName -- BeanNameAware.setBeanName()
UserLife Bean -- 获取到beanFactory -- BeanFactoryAware.setBeanFactory()
UserLife Bean -- 获取到ApplicationContext -- ApplicationContextAware.setApplicationContext()

5. InitializingBean(初始化-接口)

实现该接口,完成Bean的初始化,重写afterPropertiesSet()方法。在Bean的属性设置完成后调用。

实例化前 > 实例化 > 实例化后 > 属性设置 > 《初始化》 > 使用 > 销毁。

@Component
public class UserLife implements InitializingBean {
    public UserLife() {
        System.out.println("UserLife() -- 实例化");
    }

    /**
     * Bean初始化方法
     * 用来在bean的属性被初始化完成后，来执行一些初始化的任务的。
     * 通常在实现该方法的时候，会将一些必要的初始化操作代码放在这个方法中。
     * 以确保bean在使用之前能够得到有效的初始化。
     */
    @Override
    public void afterPropertiesSet() throws Exception {
      System.out.println("UserLife Bean -- 初始化 -- InitializingBean.afterPropertiesSet()");
    }
}

执行顺序:

所有Bean实例化之前 -- BeanFactoryPostProcessor -- postProcessBeanFactory()
UserLife Bean -- 实例化之前 -- InstantiationAwareBeanPostProcessor.postProcessBeforeInstantiation()
UserLife() -- 实例化 -- 执行构造方法
UserLife Bean -- 实例化之后 -- InstantiationAwareBeanPostProcessor.postProcessAfterInstantiation()
UserLife Bean -- 对Bean的属性设置(赋值)  -- InstantiationAwareBeanPostProcessor.postProcessProperties()
UserLife Bean -- 获取到BeanName -- BeanNameAware.setBeanName()
UserLife Bean -- 获取到beanFactory -- BeanFactoryAware.setBeanFactory()
UserLife Bean -- 获取到ApplicationContext -- ApplicationContextAware.setApplicationContext()
UserLife Bean -- 初始化 -- InitializingBean.afterPropertiesSet()

6. @PostConstruct(初始化-注解)

在Bean中,标注了@PostConstruct注解的方法也是初始化方法的一种,并且执行顺序比实现InitializingBean接口更高.

@PostConstruct
public void init(){
  System.out.println("UserLife Bean -- 初始化 -- 标注了@PostConstruct的初始化方法");
}

执行顺序:

UserLife Bean -- 实例化之前 -- InstantiationAwareBeanPostProcessor.postProcessBeforeInstantiation()
UserLife() -- 实例化 -- 执行构造方法
UserLife Bean -- 实例化之后 -- InstantiationAwareBeanPostProcessor.postProcessAfterInstantiation()
UserLife Bean -- 对Bean的属性设置(赋值)  -- InstantiationAwareBeanPostProcessor.postProcessProperties()
UserLife Bean -- 获取到BeanName -- BeanNameAware.setBeanName()
UserLife Bean -- 获取到beanFactory -- BeanFactoryAware.setBeanFactory()
UserLife Bean -- 获取到ApplicationContext -- ApplicationContextAware.setApplicationContext()
UserLife Bean -- 初始化 -- 标注了@PostConstruct的初始化方法
UserLife Bean -- 初始化 -- InitializingBean.afterPropertiesSet()

7. BeanPostProcessor(初始化前后)

实现该接口,可以在每一个Bean的**初始化前以及初始化**后进行扩展。

实例化前 > 实例化 > 实例化后 > 属性设置 > xxxAware > 《初始化前》 > 初始化 > 《初始化后》 > 使用 > 销毁。

@Component
class UserLifeBeanPostProcessor implements BeanPostProcessor{
    /**
     * 每一个Bean初始化之前的回调
     * @param bean     即将要初始化的Bean实例
     * @param beanName 即将要初始化的Bean名称
     * @return 要使用的 Bean 实例，可以是原始实例，也可以是包装的实例;
     */
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
        if (bean instanceof UserLife && beanName.equals("userLife")){
            System.out.println("UserLife Bean -- 初始化之前 -- BeanPostProcessor.postProcessBeforeInitialization()");
        }
        return BeanPostProcessor.super.postProcessBeforeInitialization(bean,beanName);
    }

    /**
     * 每一个Bean初始化之后的回调
     * @param bean     初始化完的Bean实例
     * @param beanName 初始化完的Bean名称
     * @return 要使用的 Bean 实例，可以是原始实例，也可以是包装的实例;
     */
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
        if (bean instanceof UserLife && beanName.equals("userLife")){
            System.out.println("UserLife Bean -- 初始化之后 -- BeanPostProcessor.postProcessAfterInitialization()");
        }
        return BeanPostProcessor.super.postProcessAfterInitialization(bean,beanName);
    }
}

8. DisposableBean(销毁-接口)

实现该接口,重写destroy()方法,Bean销毁时回调该方法。

实例化前 > 实例化 > 实例化后 > 属性设置 > xxxAware > 初始化前 > 初始化 > 初始化后 > 使用 > 《销毁》

@Component
public class UserLife implements DisposableBean {

    /**
     * Bean销毁回调方法
     */
    @Override
    public void destroy() throws Exception {
        System.out.println("UserLife Bean -- 销毁 -- DisposableBean.destroy()");
    }
}

9. @PreDestroy(销毁-注解)

在Bean中,标注了@PreDestroy注解的方法也是回调销毁方法的一种,并且执行顺序比实现DisposableBean接口更高.

@PreDestroy
public void destroy2(){
  System.out.println("UserLife Bean -- 销毁 -- 标注了@PreDestroy注解的销毁方法");
}
