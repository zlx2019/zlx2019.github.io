GRPC-Java篇章

本章记录一下如何使用 SpringBoot3 整合 GRPC，踩了不少坑,主要是版本兼容问题.

首先创建一个SpringBoot项目,如何创建就不多叙说.

1. 依赖配置

配置Protobuf和gRPC的插件,这里我使用的Maven,配置如下:

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.0.2</version>
        <relativePath/> <!-- lookup parent from repository -->
    </parent>
    <groupId>com.zero</groupId>
    <artifactId>springboot-grpc-platfrom</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <name>springboot-grpc-platfrom</name>
    <description>springboot-grpc-platfrom</description>
    <properties>
        <java.version>17</java.version>
        <!--GRPC 版本-->
        <grpc.version>1.51.0</grpc.version>
        <!--Google Proto包版本-->
        <protobuf.version>3.21.7</protobuf.version>
        <!--Protoc 编译器版本-->
        <protoc.version>3.21.7</protoc.version>
    </properties>
    <!--Web-->
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
        <!--SpringBoot gRPC整合包-->
        <!-- https://mvnrepository.com/artifact/io.github.lognet/grpc-spring-boot-starter -->
        <!--这里选用io.github.lognet 的整合包,之前用的net.devh,坑死了-->
        <dependency>
            <groupId>io.github.lognet</groupId>
            <artifactId>grpc-spring-boot-starter</artifactId>
            <version>5.0.0</version>
        </dependency>

        <!--Java与Proto格式化包-->
        <dependency>
            <groupId>com.googlecode.protobuf-java-format</groupId>
            <artifactId>protobuf-java-format</artifactId>
            <version>1.4</version>
        </dependency>
        <!-- 找不到Generated注解时 导入-->
        <dependency>
            <groupId>javax.annotation</groupId>
            <artifactId>javax.annotation-api</artifactId>
            <version>1.3.2</version>
        </dependency>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-configuration-processor</artifactId>
            <optional>true</optional>
        </dependency>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>
    </dependencies>

    <build>
        <!--用于检测当前系统信息,${os.detected.classifier}：这个变量获取操作系统的版本，例如osx-x86_64-->
        <extensions>
            <extension>
                <groupId>kr.motd.maven</groupId>
                <artifactId>os-maven-plugin</artifactId>
                <version>1.7.1</version>
            </extension>
        </extensions>
        <plugins>
            <!--Protobuf和Grpc代码生成插件-->
            <plugin>
                <groupId>org.xolstice.maven.plugins</groupId>
                <artifactId>protobuf-maven-plugin</artifactId>
                <version>0.6.1</version>
                <configuration>
                    <protocArtifact>com.google.protobuf:protoc:${protoc.version}:exe:${os.detected.classifier}</protocArtifact>
                    <pluginId>grpc-java</pluginId>
                    <pluginArtifact>io.grpc:protoc-gen-grpc-java:${grpc.version}:exe:${os.detected.classifier}</pluginArtifact>
                    <!--设置grpc生成代码到指定路径,-->
                    <!--<outputDirectory>${project.basedir}/src/main/java</outputDirectory>-->
                    <!--默认会生成在/src/main/java下，不显示指定则直接编译近target包-->
                    <!--生成代码前是否清空目录-->
                    <clearOutputDirectory>false</clearOutputDirectory>
                </configuration>
                <executions>
                    <execution>
                        <goals>
                            <goal>compile</goal>
                            <goal>compile-custom</goal>
                        </goals>
                    </execution>
                </executions>
            </plugin>
            <!-- jar版本冲突检测插件 -->
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-enforcer-plugin</artifactId>
                <version>1.4.1</version>
                <executions>
                    <execution>
                        <id>enforce</id>
                        <goals>
                            <goal>enforce</goal>
                        </goals>
                        <configuration>
                            <rules>
                                <requireUpperBoundDeps/>
                            </rules>
                        </configuration>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>
</project>

所有的.proto文件都放在src/main/proto目录中,插件会默认编译该文件夹中的所有proto。

2. Proto定义

下面定义一个简单的proto:

syntax = "proto3";

// 引入Google提供好的时间类型
import "google/protobuf/timestamp.proto";
import "google/protobuf/empty.proto";

// Go生成的package 第一个`member`表示在--go_out=指定的目录下创建一个member文件夹,该文件的package为member
option go_package = "./member;member";

// java生成的的package路径
option java_package = "com.zero.pb.member";
// 指定java类名
option java_outer_classname = "Member";
// 是否生成的所有java文件 合并到一个文件内
option java_multiple_files = true;
// 是否生成定义的service服务,如果为false，那怕定义了也不会生成.
option java_generic_services = true;
option optimize_for = SPEED;

// 请求参数
message MemberRequest{
  sint64 id = 1;
}
// 请求返回参数
message MemberReply{
  string user_name = 1;
  string pass_word = 2;
}

// 定义Member的服务
service MemberService{
  //定义接口
  rpc GetMember(MemberRequest)returns(MemberReply);
}

然后进行编译

mvn clean compile -DskipTests=true

然后会在target包中创建如下文件:

├── com
│   └── zero
│       └── pb
│           ├── member
│           │   ├── Member.class
│           │   ├── MemberReply.class
│           │   ├── MemberReplyOrBuilder.class
│           │   ├── MemberRequest.class
│           │   ├── MemberRequestOrBuilder.class
│           │   ├── MemberService.class
│           │   └── MemberServiceGrpc.class

3. 服务提供

服务提供

如果作为服务端,那么就需要继承MemberServiceGrpc.MemberServiceImplBase类,重写它的getMember(MemberRequest,responseObserver)方法.

@GRpcService
public class UserProvider extends MemberServiceGrpc.MemberServiceImplBase {
    /**
     * GetMember接口重写
     *
     * @param request   请求参数
     * @param responseObserver  响应参数
     */
    @Override
    public void getMember(MemberRequest request, StreamObserver<MemberReply> responseObserver) {
        // 获取请求参数的值
        long id = request.getId();
        System.out.println("GetMember API ID:" + id);
        // 构建响应参数
        MemberReply reply = MemberReply.newBuilder()
                .setUserName("admin")
                .setPassWord("root").build();

        // 如果业务错误,则响应异常
        // responseObserver.onError(new RuntimeException("xxxx"));

        // 然后将结果响应给客户端
        responseObserver.onNext(reply);
        responseObserver.onCompleted();
    }
}

注意: 这里是服务提供,一定要用 @GRpcService注解标注.

然后通过IDEA的插件进行测试即可.

GRPC localhost:6565/MemberService/GetMember

{"id":1}

结果如下:

<Response body is empty>

4. 服务调用

如果时作为消费者,如下调用服务即可:

@Component
public class UserClient {

    public void getMember(){
        // 获取连接通道
        ManagedChannel channel = ManagedChannelBuilder.forAddress("localhost", 6565).usePlaintext().build();
        // 构建客户端
        MemberServiceGrpc.MemberServiceFutureStub stub = MemberServiceGrpc.newFutureStub(channel);
        // 构建参数
        MemberRequest request = MemberRequest.newBuilder()
                .setId(100001).build();
        // 调用接口
        stub.getMember(request);
    }
}
