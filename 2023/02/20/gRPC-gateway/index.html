GRPC-Gateway语言篇章

1. 前言

GRPC-Gateway 能帮助你同时提供 gRPC 和 RESTful 风格的 API。GRPC-Gateway 是 Google protocol buffers 编译器 protoc 的一个插件。它读取 Protobuf 服务定义并生成一个反向代理服务器，该服务器将 RESTful HTTP API 转换为 gRPC。该服务器是根据服务定义中的 google.api.http 注释生成的。

下面就简单实现一个gRPC-gateway.

2. 搭建gateway

0. 复习定义Protobuf

// 指定proto语言版本
syntax = "proto3";

// 生成go文件的包路径;包的别名
option go_package = "./pb/student;student";

// proto的包路径
package gateway.proto.student;


message Student{
  uint64 id = 1;
  string username = 2;
  string password = 3;
  string email = 4;
  Sex sex = 5;
}

enum Sex{
  UNKNOWN = 0;
  MAN = 1;
  WOMAN = 2;
}

当前目录如下:

.
├── Makefile
├── go.mod
└── proto
    └── student.proto

然后进行简单编译一下,通过如下命令:

protoc -I=. --go_out=. --go-grpc_out=. ./proto/student.proto

简单解释一下:

  • -I=: 表示指定项目根目录,.表示当前就目录就是根目录(可以不加此参数).

  • --go_out=: 表示生成的go文件放在哪个目录,.表示直接放在当前目录即可。会根据proto文件中的go_package创建对应的包目录。如option go_package = "/pb/student;student";就会在当前目录创建一个pb/student的目录,然后生成的go文件都会放在这个目录。

  • --go-grpc_out=: 生成的go语言的grpc文件放在哪个目录,其规则与--go_out参数一致.

  • 最后指定的是要编译的proto文件

    生成后的结构如下:

    .
├── Makefile
├── go.mod
├── pb
│   └── student
│       └── student.pb.go
└── proto
    └── student.proto

1. 安装所需依赖

在根目录创建tools.go,指定依赖

package tools

import (
	_ "github.com/grpc-ecosystem/grpc-gateway/v2/protoc-gen-grpc-gateway"
	_ "github.com/grpc-ecosystem/grpc-gateway/v2/protoc-gen-openapiv2"
	_ "google.golang.org/grpc/cmd/protoc-gen-go-grpc"
	_ "google.golang.org/protobuf/cmd/protoc-gen-go"
)

然后执行go mod tidy.

然后执行如下命令,下载到本地的$GOPATH/bin下.

go install \
    github.com/grpc-ecosystem/grpc-gateway/v2/protoc-gen-grpc-gateway \
    github.com/grpc-ecosystem/grpc-gateway/v2/protoc-gen-openapiv2 \
    google.golang.org/protobuf/cmd/protoc-gen-go \
    google.golang.org/grpc/cmd/protoc-gen-go-grpc

2. 导入所需的proto

想让grpc能转换为http接口,需要依赖以下两个proto文件.

http.proto

annotations.proto

创建google.api目录,将这两个文件下载到该目录.

2. 定义Protobuf

接下来创建自己的业务Proto文件

// 指定proto语言版本
syntax = "proto3";
// 生成go文件的包路径;包的别名
option go_package = "/pb/student;student";
// proto的包路径
package gateway.proto.student;
import "google/protobuf/empty.proto";


// 导入google的http 和 annotations
// gateway必须依赖这两个
import "google/api/http.proto";
import "google/api/annotations.proto";



// 查询请求参数
message QueryRequest{
  uint64 id = 1;
}

// 新增请求参数
message SaveRequest{
  uint64 id = 1;
  string username = 2;
  string password = 3;
  string email = 4;
}

// 单个返回值参数
message StudentReply{
  uint64 id = 1;
  string username = 2;
  string password = 3;
  string email = 4;
}
// 序列返回值参数
message StudentListReply{
  repeated StudentReply students = 1;
}

// 定义服务
service StudentService{

  // 获取所有学生信息
  rpc Students(google.protobuf.Empty)returns(StudentListReply){
    // 映射为 Get HTTP
    // Get /v1
    option(google.api.http) = {
      get: "/v1"
    };
  }

  // 根据id获取学生信息
  rpc GetStudent(QueryRequest)returns(StudentReply){
    // 映射为 Get HTTP
    // Get /v1/xxx
    // 通过uri传参,id会自动映射到QueryRequest.id字段
    option(google.api.http) = {
      get: "/v1/{id}"
    };
  }
  // 新增学生信息
  rpc SaveStudent(SaveRequest)returns(google.protobuf.Empty){
    // 映射为 Post HTTP
    // Post /v1  {id:xxx,username:xxx,...}
    // body: * 表示将HTTP的body允许接收任何值,最终会根据字段名映射到SaveRequest的字段中.
    option(google.api.http) = {
      post: "/v1",
      body: "*"
    };
  }
}

请仔细观察,与以往的Proto文件有何不同?

区别就是这里引入了http.proto和annotations.proto文件。

并且通过这两个文件,分别对不同的rpc接口,设置了不同的http接口映射.

如:

// 获取所有学生信息
rpc Students(google.protobuf.Empty)returns(StudentListReply){
  // 映射为 Get HTTP
  option(google.api.http) = {
    get: "/v1"
  };
}

这样,后面就可以通过 http://localhost:xxx/v1 直接访问到这个grpc的接口.

接下来编译该proto文件:

protoc -I=./proto --go_out=./  --go-grpc_out=./  --grpc-gateway_out ./  ./proto/student.proto

这傻逼命令把我整了好久…日你野🐴的…

--xxx_out=: 分别指将go、grpc、grpc-gateway代码生成在哪个目录.

重点来了!!!

-I=: 指定源文件目录,从哪个目录开始寻找Proto依赖.由于我们这里不是在所有Proto文件所在的根目录,所以需要通过-I指定我们的Proto文件源目录.否则很可能找不到引用的其他依赖!!! 详细解释如下:

由于我们在proto中引用了其他目录的proto文件,所以我们要通过-I指定从哪个基础目录开始找所依赖的proto文件.

比如依赖了google/api/http.proto。但是对于根目录来说,它的位置是/proto/google/api/http.proto。

所以编译时会显示找不到该文件: proto/student.proto:12:1: Import "google/api/http.proto" was not found or had errors.

因为我们当前编译的位置是 ./proto/google/api/http.proto。

所以使用-I指定寻找依赖的基础目录为./proto,这样拼接后就成了./proto/google/api/http.proto,这样就编译成功了.

我直接将命令写到的Makefile文件中:

# 生成go、go-grpc、grpc-gateway文件
build:
	protoc -I=./proto --go_out=./  \
		  	--go-grpc_out=./  \
		  	--grpc-gateway_out ./  \
		  	./proto/student.proto

然后开始实现Proto定义的方法

type StudentService struct {
	student.UnimplementedStudentServiceServer
}

// Students 获取学生列表
func (*StudentService) Students(ctx context.Context, e *emptypb.Empty) (*student.StudentListReply, error) {
	i := 1
	listReply := &student.StudentListReply{}
	for {
		istr := strconv.Itoa(i)
		u := "username" + istr
		p := "password" + istr
		e := "email" + istr
		student := &student.StudentReply{Id: uint64(i), Username: u, Password: p, Email: e}
		listReply.Students = append(listReply.Students, student)
		i++
		if i > 10 {
			break
		}
	}
	//return nil, status.Errorf(codes.Unimplemented, "method Students not implemented")
	return listReply, nil
}

// 根据id获取学生信息

然后是开启Grpc服务与Gateway服务

func main() {
	go StartGRPC()
	go StartGateway()
	select {}

}

// StartGateway 启动Gateway
func StartGateway() {
	// 创建grpc的client,指定服务端地址
	clientConn, err := grpc.DialContext(context.Background(),
		"127.0.0.1:8888", grpc.WithBlock(),
		grpc.WithTransportCredentials(insecure.NewCredentials()))
	if err != nil {
		panic(err)
	}
	// 创建一个对外开放的HTTP服务
	mux := runtime.NewServeMux()

	// 创建gateway 服务
	gatewayServer := &http.Server{Addr: ":9999", Handler: mux}

	// 注册服务映射处理
	_ = student.RegisterStudentServiceHandler(context.Background(), mux, clientConn)
	// 启动服务
	err = gatewayServer.ListenAndServe()
	if err != nil {
		panic(err)
	}
}

func StartGRPC() {
	// 创建tcp服务
	listen, err := net.Listen("tcp", ":8888")
	if err != nil {
		panic(err)
	}
	// 创建grpc服务
	grpcServer := grpc.NewServer()
	// 注册服务
	student.RegisterStudentServiceServer(grpcServer, &StudentService{})
	// 启动grpc服务
	err = grpcServer.Serve(listen)
	if err != nil {
		panic(err)
	}
}
