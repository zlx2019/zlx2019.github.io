Java【动态代理】

发表于2023-02-15|更新于2023-02-16
|字数总计:2.5k|阅读时长:9分钟|阅读量:

1. 代理模式

代理模式提供了对目标对象额外的访问方式,即为目标对象创建代理对象。然后通过代理对象访问目标对象,这样可以在不修改原目标对象的前提下,提供额外的功能扩展。

2. 三种代理模式

  • 静态代理
  • 动态代理
  • Cglib动态代理

3. 静态代理

静态代理是最基础的代理模式之一。它可以做到在不修改目标对象的情况下,通过代理对象增强扩展。一般都是指通过硬编码的方式实现,及其笨重,一般情况下不推荐使用。下面来简单实现一个最最简单的静态代理。

原理: 静态代理需要将具体要扩展或增强的功能方法抽象到一个接口中,然后由目标对象和代理对象都实现该接口,然后由代理对象负责调用目标对象,我们就通过代理对象调用目标对象前后写额外的操作。

  1. 创建功能抽象接口

    interface Animal{
    void call();
}

  2. 创建目标对象

    class Cat implements Animal{
    @Override
    public void call() {
        System.out.println("我是目标对象,我要喵喵喵~");
    }
}

    如果不使用代理模式,其实很简单,我们直接实例化一个Cat对象,然后执行call()方法即可.如下:

    public static void main(String[] args) {
  Animal cat = new Cat();
  cat.call();
}

    输出结果:

    我是目标对象,我要喵喵喵~

    然后我们加入静态代理进行扩展,分别在目标对象执行前后提示一句话即可.

  3. 创建代理对象

    class CatProxy implements Animal{
    // 实际要被代理的目标对象
    private Animal animal;
    // 通过构造函数将目标对象传递进来
    public CatProxy(Animal animal) {
        this.animal = animal;
    }
    @Override
    public void call() {
        // 进行增强扩展
        System.out.println("目标对象要执行了~");
        animal.call(); //让目标对象执行
        System.out.println("目标对象已经执行完成~");
    }
}

    然后使用静态代理方式执行:

    public static void main(String[] args) {
  // 使用静态代理方式
  // 1.创建目标对象
  Animal animal = new Cat();
  // 2.创建代理对象,并且将目标对象传递给代理对象
  CatProxy proxy = new CatProxy(animal);
  // 3.通过代理对象,执行目标对象
  proxy.call();
}

    输出结果:

    目标对象要执行了~
我是目标对象,我要喵喵喵~
目标对象已经执行完成~

一个简单的静态代理就这样实现了.

但是我们可以发现,这种方式及其笨重,而且会产生很多代理类.

不易维护,一旦新增一个方法,那么目标对象和代理对象都要进行修改.

4. 动态代理

动态代理又分为两种实现,一种是基于JDK中的Proxy对象实现,另一种是基于第三方包Cglib实现。

首先我们了解一下JDK实现的动态代理。

注意,JDK的动态代理只能生成接口的代理。

  1. 先创建一个接口.

    interface Animal{
    void call();
    String getName();
}

  2. 通过Proxy.newProxyInstance()方法为Animal创建代理对象

    先来看一下所需参数:

    public static Object newProxyInstance(ClassLoader loader, //: 用于加载代理类的类加载器
                                      Class<?>[] interfaces,//要创建的代理对象的.Class
                                      InvocationHandler h)// 代理对象调用处理程序 实现的接口
  throws IllegalArgumentException

    具体如下使用:

    public static void main(String[] args) {
  // 创建Animal代理对象
  Animal instance = (Animal) Proxy.newProxyInstance(Thread.currentThread().getContextClassLoader(), new Class[]{Animal.class}, new InvocationHandler() {

    /**
             * 执行目标对象方法的处理方法.
             * 被代理对象的每一个方法被调用时,都会在这里经过
             *
             * @param proxy 当前的代理对象
             * @param method 此次要执行的目标对象方法
             * @param args 方法参数
             */
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
      System.out.println(method.getName() + "()方法被执行");
      return null;
    }
  });
  // 通过代理对象 执行目标对象的方法
  instance.call();
  instance.getName();
}

    输出结果:

    call()方法被执行
getName()方法被执行

    也就是说,被代理对象的方法被执行前,都会进入invoke()处理方法中.我们就可以在这里统一增强扩展.

    当然单纯这样使用时没有意义的,我们通常是通过一个接口的代理对象去动态执行接口的子类方法.

通过接口的代理对象,执行子类实现的方法

  1. 创建一个子类Cat实现于Animal.

    class Cat implements Animal{
    private String name;
    public Cat(String name) {
        this.name = name;
    }
    @Override
    public void call() {
        System.out.println("我是"+ name +",喵喵喵~");
    }
    @Override
    public String getName() {
        return name;
    }
}

  2. 创建Animal代理对象,执行Cat实现的方法.

    Animal cat = new Cat("加肥猫~");
Animal instance = (Animal) Proxy.newProxyInstance(Thread.currentThread().getContextClassLoader(), new Class[]{Animal.class}, (proxy, method, args1) -> {
  // 执行方法,获取返回值
  Object result = method.invoke(cat, args1);
  return result;
});
instance.call();
System.out.println(instance.getName());

    输出结果:

    加肥猫~
我是加肥猫~,喵喵喵~

    这是JDK动态代理的简单使用。

抽象一个动态代理工厂工具

通常我们都不会一个一个对象去代理,这太麻烦了,我们通常会进行一个通用化的处理.如实现一个工厂:

public class ProxyFactory<T> {
    /**
     * 要创建代理的目标对象
     */
    private final T target;
    public ProxyFactory(T target) {
        this.target = target;
    }
    public Object getProxyInstance(){
        return Proxy.newProxyInstance(target.getClass().getClassLoader(), target.getClass().getInterfaces(), (proxy, method, args) -> {
            System.out.println("方法开始执行,开启事物...");
            Object resultValue = method.invoke(target);
            System.out.println("方法执行结束,提交事物...");
            return resultValue;
        });
    }
}

使用该工厂:

// 创建Animal接口 具体是Cat实现
Animal cat = new Cat("汤姆");
// 将Animal接口传入工厂
ProxyFactory<Animal> factory = new ProxyFactory<>(cat);
// 获取Animal接口实例,实际上是Cat
Animal instance = (Animal) factory.getProxyInstance();

System.out.println(instance.getName());
System.out.println("======");
instance.call();

输出结果:

方法开始执行,开启事物...
方法执行结束,提交事物...
汤姆
======
方法开始执行,开启事物...
我是汤姆,喵喵喵~
方法执行结束,提交事物...

5. cglib动态代理

(Code Generation Library )是一个第三方代码生成类库，运行时在内存中动态生成一个子类对象从而实现对目标对象功能的扩展。

  • JDK的动态代理有个限制,那就是只能通过接口实现代理,如果想代理没有实现接口的类,则需要使用到cglib。
  • CGLIB是一个强大的高性能的代码生成包，它可以在运行期扩展Java类与实现Java接口。
    它广泛的被许多AOP的框架使用，例如Spring AOP和dynaop，为他们提供方法的interception（拦截）。
  • CGLIB包的底层是通过使用一个小而快的字节码处理框架ASM，来转换字节码并生成新的类。
    不鼓励直接使用ASM，因为它需要你对JVM内部结构包括class文件的格式和指令集都很熟悉。
  • 使用cglib代理,可以实现代理类无侵入。
  • cglib不能为一个被final修饰的类创建代理对象.

使用cglib需要引入它的jar包:

<dependency>
  <groupId>cglib</groupId>
  <artifactId>cglib</artifactId>
  <version>3.3.0</version>
</dependency>

如果你是Spring环境,则不需要导入,Spring-core中包含cglib的依赖.

理论不多说,上面已经说的很清楚了,直接上案例.

通过cglib代理一个类对象

  1. 创建一个基础类

    class Student{
    private String name;
    private int age;
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
    public Student() {
    }
    public void greet(){
        System.out.println("Hi Hello I Name Is " + this.name);
    }
    public void sayAge(){
        System.out.println("i age is" + age);
    }
    public void eat(String food){
        System.out.println("我最爱吃" + food);
    }
}

  2. 使用cglib代理并且使用

    public static void main(String[] args) {
    //1. 创建想被代理的对象实例
    Student student = new Student("小张", 18);
    //2. 创建cglib代理工具类
    Enhancer enhancer = new Enhancer();
    //3. 设置被代理类的Class类型
    enhancer.setSuperclass(Student.class);
    //4. 设置方法回调处理拦截器,类似于Proxy中的invoke(),但是比它更强大
    enhancer.setCallback(new MethodInterceptor() {
        /**
         *
         * @param obj 被代理的对象
         * @param method 拦截到的方法
         * @param args 方法参数
         * @param proxy used to invoke super (non-intercepted method); may be called
         * as many times as needed
         */
        @Override
        public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy) throws Throwable {
            System.out.println("================");
            System.out.println("执行" + method.getName() + "()方法");
            System.out.println("参数为:" + JSONUtil.toJsonStr(args));
            Object resultValue = method.invoke(student,args);
            if (resultValue != null){
                System.out.println("返回值为:" + JSONUtil.toJsonStr(resultValue));
            }
            System.out.println("================");
            return resultValue;
        }
    });
   
    //5. 创建代理对象
    Student instance = (Student)enhancer.create();
    instance.greet();
    instance.sayAge();
    instance.eat("汉堡");
}

    输出结果:

    ================
执行greet()方法
参数为:[]
Hi Hello I Name Is 小张
================
================
执行sayAge()方法
参数为:[]
i age is18
================
================
执行eat()方法
参数为:["汉堡"]
我最爱吃汉堡
================

    可以看到使用cglib代理,更为简单方便,并且更强大.

使用cglib创建一个动态代理工厂

/**
 * 基于cglib的动态代理工厂
 *
 * @author Zero.
 * @date 2023/2/16 12:35 AM
 */
public class ProxyFactory<T> implements MethodInterceptor {
    // 被代理的目标对象
    private final T target;
    public ProxyFactory(T target) {
        this.target = target;
    }
    /**
     * 方法拦截器
     * @param obj 当前的代理对象
     * @param method 拦截到的方法
     * @param args 方法参数
     * @param proxy 用于调用超级（非拦截方法）;可以根据需要调用多次
     * @return 方法执行结果
     */
    @Override
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy) throws Throwable {
        System.out.println("==================");
        System.out.println("开启事物....");
        //Object resultValue = method.invoke(target, args);
        // 或者
        Object resultValue = proxy.invoke(target, args);
        System.out.println("提交事物....");
        System.out.println("==================");
        System.out.println();
        return resultValue;
    }
    /**
     * 创建 target 的代理对象并且返回
     */
    public T getProxyInstance(){
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(target.getClass());
        enhancer.setCallback(this);
        return (T) enhancer.create();
    }
}

使用:

Student instance = new ProxyFactory<Student>(new Student("张三", 16)).getProxyInstance();
instance.greet();
instance.sayAge();
instance.eat("肯德基");

输出结果:

==================
开启事物....
Hi Hello I Name Is 张三
提交事物....
==================

==================
开启事物....
i age is16
提交事物....
==================

==================
开启事物....
我最爱吃肯德基
提交事物....
==================

这就是基于cglib的动态代理,再次提示一下: 被final修饰的类不能被代理哦!
