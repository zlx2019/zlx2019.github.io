Vue3学习之旅【三】Vue路由

发表于2021-04-30|更新于2021-05-07|Vue系列
|字数总计:1.3k|阅读时长:5分钟|阅读量:

Vue-Router

1. 安装

  • npm安装

    npm install vue-router --save

  • Yarn 安装

    yarn add vur-router

2.配置路由规则

/** Vue路由配置*/
/** 导出需要的方法*/
import {createRouter,createWebHistory} from 'vue-router'
/** 导入需要路由的组件*/
import Home from '@/views/Home'

//创建路由规则
const routers = [
  {
    path: '/', //触发的path
    name: 'Home',  //名称
    component: Home//路由的组件
  },
  {
    path: '/about',
    name: 'About',
    //路由组件懒加载模式
    component: ()=> import('@/views/About')
  },
  {
    path: '/music',
    name: 'Music',
    component: ()=> import('@/views/Music')
  }
]

//创建路由器
const router = createRouter({
  /*路由模式
    vue提供三种模式
       Hash: 使用Url的hash值作为路由,url携带#
       History: 以HTML5 HistoryAPI和服务器配置
       Abstract: 支持所有js运行模式
  */
  history: createWebHistory(process.env.BASE_URL),
  //指定路由规则
  routes:routers
})

export default router

3.集成到Vue中

import { createApp } from 'vue'
import App from './App.vue'
import router from './router'
import store from './store'

createApp(App).use(store).use(router).mount('#app')

4.使用方式

需要通过两个标签来使用:

<router-link>

<router-view>

Code: 

<template>
  <div id="nav">
    <router-link active-class="active" to="/">Home</router-link> |
    <router-link to="/about">About</router-link> |
    <router-link to="/music">Music</router-link> |

    <!--自定义路由触发-->
    <button @click="$router.push('/custom')">custom</button> |

    <!--获取当前路由的path-->
    {{$route.path}}

  </div>
  <router-view/>
</template>

5.多个router-view 渲染不同的组件

  • 定义路由规则

    {
  /*渲染多层组件,默认为Home组件, Name:组件*/
  path: '/parent',
  name: 'parent',
  components:{
    default: Home,
    Abouts: ()=> import('@/views/About'),
    Musics: ()=> import('@/views/Music'),
    Customs: ()=> import('@/views/Custom')
  }
}

  • vue中使用

    <template>
  <div id="nav">
   <!--路由到/parent,该规则下有三个组件,需要通过router-view 的name属性指定对应的子组件名称-->
    <router-link to="/parent">Parent</router-link>
  </div>
  
  <!--通过name属性,指定要渲染的组件-->
  <router-view name="Abouts"/>
  <router-view name="Musics"/>
  <router-view name="Customs"/>
</template>

4.子组件

  • 规则定义

    {
  /*子组件*/
  path: '/level',
  component: () => import('@/components/parent/Level'),
  children: [
    {
      /* 默认路由*/
      path: '',
      component: () => import('@/components/parent/salve/One')
    },
    {
      path: 'one',
      component: ()=> import('@/components/parent/salve/One')
    },
    {
      path: 'two',
      component: () => import('@/components/parent/salve/Two')
    }
  ]
}

  • 顶级父组件中使用Level

    <template>
	<router-link to="/level">level</router-link>
  <router-view/>
</template>

  • Level组件中使用子组件

    <template>
  <div>
    <h2>顶级父组件</h2>
    <router-link to="/level/one">one组件</router-link> |
    <router-link to="/level/two">two组件</router-link>
    <router-view/>
  </div>
</template>

5. 路由传递Rest 风格参数

  • 规则定义

    指定传递的属性名称为id,可以在路由后的页面通过$route获取到

    {
path: '/level',
component: () => import('@/components/parent/Level'),
children: [
        {
        path: 'page/:id',
        component: () => import('@/components/parent/salve/Page')
      	}
   	]
}

  • 跳转时携带参数

    <ul>
  <li v-for="item in books">
    <router-link :to="`/level/page/${item.id}`">{{ item.title }}</router-link>
  </li>
</ul>

  • 获取携带后的参数

    <template>
<div>
  <!--通过$route获取路由中的参数  -->
  <h2>路由传递的参数id:{{$route.params.id}}</h2>
</div>
</template>


或者,通过method或计算属性获取
<script>
export default {
  name: 'Params',
  computed:{
    getParams(){
      return  this.$route.params.id
    }
  }
}
</script>

6.路由传递Json对象参数

  • 规则定义

    {
path: '/level',
component: () => import('@/components/parent/Level'),
children: [
      {
        path: 'book',
        component: () => import('@/components/parent/salve/Book')
      }
   ]
}

  • 路由携带参数,直接将:to绑定为一个对象,带有path、query

    <ul>
  <li v-for="item in books">
    <router-link :to="{path: `/level/book`,query: item}">{{ item.title }}</router-link>
  </li>
</ul>

  • 获取携带的json对象

    <template>
<div>
  <h2>Book页面</h2>
  传递过来的query对象 {{$route.query}}

  <h3>title:{{$route.query.title}}</h3>
  <h3>id:{{$route.query.id}}</h3>
</div>
</template>

7.全局导航守卫

  • 导航守卫,主要是在路由前、后，进行拦截,并且处理相关的操作
/** 全局前置守卫,返回True才会进行路由
 * to: 要跳转到的路由
 * from: 从哪个路由跳转过来的
 * */
router.beforeEach((to,from,next)=>{
  //案例1,给每一种图书展示时修改网页标题
  if (to.path === '/level/book'){
    document.title = to.query.title
  }

  /** 如果是登录,则不需要校验登录信息*/
  if (to.path === '/login'){
    next()
  }

  // 案例2,每次路由前都检查是否持有token,否则就跳转到login页面
  const token = localStorage.getItem("token")
  if (token === '' || token === null){
    next('/login')
  }
  next()
})

/** 全局后置守卫
 * to: 要跳转到的路由
 * from: 从哪个路由跳转过来的
 * */
router.afterEach((to,from,next)=>{
  return true
})

8.keep-alive的使用

  • 实现组件之间的缓存,不会重复销毁创建组件

    Vue3 Router和keep-alive使用方法,可以通过exclude属性排除不激活的组件

    <!-- router 和 keep-alive 的整合使用 -->
  <router-view v-slot="{Component}">
    <transition>
      <keep-alive exclude="User">
        <component :is="Component"/>
      </keep-alive>
    </transition>
  </router-view>

  • 使用keep-alive后,不会触发 unmounted 钩子函数,并且只会触发一次created钩子函数, 只会不断触发 activated、deactivated 钩子函数
