Go【四】并发编程

发表于2021-08-22|更新于2023-03-15|Go
|字数总计:729|阅读时长:2分钟|阅读量:

前言

高并发编程是Go的一大特性也是该语言的最大特点,在原生的开发包中,提供了sync包,里面包含了很多涉及到并发编程相关工具。

1. Mutex 互斥锁

锁是通常我们用来解决并发安全性的最常用的方案之一。

锁的种类有很多,如我们常说的独占锁、共享锁、乐观锁、悲观锁、自旋锁、读写锁等等,本质上这些锁很多指的都是同一种锁,只是叫法不同,其思想及实现基本一致.

sync.Mutex是Go语言提供的最基础的锁,也就是互斥锁

lock := &sync.Mutex{} //获取一把锁
lock.Lock()						//进行加锁
lock.Unlock()					//释放锁

2. RWMutex 读写锁

读写锁又名共享锁,其本质就是在不同的性质操作中,添加不同的锁,可以高效提供程序的执行效率。

比如多个协程并发读取共享数据,因为是读取操作,无论有多少协程并行,都不会存在安全性问题,这种情况下,如果使用互斥锁会造成一定的性能浪费,我们可以通过读写锁,它能够更友好、更高效。

读写锁分为读锁与写锁,互斥关系如下

  1. 读锁与读锁之间不会互斥,他们是共享的。

  2. 写锁与写锁会互斥阻塞

  3. 写锁与读锁会互斥阻塞

所以一般如果读取操作远远大于写操作的场景下,使用RWMutex比Mutex更有优势

readWriterLock := &sync.RWMutex{} //获取一把读写锁
readWriterLock.RLock() //加读锁
readWriterLock.RUnlock() //释放读锁

readWriterLock.Lock() //加写锁
readWriterLock.Unlock() //释放写锁

3. Once 单例执行器

使用 sync.Once 对象可以使得函数多次调用只执行一次。

once := &sync.Once{}
once.Do(func() {
  fmt.Println("0")
})

once.Do(func() {
  fmt.Println("1")
})

通过Do函数传入要执行的函数,该函数全局只会执行一次

4. WaitGroup 计数器

话不多说,与Java中的CountDownLatch一致

如我们启动n个协程,我们希望主线程等待所有协程执行完成后再结束,通过WaitGroup即可实现

func main() {
	i := 10
	waitGroup := &sync.WaitGroup{}
	waitGroup.Add(i) //设置计数量

	for j := 0; j < i; j++ {
		go func(num int) {
			fmt.Println(num)
			waitGroup.Done() //计数 -1
		}(j)
	}

	waitGroup.Wait()//阻塞等待所有协程完成
}

5. Map 并发容器

没啥可说的,就是一个线程安全的Map

m := &sync.Map{}
m.Store("key1","value1") //设置元素
m.Load("key1") //获取元素
m.LoadAndDelete("key1") //获取并且移除掉该元素
m.LoadOrStore("key2","value2")//根据key获取元素,如果不存在就设置新值
m.Delete("key1")//移除元素

//遍历
m.Range(func(key, value interface{}) bool {
  fmt.Println(key,value)
  //只有返回true才会继续遍历下一组K-V
  return true
})
