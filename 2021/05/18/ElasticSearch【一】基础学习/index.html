前言

什么是ElasticSeatch?

ElasticSearch 简称ES,是目前非常主流的一款分布式开源搜索引擎,它基于Lucene 并且使用Java开发,提供了简单的Restful风格API;
ES主要以轻量级的JSON作为数据存储格式,这点与MongoDB有点类似,但是读写性能上优于mongo,它是一个接近实时的搜索平台

1.什么是全文检索?

全文检索是计算机程序通过扫描文章中的每一个词,对每一个词建立一个对应的索引,该索引指明该词在文章中出现的次数和位置,当用户查询词时,只要查找到建立好的索引即可

检索分为两步: 索(先建立索引) 检(再根据索引去查找)

2.什么是Restful?

定义: 资源的表现层状态转化 其实Restful是一种软甲架构风格

关键词详解:

  • 资源(Resource)

    网络中的一切都称之为资源,如一首歌、一张图片、数据库中的一条数据,API响应的一段JSON数据等等,每一个资源都应该具有一个唯一的资源标识符;那就是对应它自己的 URL,拿WebAPI接口举例,我们想获取一个用户的信息

    传统WebApi: http://localhost:8080/user/findOne?id=1001

    可以看出,传统WebAPi是通过’?’拼接携带参数id,其实这种情况下只有’?’前面的内容才是真正的URL,’?’之后的只是携带的参数并不属于URL内容,假如我们获取多个用户信息:

    http://localhost:8080/user/findOne?id=1001

    http://localhost:8080/user/findOne?id=1002

    http://localhost:8080/user/findOne?id=1003

    以restful来看,这三种’资源’的唯一资源标识符其实是相同的(http://localhost:8080/user/findOne),但是他们获取到的资源却是不同的,违背的restful的意愿.但是如果换成Restful风格后的API呢?

    RestfulAPI:

    http://locahost:8080/user/findOne/1

    http://locahost:8080/user/findOne/2

    http://locahost:8080/user/findOne/3

    将携带的id参数作为URL的内容,从而达到资源标识符的唯一性!

  • 状态转化(State Transfer)

    通过不同的状态转化,传达不同的操作,Restful提出了四种新的请求动词: GET(查询)、POST(更新|添加)、PUT(添加|更新)、DELETE(删除)

总结: 

Restful其实就是一种WebAPI接口的定义风格,它的原则就是一个URL只代表了一个唯一的资源,并且呢,通过http协议里面的四种动词去对应资源的基本操作(CRUD),GET用来获取资源、POST用来新增资源、PUT用来更新资源、DELETE用来删除资源。

3.ElasticSearch的基本概念

  • 索引(Index)

    索引库,类似于关系型数据库的一个database,索引名称必须为小写

  • 类型(Type)

    一个索引可以有n个类型,类似于数据库中的表(7版本中已经移除)

  • 映射(Mapping)

    类型的结构信息

4.ES倒排索引原理

ES从插入数据到检索数据的流程原理:

首先获取到一条文档后,会在元数据区持久化该文档,并且生成一个DocID,可以根据该DocId直接命中该文档在元数据区的位置,然后对文档的所有数据进行分词建立索引,
			假如只有一个名为title的text类型数据,我们来模拟多条数据
				"title": "Running code open"  	docID :1
				"title": "Running start first" 	docID :2
				"title": "test Running"			docID :3
			首先会对title进行分词,然后es会在索引区创建以下矩形(概念模型)
			
			分词后的相同单词都会有一个唯一标识
				docID: 文档对应的唯一标识(可以理解为Doc地址)
				POS: 在文档中出现的位置(field index)
				TF: 在文档中出现的次数
				倒排项(概念模型)
				-------------------------------------------------
				单词ID	单词内容		倒排列表(docID,<POS>,TF)
				1		Running		(1,<0>,1), (2,<0>,1), (3,<1>,1)
				2		code     	(1,<1>,1)
				3		open   		(1,<2>,1)
				4		start		(2,<1>,1)
				5		first		(2,<2>,1)
				6		test 		(3,<0>,1)

				--------------------------------------------------
			如以上结构,以单词1Running为例: 表示 'Running'单词
			在id为 '1' 的文档中 '<0>' 的位置出现了'1'次,
			在id为 '2' 的文档中 '<0>' 的位置出现了'1'次,
			在id为 '3' 的文档中 '<1>' 的位置出现了'1'次
		假如我们根据'Running'term查询,es会首先在列表中寻找该单词ID,然后在对应的倒排列表中获取它出现的DocID、位置、TF等数据,就可以直接命中该文档
		es中使用的数据结构: BKD Tree(自行扩展)

5.ElasticSearch的安装

  1. 首先下载安装包 https://www.elastic.co/cn/downloads/elasticsearch

    目前比较比较主流的版本为7.x以上

  2. 然后解压会有以下目录

    bin、config、jdk、plugins、lib、modules、logs、data

    • bin是可执行文件目录
    • config为配置目录
    • plugins为插件目录
    • data为数据存储目录
    • logs为日志目录
    • lib为依赖包目录

  3. 环境配置

    修改/etc/security/limits.conf,增加配置，用户退出后重新登录生效

    *               soft    nofile            655350
*               hard    nofile            655350

    修改/etc/security/limits.conf,增加配置，用户退出后重新登录生效

    *               hard    nproc           4096
*               soft    nproc           4096

    修改/etc/sysctl.conf，增加如下配置,保存后执行sysctl -p 生效

    fs.file-max=655350

    修改/etc/sysctl.conf，增加如下配置,保存后执行sysctl -p 生效

    vm.max_map_count=262144

  4. 开启远程连接

    修改elasticsearch.yml

    network.host: 0.0.0.0

  5. 启动 -d 表示后台运行

    bin/elasticsearch -d

  6. 如果启动失败可以调整jvm的大小,防止内存不够而导致无法使用

    config/jvm.options

6. Es的常用数据类型

1.文本类型

  • text
  • keyword

2.数字类型-8种

类型 说明
byte 有符号的8位整数, 范围: [-128 ~ 127]
short 有符号的16位整数, 范围: [-32768 ~ 32767]
integer 有符号的32位整数, 范围: [−231−231 ~ 231231-1]
long 有符号的64位整数, 范围: [−263−263 ~ 263263-1]
float 32位单精度浮点数
double 64位双精度浮点数
half_float 16位半精度IEEE 754浮点类型
scaled_float 缩放类型的的浮点数, 比如price字段只需精确到分, 57.34缩放因子为100, 存储结果为5734

3.日期类型

  • Date

JSON没有日期数据类型, 所以在ES中, 日期可以是:

包含格式化日期的字符串, “2018-10-01”, 或”2018/10/01 12:10:30”.

代表时间毫秒数的长整型数字.

代表时间秒数的整数.

4. 布尔类型

  • boolean

7.ES基础语句

1.查看所有索引库

GET /_cat/indices

2.创建索引库

books 为索引库名称

PUT /books
{
  "mappings": {
    "properties": {
          "title": {"type": "keyword"},
          "price":{"type": "double"},
          "content":{"type": "text"},
          "createTime":{"type": "date"}
    }
  }
}

3.插入数据

POST /books/_doc/1
{
  "title": "Golang微服务",
  "price": 18.99,
  "content":"Golang微服务进阶...",
  "createTime": "2020-08-20"
}

如果不指定ID,则会默认生成

POST /books/_doc
{
  "title": "gRPC",
  "price": 180.99,
  "content":"高性能RPC框架",
  "createTime": "1999-02-20"
}

4.查看索引库数据

GET /books/_search

5.查看索引库mapping信息

GET /books/_mapping

6. 删除文档

删除id为1的文档

DELETE /books/1

7.更新文档

更新id为1的文档,会覆盖掉原来的数据,只留下title字段

POST /books/_doc/1
{
  "title": "新的title"
}

更新指定的字段

POST /books/_doc/1/_update
{
  "title": "只更新了title"
}

8.批量插入

PUT /books/_bulk
{"index": {"_id": 1002}}
{"title": "批量插入的书籍2","price": 10.9,"content": "内容2"}
{"index": {"_id": 1003}}
{"title": "批量插入的书籍3","price": 10.9,"content": "内容3"}
{"index": {"_id": 1004}}
{"title": "批量插入的书籍4","price": 10.9,"content": "内容4"}

9.批量操作(新增、修改、删除)

“index”表示新增

“update”表示修改

“delete”表示删除

三者可以混合使用

PUT /books/_bulk
{"index": {"_id": 1005}}
{"title": "批量插入的书籍5","price": 10.9,"content": "内容5"}
{"update":{"_id": 1001}}
{"doc": {"title": "批量修改title"}}
{"delete": {"_id": 1002}}

8.DSL查询语句

1.match_all 查询全部数据

GET /books/_search
{
  "query": {
    "match_all": {}
  }
}

2. Sort 排序查询

  • Desc 降序
  • Asc 升序
GET /books/_search
{
  "query": {
    "match_all": {}
  },
  "sort": [
    {
      "price": {
        "order": "desc"
      }
    }
  ]
}

3.分页查询

  • from 表示偏移量
  • size 表示页容量
GET /books/_search
{
  "query": {"match_all": {}},
  "from": 0, 
  "size": 2
}

4. 查询只获取指定的字段

GET /books/_search
{
  "query": {"match_all": {}},
  "_source": ["title"]
}

5. 匹配查询

5.1 match 模糊匹配查询

只要包含关键字 就表示匹配成功

  • text类型才会分词匹配
  • keyword类型 按照整体来匹配
GET /users/_search
{
  "query": {
    "match": {
      "username": "name"
    }
  }
}

5.12term 精确匹配查询

  • text类型才会分词匹配
  • keyword 类型 按照整体来匹配
GET /books/_search
{
  "query": {
    "term": {
      "content": {
        "value": "1"
      }
    }
  }
}

6.范围查询

  • gte : >
  • let : <
GET /books/_search
{
  "query": {
    "range": {
      "price": {
        "gte": 10,
        "lte": 20
      }
    }
  }
}

7. 匹配前缀查询

GET /books/_search
{
  "query": {
    "prefix": {
      "title": {
        "value": "Go"
      }
    }
  }
}

8. 通配符查询

GET /books/_search
{
  "query": {
    "wildcard": {
      "title": {
        "value": "北*"
      }
    }
  }
}

9. 根据一组id查询

GET /books/_search
{
  "query": {
    "ids": {
      "values": [1001,1003]
    }
  }
}

10. 布尔查询(组合查询)

  • must: and
  • shoud: or
  • must_not: not
#must使用,查询price 10-100区间、并且createTime小于2021日期的数据
GET /books/_search
{
  "query": {
    "bool": {
      "must": [
        {
          "range": {
            "price": {
              "gte": 10,
              "lte": 100
            }
          }
        },
        {
          "range": {
            "createTime": {
              "lte": "2021-01-01"
            }
          }
        }
      ]
    }
  }
}


# should使用,查询price大于200 或者小于30的数据
GET /book/books/_search
{
  "query": {
    "bool": {
      "should": [
        {
          "range": {
            "price": {
              "gte": 100
            }
          }
        },
        {
          "range": {
            "price": {
              "lte": 30
            }
          }
        }
      ]
    }
  }
}

# must_not使用,排除掉price为180.99的数据
GET /book/books/_search
{
  "query": {
    "bool": {
      "must_not": [
        {
          "match": {
            "price": 180.99
          }
        }
      ]
    }
  }
}

11. 高亮查询

在 “1” 前后追加 span 标签

GET /books/_search
{
  "query": {
    "term": {
      "content": {
        "value": "1"
      }
    }
  },
  "highlight": {
    "pre_tags": ["<span style='color:red'>"], 
    "post_tags": ["</span>"], 
    "fields": {
      "content": {}
    }
  }
}

12. 多字段查询

根据gRpc 关键词在 title、content字段中匹配

GET /books/_search
{
  "query": {
    "multi_match": {
      "query": "gRpc",
      "fields": ["title","content"]
    }
  }
}

13. filter过滤器

#查询 '服务' 并且 price小于20的
GET /book/books/_search
{
  "query": {
    "bool": {
      "must": [
        {
          "term": {
            "title": {
              "value": "服务"
            }
          }
        }
      ],
      "filter": {
        "range": {
          "price": {
            "lte": 20
          }
        }
      }
    }
  }
}

14. 聚合查询

根据price 进行分组统计

GET /books/_search
{
  "aggs": {
    "price_group": {
      "terms": {
        "field": "price"
      }
    }
  }
}

9. IK分词器

IK分词器提供了两种分词

  • ik_max_word

    最细粒度分词

  • Ik_smart

    最粗粒度分词

1. 创建索引库时,指定分词器

PUT /users
{
  "mappings": {
    "properties": {
      "username" : {
        "type": "text",
        "analyzer": "ik_max_word"
      },
      "nikename": {
        "type": "text",
        "analyzer": "ik_smart"
      }
    }
  }
}

2. 插入数据

POST /users/_doc/1001
{
  "username": "i name is 管理员",
  "nikename" : "管理员"
}

3.分词查询

GET /users/_search
{
  "query": {
    "term": {
      "username": {
        "value": "管理"
      }
    }
  }
}
