vue3学习之旅【五】CompositionAPI

发表于2021-05-05|更新于2021-05-07|Vue系列
|字数总计:1.9k|阅读时长:8分钟|阅读量:

前言：CompositionAPI的出现原因

Composition API是Vue3中核心逻辑及代码组织方法。Vue核心团队将Composition API描述为：一组基于功能的附加API，可以灵活的组合组件的逻辑。在基于Vue2开发项目时，通过methods、watch、data等组件属性实现页面逻辑的方式称之为Option API。Option API模式不仅导致组件业务分散、产生胶水代码，而且编写的代码也需要Vue编译器将JS代码转换为真正的执行代码。Composition API通过将组件属性作为JS函数暴露出来以解决上述问题，通过Composition API实现的业务代码无需编译器中间处理，同时TypeScript的类型检查机制可以更好的保障代码的健壮。

1.原生Vue 与 CompositionAPI开发的区别

两种方式完成一个案例: 定义一个变量’count’,通过计算属性获取’count’的二倍,并且定义方法使’count’递增

  • 原生Vue
<template>
  <div class="home">
    <h2>原生Vue开发案例</h2>
    value: {{count}}<br>
    通过计算属性获取Val的二倍: {{getValDouble}}<br/>
    通过方法获取Val的二倍并且转成字符串: {{getValStr('Hello')}}<br>
    <button @click="count++">+1</button>
  </div>
</template>

<script>
export default {
  name: 'Home',
  data(){
    return{
      count: 1
    }
  },
  computed:{
    getValDouble(){
      return this.count * 2
    }
  },
  methods:{
    getValStr(str){
      return  this.getValDouble + str
    }
  }
}
</script>

  • CompositionAPI开发

    <template>
  <div class="about">
    <h1>CompositionAPI开发案例</h1>
    value:{{data.count}}<br>
    计算属性获取:{{getDouble}}<br>
    <button @click="add">+1</button>
  </div>
</template>

<script>

import {ref,reactive,computed} from 'vue'
export default {
  setup(){
    /** 定义响应式变量*/
    let data = reactive({
      count: 1
    })

    /** 定义计算属性*/
    const getDouble = computed(()=> data.count * 2)

    /** 定义方法*/
    function add(){
      data.count++
    }

    /** 统一导出,才能使用*/
    return{data,getDouble,add}
  }
}
</script>

2. Setup

setup()函数是Vue3新增的函数,是CompositionApi的入口,执行周期是在beforeCreate之前。
所以setup执行时,组件还未创建,不存在this的概念,所以也就无法访问组件的data数据
  • setup执行时机

  • setup可选参数

    setup(props,context){
  console.log('setup')
}
    • props: 父组件传递的参数
    • Context: 上下文对象,包含以下属性
      • context.attrs :获取父组件传递的并且 没有在子组件props定义的数据
      • context.slots : 获取插槽信息
      • context.parent :获取父组件
      • context.root :获取根组件
      • context.emit :向父组件派发事件
      • context.refs

3. CompositionAPI 常用的API

3.1 ref

通过vue提供的 ref 创建响应式变量

<script>
  /** 导入一些可能会用到的API*/
  import {reactive,ref} from 'vue'
export default {
name: "ComApi",
  setup(){
    let count = ref(0) //创建一个响应式对象
    //定义方法
    let modify = (val)=>{
      //如果要修改响应式对象,必须通过.value
      count.value = val
    }
    //必须返回抛出去,模板上才能用
    return{
      count,modify
    }
  }
}
</script>
3.2 reactive

通过vue提供的 ref 创建响应式对象

<script>
  /** 导入一些可能会用到的API*/
  import {reactive,ref} from 'vue'
export default {
name: "ComApi",
  setup(){
    let obj = reactive({ //创建一个响应式对象
      name: 'lxz',
      age: 18,
      sex: '男'
    })

    //定义修改响应式对象的方法
    let modifyObj = ()=>{
      obj.name = 'www.lxz.zone'
    }

    return{
      obj,modifyObj
    }
  }
}
</script>
3.3 toRefs

将原生对象包装成响应式对象

3.4 readonly

将响应式对象还原成原生对象

3.5 computed 计算属性
<script>
  /** 导入一些可能会用到的API*/
  import {reactive,ref,computed,toRefs,readonly} from 'vue'
export default {
name: "ComApi",
  setup(){
  /** 定义响应式对象*/
  const userInfo = reactive({
      name: '张三',
      passwd: 'root'
    })
    /** 定义计算属性*/
    const getUserInfo = computed(()=>{
      return userInfo
    })
    return{
      userInfo,
      getUserInfo
    }
  }
}
</script>
3.6 watch 监听器

默认监听 监听器代码块使用的所有对象

<script>
  /** 导入一些可能会用到的API*/
  import {reactive,ref,computed,watch} from 'vue'
export default {
name: "ComApi",
  setup(){
  /** 定义响应式对象*/
  let userInfo = reactive({
      name: '张三',
      passwd: 'root'
    })

    /** 定义监听器,监听userInfo的变化  immediate:是否监听执行初始值,启动时是否执行一次*/
    watch(userInfo,(newUserInfo)=>{
      console.log(newUserInfo.name)
      console.log('userInfo 发生了变化!')
    },{immediate: true})
    
    /** 监听多个值*/
    watch([user1,user2],([new1,new2])=>{
      
    })
    

    /** 定义方法修改userInfo*/
    const modify = function (){
      let str = Math.random().toString(10);
      userInfo.name = str
    }

    return{
      userInfo,
      getUserInfo,
      modify
    }
  }
}
</script>

4. CompositionAPI的声明周期

<script>
/** 导入setup的钩子函数*/
import {onBeforeMount, onMounted, onBeforeUpdate, onUpdated,onBeforeUnmount,onUnmounted} from 'vue'

export default {
  name: "SetUpPeriod",
  setup() {
    onBeforeMount(() => {
      console.log("1.onBeforeMount 挂载之前...")
    })
    onMounted(() => {
      console.log("2.onMounted 挂载后...")
    })
    onBeforeUpdate(() => {
      console.log('onBeforeUpdate 修改数据之前...')
    })
    onUpdated(()=>{
      console.log('onUpdated 修改数据后...')
    })
    onBeforeUnmount(()=>{
      console.log('3.onBeforeUnmount 销毁之前...')
    })
    onUnmounted(()=>{
      console.log('4.onUnmounted 销毁后...')
    })
  }
}
</script>

5.非响应式组件通信

利用provide(父组件)和inject(子组件)的完成通信(此方式非响应式)

  • 父组件定义

    <script>
export default {
  name: "Root",
  data() {
    return {
      title: 'Root组件定义的title'
    }
  },
  provide(){
    /** 这里声明的数据,可以在子组件中获取到*/
    return {
      title: this.title
    }
  }
}
</script>

  • 子组件定义

    <template>
  <div>
    <h2>子组件</h2>
    子组件获取父组件的数据:{{title}}
  </div>
</template>

<script>
export default {
  name: "Child",
  data() {
    return {
    }
  },
  inject:['title']
}
</script>

6.CompositionAPI -响应式组件通信

  1. 通过Vue3提供的’ref’构建响应式数据
  2. 在父组件中通过Vue3提供的’provide’向子孙组件传递数据
  3. 在子孙组件中通过Vue3提供的“inject”函数,根据父组件传递的数据key 获取值

父组件定义

<script>
import Child from "./Child";
import {ref,reactive,provide} from 'vue'
export default {
  name: "Root",
  components: {Child},
  setup(){
    /** 定义响应式数据*/
    let message = ref('Root组件定义的message')
    let obj = reactive({
      name: '呵呵'
    })
    /** 向子组件传递数据,格式为: key,value*/
    provide("message",message)
    provide("obj",obj)
    return {message}
  },
}
</script>

子组件定义

<template>
  <div>
    <h2>子组件</h2>
    子组件获取父组件的message:{{message}}<br>
    子组件获取父组件的obj:{{obj}}
  </div>
</template>

<script>
import {inject} from 'vue'
export default {
  name: "Child",
  setup(){
    /** 根据key 获取父组件传递的响应式数据*/
    let message = inject('message')
    let obj = inject('obj')
    return{
      message,
      obj
    }
  }
}
</script>

7.CompositionAPI 整合Router的使用

​ 通过vue-router包提供的’useRoute’,’useRouter’, 在setup中获取Router,并且获取参数

<script>
import {useRoute, useRouter} from 'vue-router'
export default {
  name: "RouterApi",
  setup() {
    const router = useRouter();
    const route = useRoute();
    let id = route.params.id;
    let query = route.query;
  }
}
</script>

但是,setup函数只会执行一次,如果是动态传参(resuful)形式,则需要通过监听器来使用(watch)

如下:

<script>
import {useRoute, useRouter} from 'vue-router'
import {watch,ref} from 'vue'
export default {
  name: "RouterApi",
  setup() {
    const router = useRouter();
    const route = useRoute();
    let id = ref()

    /** 监听params的变化,一旦更改后,将新值赋予给id(响应式变量)*/
    watch(route.params,(newParams)=>{
      console.log(newParams.id)
      id.value = newParams.id
    })
    return{id}
  }
}
</script>
7.1 组件级的导航守卫
<script>
import {onBeforeRouteLeave,onBeforeRouteUpdate} from 'vue-router'
export default {
  name: "RouterApi",
  setup() {
    /** 导航离开该组件的对应路由时调用*/
    onBeforeRouteLeave((to,from,next)=>{
      return window.confirm(`你确定要从${from.fullPath}离开,到${to.fullPath}吗?`)
    })

    /**
     * 在当前路由改变，但是该组件被复用时调用
     * 举例来说，对于一个带有动态参数的路径 /foo/:id，在 /foo/1 和 /foo/2 之间跳转的时候，
     * 由于会渲染同样的 Foo 组件，因此组件实例会被复用。而这个钩子就会在这个情况下被调用。
     * 可以访问组件实例 `this`
     * */
    onBeforeRouteUpdate((to,from,next)=>{

    })
    return{}
  }
}
</script>

8.CompositionAPI 整合Vuex的使用

​ 通过Vuex提供的 ‘useStore’ 在setup中获取Store

<script>
import {useStore} from 'vuex'
export default {
  name: "StoreApi",
  setup(){
    //获取Store
    const store = useStore();
    function modify(num){
      //派发事件
      store.commit('modify',num)
    }
    return{modify}
  }
}
</script>
