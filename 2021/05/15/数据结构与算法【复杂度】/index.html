前言

一种功能的实现可以有n多个算法去解决,但是每一种算法的效率有所不同,如下功能,将用两种同的算法来完成,分别比较下效率,两种效率的差异就是算法的复杂度高低之分

1.求出第n个斐波那契数

  • 什么是斐波那契数?

    从第一项数值开始,每相近的两项数进行相加,得出第三项数值,然后由第二项和第三项相加,以此类推,直至获取到第n项数值如:

    0	1	1	2	3	5	8	13....
---------------------原理如下
0+1=1
1+1=2
2+1=3
2+3=5
3+5=8
5+8=13
....
第一种算法

Java实现

/***
 * 求出第n项斐波那契数值
 * @Author lx Zhang.
 * @Date 2021/5/15 9:07 上午
 * @Param [n]
 * @return int
 **/
public static int fib(int n){
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}

测试

System.out.println(fib(5));
System.out.println(fib(30));
System.out.print(fib(40));
System.out.print(fib(50));

结果:

5 				消耗时长:1ms
832040		消耗时长:7ms
102334155	消耗时长:369ms
未得出结果.....

可以看出来,随着我们参数n的增大,消耗的时间成倍的增长,当n仅仅为60时,竟然无法算出结果,测试电脑为Mac m1 16G配置….

接下来我们换go语言实现试试

/**
	求出第n项斐波那契数值
 */
func fib(n int64) int64{
	if n <= 1 { return n}
	return fib(n - 1) + fib(n - 2)
}

测试

fmt.Println(fib(5))
fmt.Println(fib(15))
fmt.Println(fib(30))
fmt.Println(fib(40))
fmt.Print(fib(50))

结果

5       		消耗时长 0ms
610     		消耗时长 0ms
832040  		消耗时长 0ms
102334155   消耗时长 1ms
....（n为50时）

可以看出来go语言确实快了不少,但是随着参数的变大还是存在问题,显然是这个算法本身的问题

第二种算法

Java实现

public static int fib(int n) {
    if (n <= 1) return 1;
    int first = 0;
    int second = 1;
    for (int i = 0; i < n - 1; i++) {
        int sum = first + second;//先求出每两项之和
        //数值项相加后进行后移
        first = second; //上一轮的第二项,后移一位,成为下一轮的第一项
        second = sum;   //将求得的和赋值于本轮的第二项,成为下一轮的第二项
    }
    return second;//直至找到第n项
}

测试

System.out.println(fib(5));
System.out.println(fib(30));
System.out.print(fib(40));
System.out.print(fib(50));

结果

5						消耗时长:0ms
832040			消耗时长:0ms
102334155		消耗时长:0ms
1820529360	消耗时长:0ms

第二种算法明显比第一种算法效率要高得多,这就是不同的算法,不同的复杂度

2.计算从1到 n的和
第一种算法
public static int sum(int n){
    int result = 0;
    for (int i = 0; i <= n; i++) {
        result += i;
    }
    return result;
}
第二种算法
public static int sum2(int n){
    return (1 + n) * n / 2;
}

明显第二种使用到了公式,各方面都比第一种优雅、复杂度低

3.如何评判一个算法的好坏?

一般从下维度来评估一个算法的优劣

  • 正确性、可读性、健壮性(必要条件)

在符合以上三类条件后,再根据以下维度评判

  • 时间复杂度
    • 估算程序指令执行的次数(执行时间)
  • 空间复杂度
    • 估算所需占用的内存空间
